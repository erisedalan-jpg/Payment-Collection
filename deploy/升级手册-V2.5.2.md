# 升级手册 —— V2.5.2（全局 Apple 风微交互/质感打磨，纯前端增量；从 V2.5.1 升级）

## 0. 本次升级包含什么（V2.5.2，全局展示风格打磨·Apple 风微交互/质感）

只调**展示形式**（不改任何数据/口径/页面结构），全站统一一层更精致的交互质感：

1. **可点 KPI 卡悬浮上浮**：鼠标悬停可点的指标卡（各页顶部统计卡）时，卡片轻微上浮 + 柔和阴影；键盘聚焦显柔光焦点环。
2. **按钮按下回弹**：全站按钮按下时轻微缩放回弹（触觉反馈）。
3. **顶栏「数据已同步」脉冲点**：同步指示点加"呼吸"脉冲动画（用既有"已回款"绿，不引新色）。
4. **分段控件精修**：顶栏「浅色/深色」「小/中/大」切换由旧「实底深蓝 + 白字」改为更清爽的"抬起淡底芯片"（深蓝字 + 微阴影），同时修正了"实底 + 小号白字"的旧观感问题。
5. **侧栏当前项更清晰**：左侧导航选中项由极淡底色改为蓝色淡底 + 左侧 2px 蓝色指示条，一眼可辨当前页。
6. **弹窗聚焦**：打开弹窗/抽屉时，其背后的页面轻微模糊以聚焦弹窗（**弹窗本身保持不透明、文字清晰**）。
7. **细节**：高分屏（retina）下顶栏/侧栏分隔线细化为 0.5px，更接近桌面 App 质感。

> 全部动效遵循系统「减弱动态效果 / 减弱透明度」偏好自动降级；配色仍是本平台锁定的深蓝品牌色板与回款/风险状态色，**未引入任何新颜色**。

> **【头号注意】本次升级无需点「更新数据」。** 本期为**纯前端样式**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何口径，前端复用现有 `data/analysis_data.json` 即可。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面/无新页面访问 key、未改 `auth.py`（超管不变）。**
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.5.2.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.5.2.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本期与 V2.5.1 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.5.2/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本期核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.5.2/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本期无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧样式：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。
