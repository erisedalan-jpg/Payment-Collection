## 0. 本次升级包含什么（V2.5.5 → V2.5.7，三项·均纯前端）

### 来自 V2.5.7（本次）
1. **列筛选「搜索」交互修复（全站列表页，9 处）**：此前在任一列表的列头筛选框里输入搜索内容后，**必须先点「取消全选」把全部勾选清掉，再手工勾搜索出来的选项，筛选才生效**（否则直接确定 = 全选 = 无筛选）。本次修复为：**搜索后直接勾选搜索结果、点确定即生效，无需再先「取消全选」**。搜索状态下「全选 / 勾选 / 确定」都只作用于当前搜索结果（主勾选框会提示「(搜索结果)」）；不搜索时行为不变。
2. **售前服务类项目「总预算 / 已核算 / 剩余预算」展示修正（/insight/costdetail 明细）**：此前部分售前项目「成本状态=超支」但「剩余预算」显示为 0 或正数（看起来矛盾）。根因是售前项目的成本核算挂在原项目上、当前项目侧无核算数据，而超支金额来自另一套（毛利）口径。本次修正为：**超支的售前项目，剩余预算显示为负数（= 超支额）、已核算显示为对应的实际核算额（非 0）**，与「成本状态=超支」一致；无任何成本数据的售前仍显示 0（正常）。

### 来自 V2.5.6（本包一并含入，此前未部署）
3. **/insight/costdetail 成本状态口径统一**：把明细表「成本状态」列、「超支项目分布」图、「L4 部门成本情况汇总」的**超支判定统一到与顶部四卡 /「项目总览」首页 /「项目清单」一致的口径**（此前它们用的是另一套"剩余预算<0"的旧口径，导致同页「总成本超支数」卡与明细筛选出的超支项目数对不上）。升级后：在明细表把「成本状态」列筛选去掉「未超支」，得到的条数 = 顶部「总成本超支数」卡的数字，与首页、项目清单一致。同时**移除了对 XS 前缀项目的底层硬编码特殊处理**，XS 项目是否纳入统计，统一由「数据管理」页的**标签排除**开关控制（更可控、无隐藏规则）。

> **【头号注意】本次升级无需点「更新数据」。** 三项均为**纯前端**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何数据来源。前端复用现有 `data/analysis_data.json` 即可。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面/无新页面访问 key、未改 `auth.py`（超管不变）。**
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.5.7.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.5.7.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本包与 V2.5.5 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.5.7/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本包核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.5.7/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本包无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧样式：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。
