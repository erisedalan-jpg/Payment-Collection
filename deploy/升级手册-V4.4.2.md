# 升级手册 —— V4.4.2（修 PMIS「下载数据」并发冲突时前端静默闪退；**纯前端修复**）

> 适用：已用 /pm 部署在 Ubuntu 服务器（nginx:80 → app 127.0.0.1:8080，systemd，`/opt/pmplatform`，venv `/opt/pmplatform/.venv`）。
> 本包仍是**全量后端快照**（glob 打包全部根 `*.py`），但**后端逻辑较 V4.4.1 无任何改动**——本次修复只在前端 `dist`。
> 随附最小更新包：`pmplatform-update-V4.4.2.zip`。

---

## 0. 本次修什么

**症状**：两个管理员先后点「下载数据」时，**后点击的人**看不到任何进度、也不知下载脚本有没有被调用，面板一闪就消失（"闪退"）。

- **根因**：下载运行槽 `download_state` 是**服务器全局单例、所有管理员共用一个**。当 A 的下载还没跑完，B 再点，后端抢不到槽 → 走 `_json_response` 回一段**普通 JSON**（`Content-Type: application/json`），**不是** SSE 流。而前端旧代码一律按 SSE 逐行找 `data:` 帧 → 一帧都解析不出 → 提示文字为空；进度面板因 `running` 秒置 false、`progress` 停在 0 而**闪现即消失**。于是 B 侧＝"无内容 + 不知脚本是否被调 + 闪退"。
- **修法（前端最小）**：`usePmisDownload` fetch 后按 `content-type` 分流——是 `application/json`（忙分支）就直接解析并把冲突提示显示出来（"已有下载正在进行，请等其完成后再试（当前：…）" / 透传"其他数据操作进行中"），且不误触发"下载完成"回调；进度块补 `dlMessage` 显示条件，让忙/错误消息（含此前同样看不见的 `下载失败`/`无响应体`）真正可见。

> **重要澄清**：这次并发**不是数据 bug**。真正把 PMIS 数据落到 `input/` 的，是**先点、且跑完整条流水线**的那位管理员；被挡下的一次没下载任何东西。现网 `input/pmis/` 就是先点那位的结果，要重算看板照常点一次「更新数据」即可。

| 内容 | 需更新数据? |
|---|---|
| 「下载数据」并发冲突前端提示修复 | **否**（纯前端，不碰数据管线） |

> ### 头号注意
>
> **① 本次无需点「更新数据」。** 只换 `dist`（+可选覆盖后端，逻辑没变）、重启 app、浏览器 `Ctrl+F5` 硬刷即可。
>
> **② 换 dist 后务必硬刷。** 新前端资源带新哈希文件名，`Ctrl+F5` 清缓存才看得到 V4.4.2；否则可能仍加载旧 JS。
>
> **③ 既有账号/数据零影响。** 本包不含 `data/`/`input/`；账号、`yitian_store.json`、`holidays.csv`、PMIS 输入均不动。

## 1. 升级步骤（Ubuntu，约 3 分钟）

```bash
# 0) 上传并解压
cd /tmp && unzip -o pmplatform-update-V4.4.2.zip

# 1) 备份 dist（后端逻辑没变,可只备 dist;要稳妥也一并备份 .py）
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS

# 2) 覆盖前端 dist（本次核心）
cd /tmp/pmplatform-update-V4.4.2
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) （可选）全量覆盖后端 .py —— 逻辑与 V4.4.1 一致,覆盖只为保持文件同版
sudo cp -f *.py /opt/pmplatform/

# 4) 重启后端
sudo systemctl restart pmplatform
sudo systemctl status pmplatform --no-pager | head -5

# 5) 浏览器 Ctrl+F5 硬刷 /pm,左上角版本号应为 V4.4.2
```

## 3. 回滚

```bash
TS=<你升级时用的时间戳>
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-$TS /opt/pmplatform/frontend/dist
sudo systemctl restart pmplatform
```

> 回滚只换回 dist（若你第 3 步覆盖了 .py，另从 `bak-$TS` 还原即可）；`data/`/`input/` 不受影响。
