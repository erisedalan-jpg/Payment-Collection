## 0. 本次升级包含什么（V2.6.5 → V2.6.6，纯前端性能修复）

**修复"平台整体较严重卡慢"。** 真机取证定位到两个叠加根因，均已修复：

### 根因 1 —— 列筛选弹层预渲染滞留（主凶）

各列表页表头的「列筛选 ▼」和「选列」弹层，此前**页面一打开就把整列的全部筛选选项（可达数百项）预先渲染并挂到页面底层**，且随页面缓存跨页滞留——访问过成本分析等页后，**每个页面都拖着约 2.5 万个隐藏元素**，全站每次渲染/交互都被拖慢。
**修复**：弹层改为**点开才渲染、关闭即销毁**。列筛选/选列功能与交互完全不变（超长选项列表首次点开约有极短渲染，属预期）。

### 根因 2 —— 页面缓存越积越多（越用越卡）

V2.5.9 引入的"下钻返回保持视图状态"缓存机制，菜单每次进入列表页都会产生新实例，而**旧实例成为永远用不到的死缓存**（上限 10 个）。实测浏览一段时间后浏览器内存从 15MB 涨到 **220MB**、滞留 13 万个游离节点和 7.8 万个事件监听器——**用得越久越卡**。
**修复**：缓存上限 10→**2**（下钻返回保持只需要最近 1 个缓存，留 1 余量）。"下钻返回保持状态、菜单进入重置"的行为**完全不变**。

### 实测效果（4x CPU 降速模拟慢机，修复前→后）

- 满缓存内存：220.4MB → **40.1MB**；游离节点 13.4万 → 2.2万；监听器 7.8万 → 5,869
- 页面隐藏元素：在建项目页 27,062 → **1,993**
- 页面进入阻塞：成本分析 6.3s → **1.0s**；商机清单 4.1s → 1.7s；回款项目 4.0s → 2.1s；重点项目进展 4.8s → 2.3s；在建项目 3.4s → 2.0s；回款节点 2.2s → 1.0s

> **【头号注意 1】本次升级无需点「更新数据」。** 未改数据契约（schema）、未改数据处理（preprocess）、未改任何数据来源与后端接口。
>
> **【头号注意 2】纯前端修复，无需授权账号、无新增/删除页面、无新增依赖。**
>
> **【头号注意 3】本包未改后端 `server.py`（后端 `*.py` 与 V2.6.5 相同，一并覆盖仅为省事）。** 因此**替换前端 dist + 浏览器强制刷新即可生效**；systemd 重启一步做不做都行（做了无副作用）。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**建议先备份**。

```bash
# (1) 备份现有前端（用于回滚；纯前端修复，备份 dist 即可）
sudo tar czf /backup/pmplatform-dist-$(date +%F-%H%M).tgz -C /opt/pmplatform frontend/dist

# (2) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.6.6.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.6.6.zip').extractall('.')"

# (3) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.6.6/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   后端 *.py 与 V2.6.5 相同，可跳过覆盖；如要保持一致：
#   sudo cp /tmp/pmplatform-update-V2.6.6/*.py /opt/pmplatform/

# (4) 归属仍归 pmapp
sudo chown -R pmapp:pmapp /opt/pmplatform/frontend/dist

# (5) 前端 dist 按请求读磁盘、即时生效，无需重启；如已覆盖 *.py 或想保险可重启：
# sudo systemctl restart pmplatform
```

> **本包无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需授权账号。**
> 浏览器请**强制刷新（Ctrl+F5）**加载新前端。
