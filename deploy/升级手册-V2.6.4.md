## 0. 本次升级包含什么（V2.6.3 → V2.6.4，回款完成率口径修复）

**纯口径缺陷修复，无功能增删、无界面重排。** 修复「回款完成率」在选了日期区间（默认区间=本年度）时显示偏低/为 0 的问题。

**背景（问题现象）**：进「回款多维分析 `/insight/board`」下钻，某项目完成率显示 **0%**，但进该项目详情页却是 **70%**，两处对不上。排查发现：看板与回款各页的「完成率」被算成了「**所选区间内的回款流水 ÷ 全额合同**」——分子被日期区间过滤、分母不缩放。由于默认区间是**本年度（2026）**，而大量项目的回款发生在**2025 年**，这些项目的完成率就被压成了 0%/偏低（实测约 38% 的有合同项目受影响），与项目详情页、首页的「累计完成率」矛盾。

**本次修复（回款完成率/已回款恒按累计口径）**：把下列各处的「回款完成率」「已回款额」统一改为 **CLAUDE 全站统一口径 = 累计回款流水（全部时间，逐笔全加）÷ 合同总额**，不再随所选日期区间变化，与「项目详情页」「首页 KPI」完全一致：
- 「回款多维分析 `/insight/board`」——分组表「完成率」列、下钻弹窗「完成率」列（连带「回款状态」同步为累计口径，不再出现「完成率 70% 却显示未回款」）。
- 「回款项目 `/payment/projects`」——「已回款(万)」「完成率」两列。
- 「回款总览 `/payment`」——下方「回款数据」L4 表的「已回款(万)」「回款额完成率」列，以及顶部 KPI 卡的「已回款(万)」「完成率」。

**日期区间仍然有效，只是不再作用于「已回款/完成率」**：日期区间继续作用于「**计划回款、待回款、延期、回款节点计数**」这些按节点计划日归属的指标（本区间应回/待回的口径不变）。也就是说：**已回款额、回款完成率 = 全部累计**；**计划/待回款/延期/节点 = 所选区间**。

> **【头号注意 1】本次升级无需点「更新数据」。** 未改数据契约（schema）、未改数据处理（preprocess）、未改任何数据来源，也未改后端接口。前端复用现有 `data/analysis_data.json`。
>
> **【头号注意 2】纯前端修复，无需给账号授权、无新增/删除页面、无新增依赖。**
>
> **【头号注意 3】本包未改后端 `server.py`（后端 `*.py` 内容与 V2.6.3 相同，一并覆盖仅为省事）。** 因此**替换前端 dist + 浏览器强制刷新即可生效**；systemd 重启一步做不做都行（做了无副作用）。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**建议先备份**。

```bash
# (1) 备份现有前端（用于回滚；纯前端修复，备份 dist 即可）
sudo tar czf /backup/pmplatform-dist-$(date +%F-%H%M).tgz -C /opt/pmplatform frontend/dist

# (2) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.6.4.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.6.4.zip').extractall('.')"

# (3) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.6.4/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   后端 *.py 与 V2.6.3 相同，可跳过覆盖；如要保持一致：
#   sudo cp /tmp/pmplatform-update-V2.6.4/*.py /opt/pmplatform/

# (4) 归属仍归 pmapp
sudo chown -R pmapp:pmapp /opt/pmplatform/frontend/dist

# (5) 前端 dist 是按请求读磁盘、即时生效，无需重启；如已覆盖 *.py 或想保险可重启：
# sudo systemctl restart pmplatform
```

> **本包无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需授权账号。**
> 浏览器请**强制刷新（Ctrl+F5）**加载新前端。
