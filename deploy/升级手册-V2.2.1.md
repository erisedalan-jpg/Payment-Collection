# 升级手册 —— V2.2.1（回款明细改「元」+ 商机「是否重大POC」列）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的旧版本升级到 **V2.2.1**。
> 本包是「**纯代码 + /pm 前端 dist**」的轻量增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.2.1.zip`。
> 现网若早于 V2.2.0，本包代码为**全量覆盖所有 .py + dist**（升到 V2.2.1 无遗漏），但被跳过版本的**首用/数据/依赖步骤**仍需照对应手册补做（如 V2.2.0 的「重点商机跟进」页访问授权、V2.1.1 的下载依赖 `requests`/`openpyxl` 与 pmisdata、V2.1.0 的临时跟进、V2.0.0 的商机 seed 与「更新数据」）。

## 0. 本次升级包含什么

V2.2.1 是**子页面/页内局部级（Z 级）**版本。相对 V2.2.0，本轮两块：

- **`/project/:id` 回款明细表金额改「元」**：
  - 项目详情页「回款」子页的**回款节点明细表**中，「**计划回款**」「**已收**」「**未收**」三列由原来的「万元」改为以「**元**」展示原始金额。
  - 原因：旧版以万元为单位、保留两位小数四舍五入，部分金额会丢精度（如 123,456 元显示为 12.35 万）；改后直接显示精确的元值。
  - 「**账期(天)**」列（天数，非金额）与「回款」子页**顶部汇总卡片**（合同总额/流水累计/计划回款，仍按万元）**保持不变**。该口径与「回款看板」台账下钻的「已收(元)/未收(元)」一致。
- **`/opportunities`（商机清单）新增「是否重大POC」列**：
  - 新增「**是否重大POC**」列：下拉取值 **是 / 否**，位于「预估落单时间」之后、「实际中标状态」之前，默认显示。可在该页超管编辑抽屉里选填；也可在 `input/opportunities.xlsx` 加「是否重大POC」列后重导入。**存量商机**无该值时显示 `-`，不影响其它字段。
  - **`/opportunities/key`（重点商机跟进）自动联动**：该页的「范围设置」可选字段与列选择器均派生自商机列定义，新列自动成为**可用筛选条件**与**可勾选显示列**（默认显示列不变，需要时在列选择器里勾选）。

> 本版本**仅前端展示口径与商机列定义调整 + 后端商机字段白名单加一列**（`opportunities.py` 增 `majorPoc` 字段）。**不改回款/分析计算口径，不新增运行依赖，不需点「更新数据」，不新增页面、不新增页面访问 key。**
> 全部细节见仓库 `PROGRESS.md` 版本史。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以下命令以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.2.1.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.2.1.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）
sudo cp /tmp/pmplatform-update-V2.2.1/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.2.1/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过；
#   仅当现网早于 V2.1.1、尚无 pmisdata/ 时才整套拷入：
#   sudo cp -rn /tmp/pmplatform-update-V2.2.1/pmisdata /opt/pmplatform/

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **无新增 Python 依赖**：本功能全为既有标准库 + pydantic/openpyxl。无需 pip 安装任何东西。
> **无需点「更新数据」**：本版本未改 `preprocess_data.py`，看板口径不变；商机/跟进数据独立持久化，结构不变。

## 5. 回滚

```bash
sudo systemctl stop pmplatform
# 还原代码与 dist
cd /opt/pmplatform && sudo tar xzf /backup/pmplatform-code-<时间>.tgz
sudo chown -R pmapp:pmapp /opt/pmplatform
sudo systemctl start pmplatform
```

> 回滚到 V2.2.0：代码 tar 还原即可。已写入「是否重大POC」字段的商机数据，旧前端忽略未知键，无副作用；回款明细恢复为万元显示。商机/跟进 JSON 结构未变，无需处理。
