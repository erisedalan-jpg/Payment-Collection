# 升级手册 —— V2.2.0（商机清单改造 + 重点商机跟进新页）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的旧版本升级到 **V2.2.0**。
> 本包是「**纯代码 + /pm 前端 dist**」的轻量增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.2.0.zip`。
> 现网若早于 V2.1.1，本包代码为**全量覆盖所有 .py + dist**（升到 V2.2.0 无遗漏），但被跳过版本的**首用/数据/依赖步骤**仍需照对应手册补做（如 V2.1.1 的下载依赖 `requests`/`openpyxl` 与 pmisdata、V2.1.0 的临时跟进、V2.0.0 的商机 seed）。

---

## 0. 本次升级包含什么

V2.2.0 是**整页级（Y 级）**版本。相对 V2.1.1，本轮两块：

- **商机清单改造（原 /opportunities）**：
  - 该页从「**重点跟进**」分区**移到「项目」分区**（位于「已关闭项目」之后、「项目动态」之前），并**改名「商机清单」**（页面访问 key 不变，仍 `opportunities-progress`）。
  - 新增「**商机级别**」列：下拉取值 **P1 / P2 / P3 / P4**，位于「预估金额(万元)」之后、「预估落单时间」之前，默认显示。可在该页超管编辑抽屉里选填；也可在 `input/opportunities.xlsx` 加「商机级别」列后重导入。**存量商机**无该值时显示 `-`，不影响其它字段。
- **新增「重点商机跟进」页（`/opportunities/key`，「重点跟进」分区，位于「重点项目进展」之后、「临时重点跟进」之前）**：形式同「临时重点跟进」——
  - **超管**点「范围设置」用两级 AND/OR 条件从商机各列筛出重点商机清单（**默认条件**：是否TOP1000客户=TOP1000 且 是否提前介入=是 且 是否重点商机=是 且 商机状态≠赢单）；
  - **普通管理员**在清单里点单元格填写「本周工作进展 / 后续工作计划」（自动记跟进时间与跟进人，派生「跟进日期 / 跟进人」两列）；
  - **超管**点「更新（归档+清空）」把当期清单冻结为历史快照并开新一期；可按数据集导出 xlsx。
  - 清单**取数自商机数据**（按账号 L4 自动裁剪——普通管理员只见本 L4 内、且命中范围的商机），商机字段只读，仅四个跟进列可编辑。

> 后端新增纯模块 `opportunity_followup.py`（范围规整 + 跟进编辑/归档）+ `server.py` 4 个端点（`GET /api/opportunity-followup` 任意登录；`POST .../scope`、`POST .../archive` 超管专属；`POST .../update` 任意登录）；跟进数据持久化在服务器 `data/opportunity_followup.json`（运行时自动生成）。前端复用并参数化了 ScopeBuilder/ProgressEditModal（临时跟进/重点项目进展页行为不变）。
> **本版本不改回款/分析口径，不新增运行依赖，不需点「更新数据」**。
> 全部细节见仓库 `PROGRESS.md` 版本史。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以下命令以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.2.0.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.2.0.zip').extractall('.')"

# (4) 覆盖后端代码（*.py，含新增 opportunity_followup.py）
sudo cp /tmp/pmplatform-update-V2.2.0/*.py /opt/pmplatform/
#   确认新模块到位：
ls -l /opt/pmplatform/opportunity_followup.py

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.2.0/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过；
#   仅当现网早于 V2.1.1、尚无 pmisdata/ 时才整套拷入：
#   sudo cp -rn /tmp/pmplatform-update-V2.2.0/pmisdata /opt/pmplatform/

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写——跟进/归档要写 data/opportunity_followup.json）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **无新增 Python 依赖**：本功能全为 Python 标准库 + 既有 pydantic/openpyxl。无需 pip 安装任何东西（V2.1.1 已装的 `requests`/`openpyxl` 保留即可，下载功能仍用）。
> **无需点「更新数据」**：本版本未改 `preprocess_data.py`，看板口径不变；新页跟进数据独立持久化，首次访问自动生成 `data/opportunity_followup.json` 并 seed 默认范围。

## 3. 升级后必做/按需操作

1. **账号不变（不新增超管）**：本版本**未改动后端 `auth.py`**。超管仍固定为 `admin`、`wangxutong`、`zhangyingzhe` 三个。
2. **新页面访问授权（按需）**：新页 pageKey 为 `opportunity-followup`。
   - **超管**自动可见「重点商机跟进」。
   - **普通管理员**：若其账号「可访问页面」是 `*`（全部），自动可见；若是**显式页面清单**，需超管在 `/admin` 账号管理里给该账号**勾选「重点商机跟进」**后才出现。
3. **商机级别数据（按需）**：「商机级别」为新列，存量商机默认空（显 `-`）。可在「商机清单」页超管逐条编辑选填 P1–P4；或在 `input/opportunities.xlsx` 增加「商机级别」列后，于「商机清单」页**导入**（整表替换，导入前自动备份）。
4. **重点商机跟进首用**：超管进 `/opportunities/key` → 默认已带「TOP1000 且 提前介入 且 重点商机 且 状态非赢单」范围，列表即按此筛出商机；如需调整点「范围设置」改条件保存。普通管理员进页后点跟进单元格填写即可。
5. **nginx/systemd 无变化**：本版本未改 systemd 单元与 nginx 配置。

- [ ] 浏览器开 `http://<IP>/pm` → 预制超管登录成功。
- [ ] 左侧菜单「**项目**」分区下出现「**商机清单**」（在「已关闭项目」与「项目动态」之间）；旧位置「重点跟进」分区下**不再有**「重点商机进展」。
- [ ] 「商机清单」页表格含「**商机级别**」列（在「预估金额(万元)」与「预估落单时间」之间，下拉 P1–P4），默认显示。
- [ ] 左侧菜单「**重点跟进**」分区下出现「**重点商机跟进**」（在「重点项目进展」与「临时重点跟进」之间）；点开为 `/pm/opportunities/key`，列表按默认范围（TOP1000 且 提前介入 且 重点商机 且 状态非赢单）筛出商机。
- [ ] 超管在「重点商机跟进」见「范围设置 / 更新（归档+清空）/ 导出」按钮；普通管理员**看不到**这三个，但可点跟进单元格填写「本周工作进展 / 后续工作计划」并保存（刷新后仍在，含跟进时间/人）。

## 5. 回滚

```bash
sudo systemctl stop pmplatform
# 还原代码与 dist
cd /opt/pmplatform && sudo tar xzf /backup/pmplatform-code-<时间>.tgz
sudo chown -R pmapp:pmapp /opt/pmplatform
sudo systemctl start pmplatform
```

> 回滚到 V2.1.1：代码 tar 还原即可。新生成的 `data/opportunity_followup.json` 在旧版本不被读取（无害残留，可留可删）；旧版无「商机级别」列，对已写入该字段的商机数据无副作用（旧前端忽略未知键）。

## 6. 备注

- 本次为**纯代码 + dist** 增量更新，无新增依赖、无 systemd/nginx 变更、无需「更新数据」。
- 跟进数据持久化在 `data/opportunity_followup.json`（运行时自动生成，结构 = `{version, scope, current, archives}`）；归档快照永久保留供历史查看。
- **已知技术债（PROGRESS Backlog）**：`handle_opportunity_followup_*` 的「忙检查→置位」与 temp_followup/pmis_download 同款非原子（TOCTOU），实际风险低（仅跟进/归档短操作）；后续做全互斥时用 `_history_lock` 一并收紧。
- 若你需要的是**全新安装整包**（非增量），在开发机执行 `python make_deploy_zip.py` 生成 `pmplatform-deploy-V2.2.0.zip`（含全部代码 + /pm dist + data/ + input/ + pmisdata/），按 `deploy/部署-Ubuntu24-端口80-pm.md` 全新部署。
- 端口 80 为明文 HTTP（仅可信内网）；不可信网络建议上 443 + TLS（见部署手册 §11）。
