# Ubuntu 部署指南 —— 项目管理平台

> 目标:部署到一台 Ubuntu 服务器,供多名管理人员通过内网 HTTPS 并发访问。
> 架构:**nginx(对内网终止 TLS + gzip + SSE 透传) → 反代 → app(127.0.0.1:8080) → systemd 托管**。
> 本目录配套文件:`pmplatform.service`(systemd 单元)、`nginx-pmplatform.conf`(nginx 配置)、
> 仓库根 `reset_super_password.py`(改种子超管口令)。完整审计见 `docs/2026-06-20-系统审计与Ubuntu部署建议.md`。

> ⚠️ **上线前置条件**:P0-1/P0-2 服务端授权加固已合入(非超管不能直读原始数据文件、写/运维端点限超管)。
> 但 **P0-4(改默认超管口令)必须在本指南第 5 步完成**,否则源码内置的弱口令(admin/wxtnb 等)即可登录超管。

---

## 1. 系统准备
```bash
sudo apt update && sudo apt install -y python3-venv nginx git
# 专用低权用户(不可登录 shell),时区固定东八区
sudo useradd -r -s /usr/sbin/nologin pmapp
sudo timedatectl set-timezone Asia/Shanghai
# 部署目录
sudo mkdir -p /opt/pmplatform
```

## 2. 放置代码 + 构建前端产物
后端运行需要前端构建产物 `frontend/dist`(server.py 由此提供页面)。**两种方式取一**:
- **A. 服务器上构建**(需装 Node 18+):
  ```bash
  sudo git clone <仓库地址> /opt/pmplatform     # 或 scp 上传代码
  cd /opt/pmplatform/frontend && npm ci && npm run build   # 产出 frontend/dist
  ```
- **B. 开发机构建后只传产物**:本地 `cd frontend && npm run build`,把整个仓库(含 `frontend/dist`)`scp`/`rsync` 到 `/opt/pmplatform`(服务器免装 Node)。

```bash
sudo chown -R pmapp:pmapp /opt/pmplatform        # 数据目录 data/ 须 pmapp 可写
```

## 3. Python 环境(依赖瘦身)
常驻 web 服务**只需 Python 3.8+ 标准库**;数据更新功能才需三方包;
**不使用 WPS 在线同步即可不装 playwright/chromium(最重依赖)**。
```bash
sudo -u pmapp python3 -m venv /opt/pmplatform/.venv
sudo -u pmapp /opt/pmplatform/.venv/bin/pip install pydantic openpyxl
# 仅当需要 WPS 在线同步再装:
#   sudo -u pmapp /opt/pmplatform/.venv/bin/pip install playwright
#   sudo -u pmapp /opt/pmplatform/.venv/bin/playwright install chromium
```

## 4. 准备数据
- 把 PMIS 9 表放 `input/pmis/`,各 CSV(`collection_stages.csv`/`payment_records.csv`/`budget_data.csv`/
  `profit_loss_*.csv`/`delivery_analysis.csv`)、`组织架构.xlsx`、`A.xlsx` 放 `input/`。
- 首次生成主数据(需 pydantic+openpyxl):
  ```bash
  sudo -u pmapp /opt/pmplatform/.venv/bin/python preprocess_data.py   # 产出 data/analysis_data.json
  ```
  (上线后管理员也可在页面"数据管理 → 更新数据"重新处理。)

## 5. ⚠️ 改掉默认超管口令(P0-4,必做)
源码 `auth.py:_SEED_SUPERS` 内置 3 个弱口令超管(admin/wxtnb、wangxutong/niubi、zhangyingzhe/venus600),
首次启动会自动建号。**上线前逐个改密**(超管口令无法经 /admin 界面改,故用助手脚本):
```bash
# 先触发一次种子建号(或等服务首启),再逐个改密(交互输入,不回显):
sudo -u pmapp /opt/pmplatform/.venv/bin/python reset_super_password.py admin
sudo -u pmapp /opt/pmplatform/.venv/bin/python reset_super_password.py wangxutong
sudo -u pmapp /opt/pmplatform/.venv/bin/python reset_super_password.py zhangyingzhe
```
- 不需要的种子超管,建议在 `auth.py:_SEED_SUPERS` 删除后再首启(避免无用账号)。
- 进阶(可选):把 `_SEED_SUPERS` 口令从源码移除、改为首启随机生成并打印一次,彻底消除源码明文。

## 6. systemd 托管
```bash
sudo cp /opt/pmplatform/deploy/pmplatform.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now pmplatform
sudo systemctl status pmplatform          # 应为 active (running)
```
- 停止/重启:`sudo systemctl stop|restart pmplatform`(**勿**用 `python server.py --stop`,Linux 失效)。
- 日志:`journalctl -u pmplatform -f` 或 `/opt/pmplatform/log/server.log`。

## 7. nginx + TLS
```bash
sudo mkdir -p /etc/ssl/pmplatform
# 自签证书(内网用;有内网 CA 则用 CA 签发):
sudo openssl req -x509 -newkey rsa:2048 -nodes -days 825 \
  -keyout /etc/ssl/pmplatform/server.key -out /etc/ssl/pmplatform/server.crt \
  -subj "/CN=pm.intranet.example"
sudo cp /opt/pmplatform/deploy/nginx-pmplatform.conf /etc/nginx/sites-available/pmplatform
# 改 conf 里的 server_name 为你的内网域名/IP
sudo ln -s /etc/nginx/sites-available/pmplatform /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
# 防火墙:仅放行内网网段访问 443(按实际网段改)
sudo ufw allow from 10.0.0.0/8 to any port 443
```
> TLS 上线后,建议给会话 cookie 补 `Secure` 标志(`auth.py:build_set_cookie`,审计 P0-3 余项)。

## 8. 数据备份
关键本地数据(账号/口令哈希、跟进、标签、主数据、历史快照)都在 `data/`:
```bash
# 每日打包到异机(示例 cron,02:00)
0 2 * * * tar czf /backup/pmplatform-$(date +\%F).tgz -C /opt/pmplatform data/
```
重点备份:`data/accounts.json`、`data/followup_records.json`、`data/project_tags.json`、
`data/pmis_links.json`、`data/analysis_data.json`、`data/history/`。

## 9. 上线验证清单
- [ ] `systemctl status pmplatform` = running;`curl -k https://<域名>/` 返回登录页
- [ ] 用改后的超管口令能登录;旧弱口令**已失效**
- [ ] 新建一个受限管理员(限定某 L4 + 页面),登录后:
  - 只看到其 L4 范围的数据;`curl -k https://<域名>/data/accounts.json -H Cookie:...` 返回 **403**
  - `https://<域名>/api/clear-data` 等运维端点返回 **403**
- [ ] "数据管理 → 更新数据"进度条实时刷新(SSE 透传 OK)
- [ ] 浏览器开发者工具 Network:`analysis_data.json` 响应头含 `Content-Encoding: gzip`
