# 升级手册 —— V2.9.0（首页总览下半区重做；从 V2.8.5 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.8.5** 升级到 **V2.9.0**。
> 本包是「**纯前端**」增量包——**只换 `frontend/dist`**，后端代码与本次一字未改。
> 随附最小更新包：`pmplatform-update-V2.9.0.zip`。
> 若现网早于 V2.8.5，请先用对应旧手册升到 V2.8.5，再用本包。

---

## 0. 本次升级包含什么（V2.8.5 → V2.9.0）

首页（项目总览 `/`）下半区整页重做，后端零改动：

- **下半区由两栏改三栏**：「需要处理的异常 | 待办 / 临期 | 项目动态」并列（窄屏 ≤1200px 自动堆叠为单列）。
- **新增「待办 / 临期」工作队列**：顶部 4 桶计数（回款临期 / 回款已延期 / 里程碑 / 成本超支，可点击筛选）＋ 7天 / 30天窗口切换 ＋ 下方按紧急度排序的条目清单（已延期 → 今到期 → 临期 → 里程碑 → 超支），每行点击进项目详情。
  - 口径：回款临期/已延期取收款阶段节点；里程碑取里程碑节点（临期/滞后）；成本超支只取「交付成本超支」或「总成本超支 >5000」的项目。均排除异常项目（服务组 L4 缺失）。
- **异常卡默认内联前 3 行**：不再需要逐张点「展开」，卡片直接显示前 3 个项目，超出显示「查看全部 N 个 →」。
- **项目动态加强**：顶部加「本期变化」数字条（阶段推进 / 新增延期 / 回款新增 / 风险净增，取自上次同步对比，快照不足则不显示）；下方事件默认「只看要紧」（延期/超支/风险等），可切「全部」并按 L4 组织快筛。

> **【头号注意 1】✅ 纯前端，无需重启后端、无需点「更新数据」。** 本包只更新 `frontend/dist`，后端 `*.py` 与 V2.8.5 完全一致（包内附带的 .py 覆盖上去是同内容、无副作用）。
>
> **【头号注意 2】无新增页面 / 路由 / pageKey、无需授权、无新增依赖。** 从在线基线 V2.8.5 增量。改的是首页展示与新增前端计算，底层数据与口径不变。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.9.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.9.0.zip

# 1) 备份现网 dist(可选)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖前端 dist(核心)
cd /tmp/pmplatform-update-V2.9.0
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
#   (可选)后端 .py 与 V2.8.5 一致,覆盖与否均可:
#   sudo cp -f *.py /opt/pmplatform/

# 3) 无需重启后端(后端未变)、无需点「更新数据」。
#    浏览器强制刷新(Ctrl+F5),确认右上角版本号 V2.9.0。
```

> 注：若你的部署用 nginx 缓存了静态资源，强刷仍是旧版时，`sudo systemctl reload nginx` 或清 nginx 缓存后再强刷。

## 4. 回滚

纯前端，回滚极简：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
```
（无后端/数据改动，回滚只换回旧 dist、强刷即可。）
