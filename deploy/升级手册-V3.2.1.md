# 升级手册 —— V3.2.1（倚天明细表全列筛选/分页 + 五页图表下钻；从 V3.2.0 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V3.2.0** 升级到 **V3.2.1**。
> 随附最小更新包：`pmplatform-update-V3.2.1.zip`。
> 若现网早于 V3.2.0，请先用对应旧手册升到 V3.2.0，再用本包。

---

## 0. 本次升级包含什么（V3.2.0 → V3.2.1）

**纯前端交互增强，后端 / 数据管线 / 业务口径零改动。** 都集中在倚天工时域：

- **① 两张明细表：加分页 + 全列筛选，并修好「只显 3 行」**
  - `/yitian/compliance`「问题明细」表、`/yitian/analytics`「员工工时明细」表——V3.2.0 里它们因排在图表下方、冻结表头高度被压到很小，**只能看到约 3 行**。本版改为**固定较大高度（约 15 行可见）+ 分页**（每页 50 条、表下有页码 + 「共 N 条」，和 /projects 一致，表头仍冻结）。
  - **每一列都加了列头筛选**（点列名旁的 ▼，多选筛选）：合规明细除「工作成果摘要」外每列可筛，「问题」列按**问题类型**多选；员工明细 8 列均可筛。合规页顶部原来那个「全部问题类型」下拉已并入「问题」列头筛选。

- **② 倚天 5 页：图表 / 卡片 / 表格点击下钻**
  - 在 `/yitian`、`/yitian/compliance`、`/yitian/analytics`、`/yitian/trend`、`/yitian/customer` 上，点击柱图 / 环图 / 卡片 / 表格行，会**跳到对应明细表并自动筛好**：
    - 点某 L4 组织的柱 / 行 → 跳到员工明细并筛到该组；
    - 合规页点问题分布柱 / 热力图格 → 问题明细筛到该问题类型（+ 该组）；
    - 分析页点某员工的柱 / 散点 → 员工明细筛到该人并滚动到表；
    - 总览 KPI 卡：合规率 → 合规页；未填 / 加班 / 总工时 / 饱和度 → 分析页；
    - 趋势页点某周 / 月 / 季的折线点 → 跳到对应页并把日期区间设为该时段。
  - 图表本身保持总览视角不变，只是把你点的那一块"筛好"送进明细表。

> **【头号注意 1】纯前端升级——只需换 dist，无需重启后端、无需点「更新数据」、无新增页面 / 权限。**
> 本包为保持结构一致仍包含后端 `*.py`，但它们**与 V3.2.0 逐字节相同**（本次后端零改动）。只替换 `frontend/dist` 即可；覆盖 `*.py` 也无害。
>
> **【头号注意 2】务必强制刷新（Ctrl+F5）** 清掉旧 JS/CSS 缓存，否则新列头筛选 / 分页 / 下钻可能加载不到。
>
> **【头号注意 3】时间维度下钻会改日期区间。** 在趋势页点某时段的折线点跳到合规 / 分析页时，会把左上角的日期区间设为该时段（与手动改日期区间同效）——这是刻意的，想看回全时段手动改回即可。

## 2. 升级步骤（Ubuntu 服务器，约 2 分钟）

```bash
# 0) 上传 pmplatform-update-V3.2.1.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V3.2.1.zip
cd /tmp/pmplatform-update-V3.2.1

# 1) 备份现网 dist(可选但建议)
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS

# 2) 覆盖前端 dist(本次升级的核心)
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) (可选)覆盖后端 *.py —— 与 V3.2.0 相同,做不做都行
# sudo cp -f *.py /opt/pmplatform/

# 4) 无需重启后端(纯前端);若走了反向代理静态缓存,清一下缓存

# 5) 浏览器强制刷新(Ctrl+F5),确认右上角版本号 V3.2.1
```

**本次不需要在页面上做任何配置**（无新页面、无新 pageKey、无需授权、无需点「更新数据」）。

## 4. 回滚

纯前端，回滚只需还原 dist（无需动后端、无需重启）：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
# 浏览器 Ctrl+F5
```
> 业务数据、账号、倚天工时、各类配置均不受影响 —— 本次零后端 / 零数据改动。
