## 0. 本次升级包含什么（V2.5.7 → V2.5.8，性能优化·纯前端）

**背景**：此前跨页点击（切换页面）时会有短暂的「卡一下 / 页面迟迟不出来」的感觉，以 **成本分析（/insight/costdetail）、里程碑管理（/insight/milestone）、风险跟进（/risk）** 三页最明显。

**根因（经 CPU 性能剖析定位）**：点击那一下，浏览器要**一次性同步渲染整页的重型内容**——大表格（如风险跟进一次铺 400+ 行、几十列，无虚拟化）、多个图表（里程碑一屏 6 个图挂载即绘制）、以及约 16MB 业务数据的响应式代理开销。这些都堆在「点击到出页面」的那一帧里，于是卡顿。

**本次三处优化（不改变任何数据与展示口径，只改渲染时机）**：
1. **数据层轻量化**：把 16MB 只读业务数据改为浅层响应式，去掉全站派生页面遍历数据时的额外开销（全站受益，无感）。
2. **图表按需渲染**：图表改为**滚动进入视口时才绘制**（提前 200px 预热），多图页面（里程碑、商机看板等）挂载不再一次性初始化全部图表。
3. **重内容延迟渲染**：三页点击进入时**先立即显示页面骨架（标题/指标卡/工具栏）**，随后一两帧再挂载大表格与图表，把重渲染移出「点击那一帧」——**点击瞬间就能看到页面**，内容紧接着填充。

**效果**：三页点击后页面骨架在约 **15~30 毫秒**内出现（此前需等 0.6~5 秒才出内容），点击即时响应；内容随后完整呈现，数据与此前完全一致。

> **【行为变化·需知悉】** 图表现在改为「滚动到该图时才绘制」的按需加载（类似图片懒加载）。正常浏览滚动时图表会平滑出现（提前 200px 预渲染），不影响查看；仅在极快速拖动到很靠下的图表时，可能看到图表晚半秒出现，属正常。

> **【头号注意】本次升级无需点「更新数据」。** 全部为**纯前端**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何数据来源。前端复用现有 `data/analysis_data.json` 即可。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面/无新页面访问 key、未改 `auth.py`（超管不变）。**
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.5.8.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.5.8.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本包与 V2.5.7 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.5.8/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本包核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.5.8/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本包无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧样式：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。
