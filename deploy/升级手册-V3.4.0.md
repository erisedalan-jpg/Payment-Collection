# 升级手册 —— V3.4.0（阶段验收款里程碑 + 产品超支标签；从 V3.2.2 累积升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的**在线 V3.2.2** 升级到 **V3.4.0**。
> 随附最小更新包：`pmplatform-update-V3.4.0.zip`。
> 本包是**完整代码快照**：装到 V3.2.2 服务器会**累积带上 V3.2.3 + V3.3.0 + V3.4.0 三版改动**（下方第 0 节逐版说明）。若现网早于 V3.2.2，请先用旧手册升到 V3.2.2 再用本包。

## 0. 本次升级包含什么（在线 V3.2.2 → V3.4.0，累积三版）

### V3.4.0（本次主角）—— 阶段验收款里程碑 + 产品超支标签（**非纯前端，须点「更新数据」**）
- **① 阶段验收款里程碑**：PMIS 里程碑两表里本就存在、但此前从未解析的**「阶段计划/实际完成时间」**（每项目一串「阶段验收款N（PP%）：日期」）接入系统：
  - `/project/:id`「进度里程碑」明细表**整表按计划时间排序**，新的**阶段验收款节点整行高亮**（淡强调底）区分。
  - 阶段验收款节点自动进入 `/insight/milestone`（到期提醒 / 关键节点分布 / 延期清单）与 `/projects/temp` 的里程碑范围条件。
- **② 产品超支标签**：项目「预算核算」里「产品、商品成本」科目**剩余（元）为负**的项目，自动打上**「产品超支」**标签（同「佳杰」那种规则标签：自动、可在「按标签排除」和各页标签筛选里用）。
- **③ 首页待办卡（仅核查，无改动）**：确认首页「待办/临期」卡已受「按标签排除」影响（被排除项目不出现），无需调整。

> **这一步需要点一次「更新数据」才生效**：阶段里程碑与「产品超支」标签都是在「更新数据」（重跑数据管线）时算出并写入的。**不点的话**：`/project/:id` 里看不到阶段验收款节点、里程碑报表不含阶段项、产品超支项目也不会挂标签。数据源无需重新上传，用现有 `input/` 里的 PMIS 数据重跑即可。

### V3.3.0（随包一并带上）—— 倚天工时合规规则前端可配置（**改完立即生效，本域无需点「更新数据」**）
- `/data`「数据管理」页倚天工时域新增**超管专属**「合规规则配置」卡：把「问题工时」的判定规则（必填三段词 / 服务方式生效日 / 类型一致性 / 产品类别关键词 / 客户提示词等三组）改为可视化配置，支持每检查项启用/停用、JSON+Excel 导入导出、恢复默认。**保存即后端重算、立即生效**（本域不走「更新数据」）。
- 无新增页面 / 路由 / pageKey / 权限。

### V3.2.3（随包一并带上）—— /risk 选列与排序 F5 刷新持久化修复（纯前端）
- 修复「风险跟进」页调整选列/排序后按 F5 刷新会回到基础状态的问题。

**全程无新增第三方依赖。业务口径（回款达成率、成本、概算等）零改动。**

## 2. 升级步骤（Ubuntu 服务器，约 3–5 分钟）

```bash
# 0) 上传 pmplatform-update-V3.4.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V3.4.0.zip
cd /tmp/pmplatform-update-V3.4.0

# 1) 备份现网 dist 与后端(建议)
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS
sudo mkdir -p /opt/pmplatform/_pybak-$TS && sudo cp /opt/pmplatform/*.py /opt/pmplatform/_pybak-$TS/

# 2) 覆盖前端 dist
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) 覆盖后端 *.py(整批覆盖;含新增 yitian_rules_config.py)
sudo cp -f *.py /opt/pmplatform/

# 4) 重启后端(systemd 托管,服务名以现网为准)
sudo systemctl restart pmplatform    # 或你的实际服务名

# 5) 登录平台 → /data「数据管理」→ 点「更新数据」,等 SSE 进度跑完
#    (把阶段验收款里程碑、产品超支标签写进数据;用现有 input/ 数据重跑即可,无需重新上传)

# 6) 浏览器强制刷新(Ctrl+F5),确认右上角版本号 V3.4.0
```

> 若走了反向代理静态缓存，第 2 步后清一下缓存。

## 4. 回滚

还原 dist + 后端 *.py + 重启（数据无需回滚——本次只新增里程碑节点 / 标签 / 可配置项，不改既有口径）：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
sudo cp -f /opt/pmplatform/_pybak-<时间戳>/*.py /opt/pmplatform/
sudo systemctl restart pmplatform
# 浏览器 Ctrl+F5
```
> 业务数据、账号、倚天工时、各类配置均不受影响。注意：回滚后端会删掉 `yitian_rules_config.py` 的引用来源（server.py 回旧版），倚天「合规规则配置」卡随之下线，属预期。
