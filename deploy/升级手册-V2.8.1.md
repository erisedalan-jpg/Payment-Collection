# 升级手册 —— V2.8.1（审计「目标/详情」全量富化；从 V2.8.0 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.8.0** 升级到 **V2.8.1**。
> 本包是「**后端 + 前端**」增量包——更新 `server.py`、`audit.py` 等后端 `.py` 与 `frontend/dist`；**须重启后端**。
> 随附最小更新包：`pmplatform-update-V2.8.1.zip`。
> 若现网早于 V2.8.0，请先用对应旧手册升到 V2.8.0，再用本包。

---

## 0. 本次升级包含什么（V2.8.0 → V2.8.1）

**给「账号管理 → 审计」页的「目标」「详情」两列补全内容，覆盖全站约 40 个高价值写操作。** 此前这两列只有账号管理（创建/修改/删除账号）与登录失败有值，其余操作（跟进、进展、标签、商机、更新数据、上传、回滚等）两列皆空；本次全部补齐。

- **业务跟进**：跟进记录增删改（目标=项目编号·名称/记录编号；详情=跟进类型·状态，或枚举字段旧→新对比）、项目进展与四类跟进（临时/风险/商机/回款重点）的范围设置·更新·归档·删归档、标签保存。
- **商机**：新建/更新/删除/导入（目标=商机名称，更新详情=改动字段旧→新对比）。
- **数据运维**：更新数据/清空/停止/PMIS 拉取/上传/更新 Cookie/数据回滚·撤销/人工导入·回滚（目标=文件名/版本号等，详情=动作说明）。

> **【隐私红线】** 详情**绝不记录**密码、cookie 值/预览、完整请求体；长文本业务正文（跟进内容、进展/回顾文字）只标注「（已填写/已改）」不落正文，需原文用记录编号回业务页查。
>
> **【行为要点】** 更新数据/PMIS 拉取这类异步动作，只在**成功启动**时记「触发」；若因「其他操作进行中」被拒，则不记「触发」（该次仍留有账号/时间/结果，仅详情为空）。

> **【头号注意 1】⚠ 须重启后端。** 本包改了 `server.py`、`audit.py`，覆盖后必须 `systemctl restart` 才生效。
>
> **【头号注意 2】✅ 不需点「更新数据」。** 审计独立于 `data/analysis_data.json`，本次不动数据管线；无需重新处理数据。
>
> **【头号注意 3】✅ 无需改 nginx。** 真实来访 IP 走已配置的 `X-Forwarded-For`，本次无新增端点/页面/pageKey、无需授权、无新增依赖。
>
> **【头号注意 4】前端零功能改动。** 审计页的列、筛选、导出结构不变；只是「目标/详情」两列从此有值。dist 一并更新只为版本号显示 V2.8.1。
>
> **【头号注意 5】仅新增记录被富化，历史审计不回填。** 升级前已产生的空「目标/详情」记录保持原样（无法重建旧请求上下文）；升级后的新操作才带内容。

## 1. 最小更新包内容（`pmplatform-update-V2.8.1.zip`）

解压出顶层目录 `pmplatform-update-V2.8.1/`，含：
- 全部后端根 `*.py`（**本期实质变化：`server.py`、`audit.py`**；其余同 V2.8.0，覆盖无副作用）。

- **不含** `client/`、`data/`、`input/`、`tests/`、`docs/`、`node_modules/`。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.8.1.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.8.1.zip
cd /tmp/pmplatform-update-V2.8.1

# 1) 备份现网后端与 dist(可选)
sudo cp /opt/pmplatform/server.py /opt/pmplatform/server.py.bak-$(date +%Y%m%d-%H%M)
sudo cp /opt/pmplatform/audit.py  /opt/pmplatform/audit.py.bak-$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py(核心) 与前端 dist
sudo cp -f *.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) 重启后端(必须)
sudo systemctl restart pmplatform      # 服务名以现网为准
sudo systemctl status  pmplatform --no-pager | head -5

# 4) 无需点「更新数据」。浏览器强制刷新(Ctrl+F5)访问 /data,确认版本号 V2.8.1。
```

> 注：服务名若不同，用 `systemctl list-units | grep -i pmplatform` 查实际单元名。若 nginx 缓存静态资源，强刷仍旧版时 `sudo systemctl reload nginx` 或清缓存后再强刷。

## 3. 升级后验证清单

- [ ] 右上角/关于处版本号显示 **V2.8.1**。
- [ ] 用超管登录，进「账号管理 → 审计日志」页。
- [ ] 做几个操作各一次：改一条跟进记录、保存标签、更新/新建一个商机、点一次「更新数据」。
- [ ] 回审计页刷新，确认这些新记录的**「目标」「详情」两列有值**（如跟进记录目标=项目编号·名称、详情=类型/状态；更新数据详情=触发数据重新处理）。
- [ ] 隐私核对：做一次「更新本机 PMIS Cookie」，审计详情只显「更新 PMIS Cookie」，**不含 cookie 内容**。
- [ ] 审计页筛选/导出功能照常可用。

---

## 4. 回滚

```bash
# 后端 + 前端一起回滚
sudo cp -f /opt/pmplatform/server.py.bak-<时间戳> /opt/pmplatform/server.py
sudo cp -f /opt/pmplatform/audit.py.bak-<时间戳>  /opt/pmplatform/audit.py
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
sudo systemctl restart pmplatform
```
（本次无数据/schema 改动，回滚只换回旧 `.py` + 旧 dist、重启即可；已产生的审计记录不受影响。）
