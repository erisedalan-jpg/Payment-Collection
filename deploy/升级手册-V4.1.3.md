# 升级手册 —— V4.1.3（工时明细新增「工作成果」全文列；**从 V4.1.2 单版升**）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V4.1.2** 升到 **V4.1.3**。
> 本包是「**非纯前端**」增量包——**换 dist + 覆盖后端 + 重启 app + 点一次「更新数据」**。
> 随附最小更新包：`pmplatform-update-V4.1.3.zip`。
> 若现网不是 V4.1.2，请先用对应旧手册升到 V4.1.2，再用本包。

---

## 0. 本次升级包含什么（V4.1.2 → V4.1.3）

| 版本 | 主要内容 | 动数据结构? | 需更新数据? |
|---|---|---|---|
| V4.1.3 | `/yitian/detail` 工时明细新增「**工作成果**」全文列（整列展示，默认可见、可导出） | 是（`yitian_data.json` 增 `ct` 字段） | **是** |

具体：此前工作成果正文**不下发前端**（只有"问题"行带 120 字摘要作悬浮提示）。V4.1.3 起把**工作成果全文**作为明细表的一列正常展示（默认可见、`wrap` 换行、随「选列」和导出）。

> ### 这一跳有两件事绝对不能漏：
>
> **【头号注意 1】⚠ 升级后必须点一次「更新数据」。**
> 工作成果全文（`yitian_data.json` 的新 `ct` 字段）是**数据管线产出**的。不重跑管线，明细页的「工作成果」列会**全部为空**（旧 `yitian_data.json` 没有该字段）。升级后进「数据管理 → 更新数据」，等 SSE 跑完即可。
>
> **【头号注意 2】⚠ 本版必须覆盖后端 + 重启 app。** 改动了 `yitian.py`（下发 `ct`）与 `schema.py`（数据契约加 `ct`）。只换 dist、不覆盖后端并重启，更新数据时会因 schema 不匹配而无法产出新字段。
>
> **【知情提示】隐私边界变化。** 工作成果全文此前刻意不下发前端（当作隐私）。本版起**下发全文**：凡能访问 `/yitian/detail` 的账号，在其 L4 组织范围内都能看到每条工时的工作成果全文，且随导出 xlsx。这是经确认的开放（管理者审阅工时质量所需）。**员工电话/所在省市/入职省市/岗位等隐私列仍一律不读、不下发**——那条隐私没变。

## 1. 最小更新包内容（`pmplatform-update-V4.1.3.zip`）

解压出顶层目录 `pmplatform-update-V4.1.3/`，含：

- 全部后端根 `*.py`（含本版改动的 `yitian.py` / `schema.py`）
- `frontend/dist/`（**已用 `--base=/pm/` 构建**）
- `requirements.txt`、`deploy/升级手册-V4.1.3.md`、`pmisdata/` 脚本
- **不含** `client/`、`data/`、`input/`、`tests/`、`docs/`、`node_modules/`

---

## 2. 升级步骤（Ubuntu 服务器，约 4 分钟）

```bash
# 0) 上传 pmplatform-update-V4.1.3.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V4.1.3.zip

# 1) 备份 dist + 后端(回滚用)
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS
sudo mkdir -p /opt/pmplatform/bak-$TS && sudo cp /opt/pmplatform/*.py /opt/pmplatform/bak-$TS/

# 2) 覆盖后端 .py
cd /tmp/pmplatform-update-V4.1.3
sudo cp -f *.py /opt/pmplatform/

# 3) 覆盖前端 dist
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 4) 重启后端(必须)
sudo systemctl restart pmplatform      # 若服务名不同,按现网实际名称
sudo systemctl status pmplatform --no-pager | head -5

# 5) 浏览器强刷(Ctrl+F5),确认左上角版本号 V4.1.3
# 6) ★ 进「数据管理 → 更新数据」,等 SSE 跑完 —— 这一步不能省(见头号注意 1)
```

> 注：若 nginx 缓存了静态资源，强刷仍是旧版时 `sudo systemctl reload nginx` 后再刷。

## 3. 升级后验证清单

- [ ] `/yitian/detail` 工时明细表**默认可见**「工作成果」列，显示**工作成果全文**（换行展示，非 120 字摘要）
- [ ] 「选列」里可增减「工作成果」列；勾选后**导出的 xlsx 含工作成果全文**
- [ ] 表内其它列、筛选、下钻、合规状态/问题原因均正常

## 4. 已知的行为变化（不是缺陷）

1. **工作成果全文下发前端**（V4.1.3）：`yitian_data.json` 体积随之增大（约 251KB → ~1MB 量级，含全文），前端加载略慢，属正常。
2. **可见范围**：工作成果全文对能访问 `/yitian/detail` 的账号在其 L4 范围内可见、随导出；隐私列（电话/省市/岗位）仍不下发。
3. **问题原因列的悬浮摘要保留不变**（仍是问题行的 120 字摘要），与新增的整列全文并存。

---

## 5. 回滚（回到 V4.1.2）

```bash
TS=<你升级时用的时间戳>
# 1) 还原后端与 dist
sudo cp -f /opt/pmplatform/bak-$TS/*.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-$TS /opt/pmplatform/frontend/dist
sudo systemctl restart pmplatform
```

> **回滚安全性**：`yitian_data.json` 里多出的 `ct` 字段（若已点过更新数据）对 V4.1.2 是无害的多余字段（旧版不消费）；回滚后**无需再点更新数据**。`input/` 与其它 `data/` 不受影响。
