# 升级手册 —— V2.6.10（成本预算口径兜底 + 分歧告警；从 V2.6.9 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.6.9** 升级到 **V2.6.10**。
> 本包是「**后端代码 + 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.6.10.zip`。
> 若现网早于 V2.6.9，请先用对应旧手册（累积包 V2.6.9）升到 V2.6.9，再用本包。

## 0. 本次升级包含什么（V2.6.9 → V2.6.10）

巡检发现 **7 个项目**在 PMIS《项目状态信息》的「项目总预算」与损益表 `profit_loss` 的「预算成本」两个源口径不一致（例如项目 `QAGD-SS-202603249009`：PMIS 总预算 5640，损益预算成本 7728.5）。**根因在 PMIS 源数据导出**（两个子系统对同一项目给了不同的预算数），非本平台计算缺陷；因 PMIS 修改周期长，本次先在平台侧做兜底：

- **成本口径兜底**：当某项目 PMIS 总预算 与 损益预算成本 分歧（两值皆存在、损益预算成本>0、且差额>1 元）时，`/project/:id` 预算核算页 PMIS 汇总块的**分母改用损益「预算成本」**，并连带重算 剩余预算/消耗比/项目超支（**只动分母，已发生的「核算」不变**；PMIS「成本状态」文本、交付超支 不动）。一致的项目（631/638）与只有单边数据的项目**原样不变**。
- **实测影响**：仅这 7 个项目数值变化；**超支项目数 36 → 35**（`WSGF-SS-202603259038` 由「超支 106.8%」改为「不超支 99.4%」，其余 6 个不改变超支侧、仅消耗比降几个点）。首页/`/projects` 的「成本超支」卡走另一套损益域口径、原本就一致，改完两套口径更靠拢。
- **新增数据质量告警**：数据治理（数据质量）页新增「预算口径分歧：总预算≠损益预算成本」告警组，列出这 7 个分歧项目（项目编号 / PMIS总预算 / 损益预算成本 / 差额），可导出，供你带去 PMIS 侧核对源数据。

> **【头号注意 1】⚠️ 本次升级必须点「更新数据」。** 改动在数据预处理管线（`preprocess_data.py` + `pmis.py`），需重跑生成新的 `data/analysis_data.json` 才生效。（未改数据契约结构 schema：`dataQuality.budgetSourceMismatch` 为新增可选键、向后兼容旧 JSON；现有 data/ 与账号原样保留。）
>
> **【头号注意 2】建议顺带重启后端进程（systemd）。** 标准 `.venv` python 部署下「更新数据」以子进程方式重跑新 `preprocess_data.py`、会读到新代码；但为稳妥（尤其打包 exe 版走进程内执行），建议覆盖文件后先重启一次再点「更新数据」。
>
> **【头号注意 3】无新增/删除页面、无新增 pageKey、无需授权账号、无新增依赖。**

## 2. 升级步骤（Ubuntu 服务器，约 2 分钟）

```bash
# 0) 上传 pmplatform-update-V2.6.10.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.6.10.zip

# 1) 备份现网(至少备份代码与 dist,data/input 不动本包也不覆盖)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py + 前端 dist(不覆盖 data/ 与 input/)
cd /tmp/pmplatform-update-V2.6.10
sudo cp -f *.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f pmisdata/*.py pmisdata/*.sh /opt/pmplatform/pmisdata/ 2>/dev/null || true

# 3) 重启后端(建议)
sudo systemctl restart pmplatform    # 服务名以现网为准

# 4) 浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号为 V2.6.10

# 5) 【必做】在页面点「更新数据」,等 SSE 进度跑完(重算 analysis_data.json,口径兜底才生效)
```

- [ ] 打开 `/project/WSGF-SS-202603259038`「预算核算」：项目超支显示「否」（消耗比约 99.4%）。
- [ ] 数据治理（数据质量）页出现「预算口径分歧」告警，条数 **7**，可查看/导出清单。
- [ ] 其它项目的成本核算数值无变化（631/638 一致项目不受影响）。

---

## 4. 回滚

若升级后异常：
```bash
sudo systemctl stop pmplatform
sudo rm -rf /opt/pmplatform
sudo mv /opt/pmplatform.bak-<时间戳> /opt/pmplatform
sudo systemctl start pmplatform
```
（回滚后如需恢复旧口径数据，再点一次「更新数据」用旧 `preprocess_data.py` 重算即可。）
