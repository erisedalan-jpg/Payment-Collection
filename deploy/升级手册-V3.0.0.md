# 升级手册 —— V3.0.0（倚天工时域 /yitian；从 V2.10.0 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.10.0** 升级到 **V3.0.0**。
> 随附最小更新包：`pmplatform-update-V3.0.0.zip`。
> 若现网早于 V2.10.0，请先用对应旧手册升到 V2.10.0，再用本包。

---

## 0. 本次升级包含什么（V2.10.0 → V3.0.0）

新增**倚天工时域**：把原本独立的「工时填写合规性检查工具」（`timesheet_checker_v3.py`，写死 2026 年日历、产出单体 HTML 报告）**重构**进本平台，成为 `/yitian` 下的 5 个页面。

- **新增 5 个页面**（侧栏新分区「倚天工时」）：
  - `/yitian` 倚天工时总览 —— KPI（总工时 / 平均饱和度 / 未填人数 / 加班人数 / 合规率）+ 工时类型占比（环图 + 柱状图）+ L4 组织分层汇总（含固定合计行）
  - `/yitian/compliance` 工时合规检查 —— 问题码分布 + 问题明细表（按问题类型筛选、列排序、导出 xlsx）
  - `/yitian/analytics` 工时统计分析 —— 饱和度 TOP10 + 未按时填写清单 + **完全未填清单**（原工具看不到的人）+ 员工工时明细
  - `/yitian/trend` 工时趋势分析 —— 7 张趋势图，X 轴可在「计算周（周五~周四）」与「自然周（周一~周日）」间切换
  - `/yitian/customer` 客户支持分析 —— TOP1000 大客户支持（按 L4，含合计行）+ 跨 BG 支持
- **数据累加**（本期核心）：倚天每周导出的工时表**只含当周**。系统新增**累积库**，按「工时ID」把历次导入**累加**成长期数据集 —— 新行追加、已存在的行**覆盖更新**（员工事后补填/修正后重导一遍即可修正历史；重复导入同一份文件**不会变双份**）。
- **年度无关**：不再有写死的年份。工作日由 `input/yitian/holidays.csv`（可选）决定，跨年只需补几行、不用改代码。
- **合规检查范围可配**：`/data` →「倚天工时 · 合规检查范围」（**仅超管**）可勾选哪些工时类型不计入合规率（默认剔除管理类/业务类/假期类，与原工具口径一致），**改完立即生效、无需点「更新数据」**。
- **累积数据管理**：`/data` →「倚天工时 · 累积数据管理」（**仅超管**）显示已累积多少行、覆盖哪段日期，并可**按日期区间删除**或**一键清空**。
- **权限**：工时是员工级敏感数据，按 `allowedL4` **服务端切数据** —— 普通管理员只拿得到本 L4 的员工与问题明细（不是前端隐藏，是后端根本不发）。

> **【头号注意 1】⚠ 非纯前端，必须覆盖后端 `*.py` 并重启后端。** 本包新增 6 个后端模块（`yitian.py` / `yitian_calendar.py` / `yitian_check.py` / `yitian_rules.py` / `yitian_settings.py` / `yitian_store.py`），并改动 `server.py` / `config.py` / `projects.py` / `schema.py` / `preprocess_data.py` / `data_scope.py` / `audit.py`。只换 dist 不重启后端，倚天接口会 404。
>
> **【头号注意 2】⚠ 必须点一次「更新数据」。** 倚天数据要经数据管线生成（读 `input/yitian/工时.xlsx` → 累加进累积库 → 产出 `data/yitian_data.json`）。不点则 `/yitian` 提示"数据不存在"。
>
> **【头号注意 3】⚠ 必须给账号授权 5 个新页面。** 新增 pageKey：`yitian` / `yitian-compliance` / `yitian-analytics` / `yitian-trend` / `yitian-customer`。**超管自动全见**；普通管理员需在「账号管理」里勾选。**一个都不授权的账号看不到侧栏「倚天工时」分区**（这是预期）。
>
> **【头号注意 4】首次使用需上传工时表。** 超管在 `/data` →「上传文件」→「倚天工时域（input/yitian/）」上传 `工时.xlsx`（倚天导出原样，不要改列名）。`holidays.csv` 可选（页面有「下载 holidays.csv 模板」按钮）；不提供时工作日按纯周一~周五近似，含法定节假日的周期**饱和度会偏低、未填名单会误报**，页面顶部会持续显示黄色告警条。

## 1. 最小更新包内容（`pmplatform-update-V3.0.0.zip`）

解压出顶层目录 `pmplatform-update-V3.0.0/`，含：
- `frontend/dist/`（**已用 `--base=/pm/` 构建**）。

- **不含** `client/`、`data/`、`input/`、`tests/`、`docs/`、`node_modules/`（升级不覆盖服务器现有数据 / 账号 / 输入）。

**无新增第三方依赖**（openpyxl / pydantic 早已在用；**未引入 pandas**）。

## 2. 升级步骤（Ubuntu 服务器，约 3 分钟）

```bash
# 0) 上传 pmplatform-update-V3.0.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V3.0.0.zip
cd /tmp/pmplatform-update-V3.0.0

# 1) 备份现网 dist 与后端 .py(可选但强烈建议 —— 本次后端改动较大)
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$TS

# 2) 覆盖前端 dist
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) 覆盖后端 *.py(本次必须 —— 含 6 个新模块)
sudo cp -f *.py /opt/pmplatform/

# 4) 重启后端(本次必须)
sudo systemctl restart pmplatform      # 服务名以实际为准

# 5) 浏览器强制刷新(Ctrl+F5),确认右上角版本号 V3.0.0
```

**然后在页面上做三件事（缺一不可）：**

1. **上传工时表**：超管登录 → `/data` → 右侧「上传文件」→ 最下方「倚天工时域（input/yitian/）」→ 选择 `工时.xlsx` → 点「上传倚天文件」。
   （可选：点「下载 holidays.csv 模板」，按格式填好当年节假日后一并上传。）
2. **点「更新数据」**：等进度跑完。日志末尾应出现类似：
   ```
   [OK] 倚天工时导入: 新增 540 行 / 更新 0 行 / 累积库共 540 行
   [OK] 倚天工时域: 540 行 / 85 人 / 2026-04-17 ~ 2026-04-23 / 日历源 fallback → .../data/yitian_data.json
   ```
3. **授权页面**：超管 →「账号管理」→ 给需要看倚天的普通管理员勾选 5 个新页面。

> 注：`data/yitian_store.json`（累积库）、`data/yitian_data.json`（下发数据）、`data/yitian_settings.json`（合规范围配置）均由后端**首次读写时自动创建**，无需手工建。它们在 `/opt/pmplatform/data/` 下，随现有 data 目录权限即可（app 进程可写）。

## 3. 日常使用：每周导一次工时

倚天导出的工时表**只含当周**。以后每周：

1. `/data` 上传新的 `工时.xlsx`（**直接覆盖上传即可**，同名文件会被替换）
2. 点「更新数据」

系统会把这一周的行**累加**进累积库（日志会显示「新增 N 行 / 更新 M 行 / 累积库共 X 行」），趋势图与统计的时间跨度随之自动扩大。

- **重复导入同一份文件**：只会「更新」不会「新增」，**不会变双份**。
- **员工事后补填/修正了工作成果**：把修正后的那一周重导一遍，历史记录会被覆盖更新。
- **导错了文件**：`/data` →「倚天工时 · 累积数据管理」可**按日期区间删除**或**一键清空**（两者都有二次确认）。
  > ⚠ **注意**：删除的历史周**无法从当前 `工时.xlsx` 恢复**（它只含最近一次导出的那一周）。误删需要重新拿到那一周的导出文件。

## 5. 回滚

回滚需还原 dist **与**后端 `*.py`，并重启：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
sudo cp -f /opt/pmplatform.bak-<时间戳>/*.py /opt/pmplatform/
sudo systemctl restart pmplatform
```
> 回滚后 `data/yitian_*.json` 三个文件可保留（旧版后端不读它们，无副作用）；如需彻底清除，手工删除即可。**业务数据、账号、门户配置均不受影响** —— 倚天域与主域完全解耦，`analysis_data.json` 的产出与口径本次零改动。
