# 升级手册 —— V3.2.0（视觉增强：表格首行冻结 + 倚天域/数据页重设计；从 V3.1.0 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V3.1.0** 升级到 **V3.2.0**。
> 随附最小更新包：`pmplatform-update-V3.2.0.zip`。
> 若现网早于 V3.1.0，请先用对应旧手册升到 V3.1.0，再用本包。

## 0. 本次升级包含什么（V3.1.0 → V3.2.0）

**纯前端视觉增强，后端 / 数据管线 / 业务口径零改动。** 三块：

- **① 14 张长表加「首行冻结」**：滚动表格下半部分时，表头列名始终可见。涉及 /projects、/projects/closed、/opportunities、/insight/milestone（延期/到期提醒/在建里程碑计划三表）、/insight/costdetail、/projects/key、/opportunities/key、/projects/temp、/risk、/payment/key、/payment/projects、/payment/nodes、/yitian/compliance、/yitian/analytics。用 Element Plus 原生固定表头（表头钉死、表体在表内滚动）。**交互变化**：这些长表由「整页滚动」变为「表内滚动」，筛选栏/统计卡固定在上方不动（多数场景更好用）。

- **② 倚天工时域 5 页重设计**（增可视化，功能与数据不变）：
  - `/yitian` 总览：合规率改**环形**（按 ≥90% 绿 / <90% 黄阈值上色）+ 新增「L4 组织工时」实际/基础分组柱。
  - `/yitian/compliance` 合规：彩色徽章列表 → **问题分布柱** + 新增「问题按 L4 分布柱」+ 新增「**问题码 × L4 热力图**」+ 顶部合规率健康带。
  - `/yitian/analytics` 分析：纯表格 → 新增人数结构条 + 饱和度 TOP10 柱（含基础工时参考线）+「**加班/欠填发散条形**」+「**饱和度分布散点**」。
  - `/yitian/trend` 趋势：新增「**周 / 月 / 季**」切换 + 折线加均值线/峰谷/**缩放条** + 「总工时·合规率」**双轴** + 类型占比改**百分比堆叠**。
  - `/yitian/customer` 客户：新增 TOP1000 各 L4 堆叠柱 + 跨 BG 分组柱 +「**TOP 客户排行**」。

- **③ `/data` 数据管理页按功能域重排**（不加图表）：从「获取→更新→维护折叠」的流程轴，改为六大功能卡——更新看板（主操作）/ PMIS 域 / 项目域文件 / 倚天工时域 / 项目标签 / 维护与历史。**所有功能、上传、更新数据 SSE 进度、超管项、清空数据一个不少，只是版面更清晰。**

> **【头号注意 1】这是纯前端升级——只需换 dist，无需重启后端、无需点「更新数据」、无新增页面/权限。**
> 本包为保持结构一致仍包含后端 `*.py`，但它们**与 V3.1.0 逐字节相同**（本次后端零改动）。你可以只替换 `frontend/dist`；覆盖 `*.py` 也完全无害（内容一样）。
>
> **【头号注意 2】倚天 5 页与部分表页的图表/表头是运行时渲染，务必强制刷新（Ctrl+F5）** 清掉旧的 JS/CSS 缓存，否则可能加载到旧构建。
>
> **【头号注意 3】首行冻结的长表由整页滚动改为表内滚动**——若有用户反馈「表格出现了内部滚动条」，这是预期行为（为了表头钉死）。

## 2. 升级步骤（Ubuntu 服务器，约 2 分钟）

```bash
# 0) 上传 pmplatform-update-V3.2.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V3.2.0.zip
cd /tmp/pmplatform-update-V3.2.0

# 1) 备份现网(可选但建议)
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS

# 2) 覆盖前端 dist(本次升级的核心)
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) (可选)覆盖后端 *.py —— 与 V3.1.0 相同,可做可不做
# sudo cp -f *.py /opt/pmplatform/

# 4) 无需重启后端(纯前端);如你在步骤3覆盖了 *.py 也无妨,重启与否都行
#    若走了反向代理静态缓存,清一下缓存

# 5) 浏览器强制刷新(Ctrl+F5),确认右上角版本号 V3.2.0
```

**本次不需要在页面上做任何配置**（无新页面、无新 pageKey、无需授权、无需点「更新数据」）。

## 4. 回滚

纯前端，回滚只需还原 dist（无需动后端、无需重启）：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
# 浏览器 Ctrl+F5
```
> 业务数据、账号、倚天工时、门户/概算配置均不受影响 —— 本次零后端/零数据改动。
