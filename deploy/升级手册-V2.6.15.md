# 升级手册 —— V2.6.15（签约单位列 + 售前回退 + 佳杰规则标签；从 V2.6.14 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.6.14** 升级到 **V2.6.15**。
> 本包是「**后端代码 + 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.6.15.zip`。
> 若现网早于 V2.6.14，请先用对应旧手册升到 V2.6.14，再用本包。

---

## 0. 本次升级包含什么（V2.6.14 → V2.6.15）

三项围绕「签约单位」的改动：

- **`/projects` 新增「签约单位」列（默认隐藏）**：在列选择器（表头「列」按钮）里勾出即可显示；支持列头枚举筛选。
- **售前服务类项目签约单位回退原项目**：售前项目本身通常无签约单位，现改为显示其**原项目**的签约单位，体现在 `/projects` 该列与 `/project/:id` 详情页顶部。后端新增 `Project.signUnit` 单一来源字段（售前=原项目、非售前=本项目）。
- **「佳杰」规则标签自动化**：签约单位（回退后）精确等于「上海伟仕佳杰科技有限公司」的项目**自动打上「佳杰」标签**，新项目进入清单后自动跟随，**全站一致**（凡显示标签/按标签筛选处都体现，含 /projects、导出、里程碑、成本明细、回款各页）。
  - 实测规则命中约 **128** 个项目（此前手动只标了 23 个），是"手动维护跟不上、改为规则驱动"的预期效果。
  - **规则标签为运行期派生，绝不写入 `project_tags.json`**：若某项目签约单位日后被纠正，佳杰标签自动回收；你手动打的标签与之独立、完全保留。

> **【头号注意 1】⚠️ 本次升级须点一次「更新数据」。** 改动涉及后端数据契约（`schema.py` 新增 `signUnit`）与数据管线（`preprocess_data.py` 生成 `tagSeed`），必须重跑一次「更新数据」生成新的 `data/analysis_data.json`（含 signUnit + tagSeed）后，签约单位列、售前回退、佳杰规则才会生效。
>
> **【头号注意 2】⚠️ 须重启后端。** 本包更新了 `server.py`（标签库首次播种逻辑调整，见下），覆盖后必须重启 systemd 服务。
>
> **【头号注意 3】✅ 对已有标签数据零影响。** 已实测核对：现有 `project_tags.json` 里 59 个已打标签项目、全部手动标签（框架合同 16 / BH项目 12 / 项目已关闭 4 / 退换货 2 / 0元订单 1 / 佳杰 23）**一条不丢**；佳杰在合并后由 23 扩到约 135（=128 规则 ∪ 7 个规则判不出的手动特例）。本包不覆盖、不改写服务器现有 `data/project_tags.json`。
>
> **【头号注意 4】无新增/删除页面、无新增 pageKey、无需授权账号、无新增依赖。** 本包从在线基线 V2.6.14 增量；若现网非 V2.6.14 请先升到 V2.6.14。

## 2. 升级步骤（Ubuntu 服务器，约 2 分钟）

```bash
# 0) 上传 pmplatform-update-V2.6.15.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.6.15.zip

# 1) 备份现网(至少备份代码与 dist;data/input 不动、本包也不覆盖)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py + 前端 dist(不覆盖 data/ 与 input/)
cd /tmp/pmplatform-update-V2.6.15
sudo cp -f *.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f pmisdata/*.py pmisdata/*.sh /opt/pmplatform/pmisdata/ 2>/dev/null || true

# 3) 重启后端(必须,server.py 有改动)
sudo systemctl restart pmplatform    # 服务名以现网为准

# 4) 浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号为 V2.6.15

# 5) 【必须】点一次「更新数据」
#    进入「数据」页 → 点「更新数据」,等 SSE 进度跑完(重跑生成含 signUnit+tagSeed 的
#    新 data/analysis_data.json);现有 data/project_tags.json 不受影响。
```

- [ ] `/project/:id` 详情页：佳杰这类规则标签显示为中性灰、**无删除叉**（只读，鼠标悬停提示"按签约单位自动标记"）；你手动打的标签仍可增删。

## 4. 回滚

若升级后异常：
```bash
sudo systemctl stop pmplatform
sudo rm -rf /opt/pmplatform
sudo mv /opt/pmplatform.bak-<时间戳> /opt/pmplatform
sudo systemctl start pmplatform
```
（本包不改数据；回滚后如需恢复旧口径可再跑一次「更新数据」。现有 `project_tags.json` 全程未被本包改写。）
