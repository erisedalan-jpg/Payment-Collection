# 升级手册 —— V2.6.12（关注原因「总成本超支」拆两档；从 V2.6.11 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.6.11** 升级到 **V2.6.12**。
> 本包是「**后端代码 + 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.6.12.zip`。
> 若现网早于 V2.6.11，请先用对应旧手册升到 V2.6.11，再用本包。

---

## 0. 本次升级包含什么（V2.6.11 → V2.6.12）

一点纯前端口径细化（**无后端/数据结构变化、无口径来源改动**）：

- **`/projects` 关注原因列「总成本超支」拆成两档**：原来一个「总成本超支」标识，现按项目超支额 `overspendAmount` 是否 **> 5000 元**（严格大于，与「成本明细」页卡「超支大于5000」同阈值）拆成两个——
  - **总成本超支大于5000**（超支额 > 5000 元）
  - **总成本超支小于5000**（超支额 ≤ 5000 元；含仅 PMIS「项目超支」标记/消耗比>1 命中、无正超支额的项目）
- 关注原因列的 pill、以及该列的**列头多选筛选**都会自动出现这两档标签（不再有笼统的「总成本超支」）。鼠标悬停 pill 仍显示具体超支金额。
- **不变**：首页「成本超支」风险桶的计数与展示、首页该桶下钻到 `/projects` 的结果、「成本明细」页「总成本超支数」卡——两档合计仍等价原「总成本超支」，口径与数字均不变。

> **【头号注意 1】✅ 本次升级无需点「更新数据」。** 改动全部在前端 `frontend/dist`（后端根 `*.py` 本期无业务改动，随包覆盖仅保持版本一致）。**未改** 数据预处理管线、数据契约（schema）、任何计算口径来源。覆盖文件后浏览器**强刷（Ctrl+F5）**即可生效，现有 `data/analysis_data.json` 与账号原样保留。
>
> **【头号注意 2】无新增/删除页面、无新增 pageKey、无需授权账号、无新增依赖。** 可选重启后端一次更稳妥。
>
> **【头号注意 3】本包从在线基线 V2.6.11 增量。** 若现网非 V2.6.11，请先升到 V2.6.11。

## 1. 最小更新包内容（`pmplatform-update-V2.6.12.zip`）

解压出顶层目录 `pmplatform-update-V2.6.12/`，含：

- `requirements.txt`、`deploy/升级手册-V2.6.12.md`、`pmisdata/` 下载流水线脚本
- **不含** `data/`、`input/`、`tests/`、`docs/`、`node_modules/`

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.6.12.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.6.12.zip

# 1) 备份现网(至少备份代码与 dist,data/input 不动本包也不覆盖)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py + 前端 dist(不覆盖 data/ 与 input/)
cd /tmp/pmplatform-update-V2.6.12
sudo cp -f *.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f pmisdata/*.py pmisdata/*.sh /opt/pmplatform/pmisdata/ 2>/dev/null || true

# 3) 重启后端(可选、稳妥)
sudo systemctl restart pmplatform    # 服务名以现网为准

# 4) 浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号为 V2.6.12
#    本次纯前端展示改动,无需点「更新数据」。
```

## 3. 升级后验证清单

- [ ] 右上角/关于处版本号显示 **V2.6.12**。
- [ ] `/projects` 关注原因列：原「总成本超支」pill 现显示为「总成本超支大于5000」或「总成本超支小于5000」；该列列头筛选出现这两档选项。
- [ ] 首页「成本超支」桶计数不变；点该桶下钻到 `/projects` 结果不变。
- [ ] `/insight/costdetail`「总成本超支数」卡数值不变。
- [ ] 无需点「更新数据」；其它页面数值无变化。

---

## 4. 回滚

若升级后异常：
```bash
sudo systemctl stop pmplatform
sudo rm -rf /opt/pmplatform
sudo mv /opt/pmplatform.bak-<时间戳> /opt/pmplatform
sudo systemctl start pmplatform
```
（本包不改数据，回滚后无需重跑「更新数据」。）
