# 升级手册 —— V2.1.0（生产交付 · 连升第二跳）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的旧版本，升级到 **V2.1.0**。
> **本次为「连升第二跳」**：现网若早于 V2.0.0，请**先套用 V2.0.0 更新包**（见 `deploy/升级手册-V2.0.0.md`）**再套用本 V2.1.0 更新包**。两包都是「纯代码 + /pm 前端 dist」的轻量增量包，**均不含 data/ 与 input/**，不动服务器上现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.1.0.zip`。

---

## 0. 本次升级包含什么

V2.1.0 是**整页级（Y 级）**版本（用户钦定按 Y 级计）。相对 V2.0.0，本轮新增两项：

- **新增「临时重点跟进」页 `/projects/temp`**：「重点跟进」分区下、「重点商机进展」之后。形式同 `/projects/key`（重点项目进展），但**项目范围由超级管理员自定义**——
  - **超管**可在页内「**范围设置**」里用筛选条件圈定项目集：支持**两级 AND/OR 逻辑组合**、**条件级取反**（属于/不属于、区间内/区间外、包含/不包含），维度涵盖**项目级属性**（客户/经理/AR·SR/L4/级别/类型/阶段/状态/健康度/风险/回款率/成本消耗比/TOP1000/象限/暂停/超支/标签/里程碑状态/合同金额/完工%/终验时间等）以及**回款节点、里程碑明细两个子表**（按「任一行满足」判定）。**范围条件可保存**、下次自动按最新数据重算成员（动态范围）。底部实时显示「命中 N 个项目」。
  - **普通管理员**：看不到任何筛选/范围设置入口，只能**编辑表格里「本周工作进展 / 后续工作计划」两列**，且**只看到本人 L4 组织 ∩ 超管所定范围**的项目。
  - 其余功能（历史数据切换、选列、列内筛选、排序、**更新＝归档+清空**、**导出** ）与 `/projects/key` 一致；归档/导出为超管专属。
- **商机页「新增商机」改为先编辑后加行**：`/opportunities` 点「新增商机」不再直接在表尾插入空行，而是**先弹出编辑抽屉**；填好「保存」后才创建并追加该行，「取消」则不产生任何行——避免空行干扰。

> 后端新增模块 `temp_followup.py` + 4 个接口（`/api/temp-followup` GET 任意登录；`/api/temp-followup/update` 任意登录；`/api/temp-followup/scope`、`/api/temp-followup/archive` 超管专属）；临时跟进数据本地持久化于 `data/temp_followup.json`。依赖无新增（仍为标准库 + pydantic + openpyxl）。

> 全部细节见仓库 `PROGRESS.md` 版本史。

## 1. 最小更新包内容（`pmplatform-update-V2.1.0.zip`）

解压得到 `pmplatform-update-V2.1.0/`：
- `*.py` —— 全部后端运行代码（含新增 `temp_followup.py`、以及 `opportunities.py`/`data_scope.py` 等；覆盖旧同名文件）。
- `frontend/dist/` —— **已用 `--base=/pm/` 构建**的前端产物（覆盖旧 dist）。
- `requirements.txt` —— 依赖参考（与旧版一致：`pydantic`、`openpyxl`；通常无需重装）。
- `升级手册-V2.1.0.md` —— 本文件。

**刻意不含** `data/`、`input/`、`deploy/`(systemd/nginx 配置无变化)、`tests/`、`docs/`、`node_modules/`——升级**不覆盖**服务器现有数据/账号/输入文件，nginx 与 systemd 配置沿用。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以下命令以部署在 `/opt/pmplatform` 为例。**务必先备份**。若现网早于 V2.0.0，请先按 `升级手册-V2.0.0.md` 套完 V2.0.0 包，再做下面这一跳。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.1.0.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.1.0.zip').extractall('.')"

# (4) 覆盖后端代码（*.py，含新增 temp_followup.py）
sudo cp /tmp/pmplatform-update-V2.1.0/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.1.0/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> 依赖：V2.1.0 仍只需标准库 + `pydantic` + `openpyxl`（与旧版同，无新增依赖）。若旧环境已能「更新数据」，无需重装。

## 3. 升级后必做/按需操作

1. **账号不变（不新增超管）**：本版本**未改动**后端鉴权与种子逻辑（`auth.py` 无变更）。超管账户仍固定为且仅为 `admin`、`wangxutong`、`zhangyingzhe` 三个，绝不新增第四个。临时跟进的「范围设置」只读取已有 `isSuper` 判定，不创建任何账号。
2. **临时重点跟进首用（重要）**：`/projects/temp` 首次为**空**——需**超管**登录后点页内「**范围设置**」，添加筛选条件圈定项目集并「保存」。保存空范围则页面无项目（设计如此）。范围保存在 `data/temp_followup.json`，之后所有人（按各自 L4）共用这套范围；普通管理员据此编辑进展列。
   - 范围为**动态**：保存的是条件，每次按最新数据重算成员；新符合条件的项目会自动进入、不再符合的自动退出。
3. **深色底色等既有特性**：随 dist 即时生效，无需额外操作。
4. **TOP1000 / 回款口径（若从更早版本一路升来）**：如本次是从早于 V1.19/V1.17.1 的版本连升而来，记得放 `input/TOP1000.xlsx` 后点一次「数据管理 → 更新数据」，让 TOP1000/象限维度与回款「待回款」修正落到 `analysis_data.json`。
   > `/projects/temp` 临时跟进与 `/opportunities` 商机页**均不依赖**「更新数据」（它们独立持久化于 `data/*.json`，不读 `analysis_data.json`）。

## 4. 上线验证清单

- [ ] `systemctl status pmplatform` = running；`curl -I http://<IP>/pm/` 返回 200。
- [ ] 浏览器开 `http://<IP>/pm` → 登录页 → 预制超管登录成功。
- [ ] 左侧「重点跟进」分区出现 **「临时重点跟进」**（位于「重点商机进展」之后）；打开 `/pm/projects/temp` 能加载。
- [ ] **超管**在临时跟进页可见「**范围设置 / 更新（归档+清空）/ 导出**」；点「范围设置」能加组、加条件（项目级 + 回款节点/里程碑子表）、切 AND/OR、底部「命中 N 个项目」随条件变化，保存后页面按范围显示项目。
- [ ] 在某项目「本周工作进展/后续工作计划」单元格点击可编辑保存（保存后该行跟进日期/跟进人刷新）。
- [ ] 新建一个**受限普通管理员**（限某 L4 + 勾选「临时重点跟进」页）：登录后该页**只看到本人 L4 ∩ 范围**的项目、可编辑两列进展、**无**范围设置/更新/导出入口。
- [ ] `/pm/opportunities` 点「新增商机」**先弹编辑抽屉**；填字段保存后才新增一行、点取消不留空行。
- [ ] 安全：未登录 `curl http://<IP>/pm/api/temp-followup` 应 **401**；非超管打 `/pm/api/temp-followup/scope`（保存范围）应 **403**；`/pm/api/temp-followup/update`（编辑进展）普通登录可用。
- [ ] `/pm/data/analysis_data.json` 200 且响应头含 `Content-Encoding: gzip`。

## 5. 回滚

```bash
sudo systemctl stop pmplatform
# 还原代码与 dist
cd /opt/pmplatform && sudo tar xzf /backup/pmplatform-code-<时间>.tgz
# 如数据被污染再还原数据（一般无需）：
# sudo tar xzf /backup/pmplatform-data-<时间>.tgz -C /opt/pmplatform
sudo chown -R pmapp:pmapp /opt/pmplatform
sudo systemctl start pmplatform
```

> 回滚到 V2.0.0：代码 tar 还原即可。临时跟进数据 `data/temp_followup.json` 在旧版本不被读取（无害残留），如需彻底清除可删除该文件。

## 6. 备注

- **临时跟进数据持久化在 `data/temp_followup.json`**（属 `data/`，已被每日 `tar data/` 备份覆盖）。结构：`{scope: 范围条件, current: 当期进展, archives: 历史快照}`。归档（更新）会把当期冻结进 `archives` 并清空 `current`。
- 本次为**纯代码 + dist** 轻量更新，未改 systemd/nginx 配置，未改依赖。
- 若你需要的是**全新安装的整包**（非增量更新），在开发机执行 `python make_deploy_zip.py` 生成 `pmplatform-deploy-V2.1.0.zip`（含全部代码 + /pm dist + data/ + input/），按 `deploy/部署-Ubuntu24-端口80-pm.md` 全新部署。
- 端口 80 为明文 HTTP（仅可信内网）；不可信网络建议上 443 + TLS（见部署手册 §11）。
