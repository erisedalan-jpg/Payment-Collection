# 升级手册 —— V2.2.2（风险子页面全列展示 + 商机放开普通管理员写[仅本人 L4]）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的旧版本升级到 **V2.2.2**。
> 本包是「**纯代码 + /pm 前端 dist**」的轻量增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.2.2.zip`。
> 现网若早于 V2.2.1，本包代码为**全量覆盖所有 .py + dist**（升到 V2.2.2 无遗漏），但被跳过版本的**首用/数据/依赖步骤**仍需照对应手册补做（如 V2.2.0 的「重点商机跟进」页访问授权、V2.1.1 的下载依赖 `requests`/`openpyxl` 与 pmisdata、V2.0.0 的商机 seed 与「更新数据」）。

## 0. 本次升级包含什么

V2.2.2 是**子页面/页内局部级（Z 级）**版本。相对 V2.2.1，本轮两块：

- **`/project/:id` 风险子页面全量列展示 + 长文本换行**：
  - 项目详情页「风险」子页的**风险明细表**由原来的精选 13 列改为**展示全部 43 列**（风险数据导出表的全部字段）。
  - 长文本列（风险描述、待办任务、备注等）**全文换行展示、不再省略号截断**。
  - 数据本就在后端，纯前端展示调整，**不改后端、不改数据**。
- **`/opportunities`（商机清单）放开普通管理员写权限（仅本人 L4 范围）**：
  - **行为变化**：普通管理员现在**可以编辑、可以新增商机**（此前为只读）。
  - **L4 隔离仍然成立**：普通管理员**只能编辑/新增本人 L4 组织范围内**的商机；改他人 L4 的行、或把商机移出本人 L4，后端一律拒绝（403）。新增时若该账号只有一个 L4，自动落在该 L4。
  - **删除 / 导入 / 导出 / 批量选择仍为超级管理员专属**（未放开）。
  - 超级管理员行为不变（全量可写）。

> 本版本为**前端展示 + 商机写权限放开（含后端 L4 校验）**。涉及后端 `server.py`、`opportunities.py` 改动（已在 `*.py` 全量覆盖内）。**不改回款/分析计算口径，不新增运行依赖，不需点「更新数据」，不新增页面、不新增页面访问 key。**
> 全部细节见仓库 `PROGRESS.md` 版本史。

> 说明：V2.0.0 原始设计中「商机对普通管理员只读」，本版本按需求方决定**主动放开为可写（限本人 L4）**。这是一次刻意的权限调整，非缺陷修复。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以下命令以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.2.2.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.2.2.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）
sudo cp /tmp/pmplatform-update-V2.2.2/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.2.2/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过；
#   仅当现网早于 V2.1.1、尚无 pmisdata/ 时才整套拷入：
#   sudo cp -rn /tmp/pmplatform-update-V2.2.2/pmisdata /opt/pmplatform/

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **无新增 Python 依赖**：本功能全为既有标准库 + pydantic/openpyxl。无需 pip 安装任何东西。
> **无需点「更新数据」**：本版本未改 `preprocess_data.py`，看板口径不变；商机/跟进数据独立持久化，结构不变。

## 3. 升级后必做/按需操作

1. **账号不变（不新增超管）**：本版本**未改动 `auth.py`**。超管仍固定为 `admin`、`wangxutong`、`zhangyingzhe` 三个。
2. **无新增页面访问授权**：本版本**未新增页面**（风险是项目详情的子页；商机清单/重点商机跟进页均已存在）。各账号可访问页面无须改动。
3. **普通管理员商机写权限（重要行为变化，需知会相关账号）**：
   - 升级后，能访问「商机清单」的**普通管理员可编辑、可新增商机**，但**仅限本人 L4 组织范围**（后端强制；越权返回 403）。
   - **删除 / 导入 / 导出**仍仅超管可用。
   - 如不希望某普通管理员能改商机，可在「页面访问控制」里取消其对商机清单页（pageKey `opportunities-progress`）的访问授权（连查看一并收回）。
4. **风险全列查看**：进任一项目 `/project/:id` →「风险」子页，明细表应展示全部列、长文本换行。
5. **nginx/systemd 无变化**：本版本未改 systemd 单元与 nginx 配置。

## 5. 回滚

```bash
sudo systemctl stop pmplatform
# 还原代码与 dist
cd /opt/pmplatform && sudo tar xzf /backup/pmplatform-code-<时间>.tgz
sudo chown -R pmapp:pmapp /opt/pmplatform
sudo systemctl start pmplatform
```

> 回滚到 V2.2.1：代码 tar 还原即可（商机写权限恢复为超管专属、风险明细恢复 13 列）。普通管理员升级期间写入的商机数据仍保留在 `data/opportunities.json`，旧前端按 L4 只读照常显示，无副作用。商机/跟进 JSON 结构未变，无需处理。
