# 升级手册 —— V2.6.13（成本明细风险两列 + 三页范围补齐；从 V2.6.12 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.6.12** 升级到 **V2.6.13**。
> 本包是「**后端代码 + 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.6.13.zip`。
> 若现网早于 V2.6.12，请先用对应旧手册升到 V2.6.12，再用本包。

---

## 0. 本次升级包含什么（V2.6.12 → V2.6.13）

两点纯前端展示/筛选增强（**无后端/数据结构变化、无口径来源改动**）：

- **`/insight/costdetail` 项目成本明细加两列**：
  - **项目风险**：项目最高风险等级 + 未关闭风险数（与 `/projects`「风险」列同款，如「高(3)」）。
  - **风险大类**：该项目所有风险记录的「风险大类」去重后清单式列出（多条一行一项）；无风险显「-」。
  - 「项目风险」列可排序、可列头筛选；「风险大类」为清单列，仅展示（不进列头筛选）。导出 Excel 亦含这两列。
- **`/risk`、`/projects/temp`、`/payment/key` 三页范围设置「项目」类字段补齐到 /projects 全部列**：
  - `/projects/temp`、`/payment/key`：新增可按「关注原因」筛选。
  - `/risk`：新增可按 项目阶段 / 完工进展 / 项目最高风险等级 / 未关闭风险数 / 预算消耗比 / 回款完成率 / 健康度 / 关注原因 / 回款状态 / TOP1000 / 象限 等项目级字段筛选。
  - （「标签」范围筛选本次暂未纳入，后续单独处理。）

> **【头号注意 1】✅ 本次升级无需点「更新数据」。** 改动全部在前端 `frontend/dist`（后端根 `*.py` 本期无业务改动，随包覆盖仅保持版本一致）。**未改** 数据预处理管线、数据契约（schema）、任何计算口径来源。覆盖文件后浏览器**强刷（Ctrl+F5）**即可生效，现有 `data/analysis_data.json` 与账号原样保留。
>
> **【头号注意 2】无新增/删除页面、无新增 pageKey、无需授权账号、无新增依赖。** 可选重启后端一次更稳妥。
>
> **【头号注意 3】本包从在线基线 V2.6.12 增量。** 若现网非 V2.6.12，请先升到 V2.6.12。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.6.13.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.6.13.zip

# 1) 备份现网(至少备份代码与 dist,data/input 不动本包也不覆盖)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py + 前端 dist(不覆盖 data/ 与 input/)
cd /tmp/pmplatform-update-V2.6.13
sudo cp -f *.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f pmisdata/*.py pmisdata/*.sh /opt/pmplatform/pmisdata/ 2>/dev/null || true

# 3) 重启后端(可选、稳妥)
sudo systemctl restart pmplatform    # 服务名以现网为准

# 4) 浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号为 V2.6.13
#    本次纯前端展示改动,无需点「更新数据」。
```

- [ ] `/projects/temp`、`/payment/key` 范围设置：项目类多出「关注原因」。
- [ ] 无需点「更新数据」；其它页面数值无变化。

## 4. 回滚

若升级后异常：
```bash
sudo systemctl stop pmplatform
sudo rm -rf /opt/pmplatform
sudo mv /opt/pmplatform.bak-<时间戳> /opt/pmplatform
sudo systemctl start pmplatform
```
（本包不改数据，回滚后无需重跑「更新数据」。）
