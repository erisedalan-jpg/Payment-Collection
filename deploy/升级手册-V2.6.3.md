# 升级手册 —— V2.6.3（回款域重构 4 期累积包；从 V2.5.9 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.5.9** 升级到 **V2.6.3**。
> 本包是「**纯代码 + /pm 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.6.3.zip`。
> **累积包**：一次从在线基线 **V2.5.9** 升到 **V2.6.3**，累积含 V2.6.0 / V2.6.1 / V2.6.2 / V2.6.3 全部改动。若现网早于 V2.5.9，请先按对应手册升到 V2.5.9，再用本包。

## 0. 本次升级包含什么（V2.5.9 → V2.6.3，回款域重构 4 期）

这是「回款域重构 + 统一标签筛选 + 回款跟进新页 + 列表分页」大改的一次性交付（分 4 期开发、累积成一个包）。**背景**：此前「在建项目总数 638」与「回款页看到 557」不一致——查明是 **81 个在建项目没有收款阶段节点**（并非数据错误），本次统一口径并显式化。

**V2.6.0（P1）回款域重构**
- 回款总览 `/payment` 重做：「项目数」卡改为**整体在建（638）** + 副字「N 个项目无回款阶段」（整卡可点跳「在建项目」）；「延期项目数」卡可下钻、「回款节点数」卡跳「回款节点」；删「待回款金额」「服务组达成排名」两卡；「回款数据」表增大；**新增「无回款阶段数据项目」清单**（可点进详情、可导出、分页）。
- 回款项目 `/payment/projects` 升级为「同在建项目」的表格（排序/列头筛选/选列/导出/标签筛选）+ 新增「项目级别」列、去「来源」列；下钻抽屉加宽；无收款阶段的项目点开显示「该项目无回款阶段数据」+「查看完整详情」按钮。
- 回款节点 `/payment/nodes` 去掉「维度」切换，保留 5 卡；主表加「项目经理」「L4 组」列 + 排序/筛选/标签/导出。
- **删除**「回款进度」`/payment/plan`、「风险项目」`/payment/risk`、「回款台账」`/ledger` 三页（侧栏移除；旧网址自动跳「回款总览」，不 404）。

**V2.6.1（P2）回款重点跟进新页 + 修复**
- **新增「回款重点跟进」页 `/payment/key`**（侧栏「重点跟进」区、风险跟进下）：仿「临时重点跟进」的范围引擎 + 进度编辑（跟进动作 / rev结论 / 下次rev时间），**归档为「仅归档不清空」**（同风险跟进，跟进留存）。
- 修复：回款项目/回款节点标签筛选下拉缩窄；回款总览「项目数」卡整卡可点、副字纯展示；「无回款阶段数据项目」清单加分页。

**V2.6.2（P3）统一标签筛选（含「无标签」）**
- 「在建项目」`/projects`、「项目分析」`/insight`、「成本分析」`/insight/costdetail`（项目成本明细表）、「里程碑管理」`/insight/milestone`（下方三表）新增**统一标签筛选**：一个多选下拉，选项 = `无标签` + 各标签，可多选并集（选「无标签」= 看没打标签的项目）。此筛选每页独立，与顶部「按标签排除」（影响全站统计口径的那个）是两套不同机制、互不影响。

**V2.6.3（P4）列表分页 + 风险页筛选**
- 「重点项目进展」`/projects/key`、「重点商机跟进」`/opportunities/key`、「临时重点跟进」`/projects/temp`、「回款重点跟进」`/payment/key`、「风险跟进」`/risk` 五页加**分页 + 总数统计**（「共 N 条」，同「在建项目」页），避免单页渲染过大。
- 「风险跟进」`/risk` 的 **rev结论、下次rev时间两列新增列头筛选**。

> **【头号注意 1】本次升级无需点「更新数据」。** 全程未改数据契约（schema）、未改数据处理（preprocess）、未改任何数据来源。前端复用现有 `data/analysis_data.json`；回款重点跟进的记录存于新文件 `data/payment_key_followup.json`（**首次访问自动创建**，无需手工建）。
>
> **【头号注意 2·必做】给普通管理员授权新页面「回款重点跟进」。** V2.6.1 新增了页面访问权限项 `payment-key`。升级后，用预制超管登录 → 「系统管理 / 账号管理」→ 编辑需要用该页的普通管理员 → 「可访问页面」勾上「**回款重点跟进**」→ 保存。（超管默认可见所有页面，无需操作。）另 V2.6.0 删除了 3 个页面权限项（payment-plan / payment-risk / ledger），原来勾过的会自动失效、无副作用。
>
> **【头号注意 3·部署要点】本包改了后端 `server.py`（V2.6.1 新增回款跟进接口），务必按下面步骤重启 systemd 服务**——后端 Python 代码只有重启进程才生效（前端 dist 是按请求读磁盘）。
>
> **无新增 Python 依赖、无 systemd/nginx 配置变更、无新数据源文件。** 后端新增模块 `payment_key_followup.py`（随 `*.py` 一起覆盖）。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.6.3.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.6.3.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（含新增 payment_key_followup.py + 改动的 server.py）
sudo cp /tmp/pmplatform-update-V2.6.3/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.6.3/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写；新 json 首访自建也需可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务（后端改动靠这一步生效）
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本包无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx。** 但**必须重启服务**（步骤 6/7）——否则 V2.6.1 的回款跟进接口不生效、`/payment/key` 会报错。
> **起服务后，务必执行「头号注意 2」给普通管理员授权「回款重点跟进」。**
> 若浏览器仍显示旧样式/旧行为：强制刷新（Ctrl+F5）。

## 4. 回滚

```bash
sudo systemctl stop pmplatform
# 还原代码与 dist
cd /opt/pmplatform && sudo tar xzf /backup/pmplatform-code-<时间>.tgz
sudo chown -R pmapp:pmapp /opt/pmplatform
sudo systemctl start pmplatform
```

> 本包纯前端 + 新增后端 followup 接口/文件，未改 schema/数据/既有本地 JSON 结构，回滚后**无需**点「更新数据」，数据与账号不受影响。回滚后被删的三页会重新出现、`/payment/key` 与新特性消失。`data/payment_key_followup.json` 可保留（回滚版本不读它，无副作用）。
