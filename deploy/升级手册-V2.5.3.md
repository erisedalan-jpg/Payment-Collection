# 升级手册 —— V2.5.3（侧栏版式统一 + 首页「成本超支」计数去重，纯前端增量；从 V2.5.2 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.5.2** 升级到 **V2.5.3**。
> 本包是「**纯代码 + /pm 前端 dist**」的轻量增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.5.3.zip`。
> **本期为纯前端单版本增量**（在线基线已是 V2.5.2）。若现网早于 V2.5.2，请先按 `deploy/升级手册-V2.5.2.md` 升到 V2.5.2，再用本包升 V2.5.3。

## 0. 本次升级包含什么（V2.5.3，两项·均纯前端）

1. **左侧导航版式统一**：原先「项目 / 工具 / 系统管理」三个分区下的子项字号偏大、缩进偏小，与「项目分析 / 重点跟进 / 回款」三个分区下的子项不一致、不对齐。本次**统一为与「项目分析」一致**——六个分区下的子项字号、左缩进全部对齐。
2. **首页「需要处理的异常 → 成本超支」卡片计数去重**：旧逻辑把「整体预算超支」与「交付成本超支」两类**分别累加**，同一个项目若**既整体超支又交付超支**会被**重复计数**。本次改为**按项目去重**（同一项目只计一次）。
   - **影响**：升级后该卡片数字会**变小**（去掉重复计的项目），这是**预期的修正**。例如某现场由旧的 107 修正为 87（实际数取决于现网数据：=「整体超支项目」∪「交付超支项目」的去重总数）。其余分诊卡（回款延期/里程碑滞后/风险未闭环等）不受影响。

> **【头号注意】本次升级无需点「更新数据」。** 本期为**纯前端**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何口径来源；「成本超支」去重只是前端分类计数方式的修正，分子分母数据源不变。前端复用现有 `data/analysis_data.json` 即可。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面/无新页面访问 key、未改 `auth.py`（超管不变）。**
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.5.3.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.5.3.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本期与 V2.5.2 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.5.3/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本期核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.5.3/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本期无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧样式：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。

```bash

cd /opt/pmplatform && sudo tar xzf /backup/pmplatform-code-<时间>.tgz
sudo chown -R pmapp:pmapp /opt/pmplatform
sudo systemctl start pmplatform
```
