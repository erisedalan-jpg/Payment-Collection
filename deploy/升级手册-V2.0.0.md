# 升级手册 —— V2.0.0（生产交付）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的旧版本，原地升级到 **V2.0.0**。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 本次随附「**最小更新包**」`pmplatform-update-V2.0.0.zip`：**只含代码 + /pm 前端 dist + 本手册**，**不含 data/ 与 input/**——升级时**不动**服务器上的现有数据与账号。

## 0. 本次升级包含什么

V2.0.0 是**大版本**（用户钦定）。相对最后一次生产版本（部署包名为 `pmplatform-deploy-V1.16.4.zip`），累计合入并一次交付以下内容，**其中三项是本轮（V2.0.0）新做**：

**V2.0.0 本轮三块：**
- **新增「重点商机进展」页 `/opportunities`（线上可编辑表格，主体）**：「重点跟进」分区下。后端新增 `opportunities.py` + 5 个接口；本地持久化 `data/opportunities.json`；首次访问从 `input/opportunities.xlsx` 读初始数据。**超级管理员**可编辑/新增/删除/导入/导出；**普通管理员只读且仅见本人 L4 组织的行**（后端按 `allowedL4` 裁剪）。
- **深色底色调亮**：深色模式两近黑底色调亮（柔和黑/深灰），观感更柔和。
- **`/projects/key` 重点项目进展页增强**：历史快照选项卡改下拉菜单；导出弹窗加「全选」。

**自 V1.16.4 以来累计（若你的生产早于这些版本，本次一并生效）：**
- **登录与权限体系**（V1.17.x）：登录页 + 页面门禁 + L4 数据隔离 + 超管建号/调权（`/admin`）；普通管理员**首次登录强制改密**。
- **TOP1000 客户属性**（V1.19.x）：按最终客户派生 `top1000`/象限，多页加维度（需 `input/TOP1000.xlsx` + 点「更新数据」生效）。
- **侧边栏分区可折叠**（V1.19.2）、**重点跟进分区 + `/projects/key` 进展页**（V1.20.0）、**`/insight/risk` 风险看板透视+下钻**（V1.18.0/V1.20.1）、**`/insight/milestone` 到期提醒表改造**（V1.20.2）。
- 生产 bug 修复（V1.17.1）：**跨账号缓存数据泄漏**（安全）、**回款「待回款=0」**（需点「更新数据」生效）。

> 全部细节见仓库 `PROGRESS.md` 版本史。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以下命令以部署在 `/opt/pmplatform` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.0.0.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.0.0.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）。先删旧版可能已废弃的模块再覆盖更稳妥；最少要做覆盖：
sudo cp /tmp/pmplatform-update-V2.0.0/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.0.0/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

# (6) 放新增数据源（可选但推荐，见 §3）
#   - 商机初始数据： 把 opportunities.xlsx 放到 /opt/pmplatform/input/
#   - TOP1000 客户： 把 TOP1000.xlsx 放到 /opt/pmplatform/input/

# (7) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (8) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> 依赖：V2.0.0 仍只需标准库 + `pydantic` + `openpyxl`（与旧版同，无新增依赖）。若旧环境已能「更新数据」，无需重装。

## 3. 升级后必做/按需操作

1. **账号（自动，幂等；不会新增超管）**：超管账户**固定为且仅为这三个**——`admin`、`wangxutong`、`zhangyingzhe`。服务启动时 `auth.py` 的种子只做一件事：**确保这三个存在**——`data/accounts.json` 里缺哪个就补哪个、**三个都在则完全无改动**，**绝不新增第四个超管**，也不改动任何既有账号/口令/权限。预制口令：`admin`/`wxtnb`、`wangxutong`/`niubi`、`zhangyingzhe`/`venus600`。
   - V2.0.0 **未改动**后端鉴权与种子逻辑（`auth.py` 无变更）；商机功能只读取已有 `isSuper`，不创建任何账号。
   - 用预制口令登录即可；建议登录后在 `/admin` 改超管口令、按需新建受限管理员（限某 L4 + 页面）。**新建的普通管理员首次登录会被强制改密。**
2. **商机页初始数据（按需）**：`/opportunities` 首次被访问时，若无 `data/opportunities.json`，会**自动**从 `input/opportunities.xlsx` 读初始数据并落地（**无需**点「更新数据」）。
   - 若放了 `input/opportunities.xlsx`：首次进页即见初始商机；之后超管在页内增删改/导入会持久化到 `data/opportunities.json`。
   - 若**没放**：页面初始为空，超管可「新增商机」逐条录入或「导入」一个 xlsx。
   - 表头中文名须与页面列名一致（如「客户名称」「L4组织」「商机状态」「预估金额(万元)」等）；金额列全角/半角括号都接受。
3. **点一次「数据管理 → 更新数据」（重要，让以下口径生效）**：以下两项需重跑预处理后才落到 `analysis_data.json`：
   - **TOP1000 客户属性**：需先放 `input/TOP1000.xlsx`，再「更新数据」，`/projects`、`/insight` 等的 TOP1000/象限维度才有值；否则显「否/空」。
   - **回款「待回款」修正（V1.17.1）**：「更新数据」后日历/看板待回款金额才正确。
   - 命令行等价：`sudo -u pmapp /opt/pmplatform/.venv/bin/python /opt/pmplatform/preprocess_data.py`
   > `/opportunities` 商机页**不依赖**这一步（它独立持久化，不读 `analysis_data.json`）。
4. **深色底色调亮**：随 dist 即时生效，无需额外操作。

## 4. 上线验证清单

- [ ] `systemctl status pmplatform` = running；`curl -I http://<IP>/pm/` 返回 200。
- [ ] 浏览器开 `http://<IP>/pm` → 登录页 → 预制超管登录成功。
- [ ] 左侧「重点跟进」分区出现 **「重点商机进展」**；打开 `/pm/opportunities` 页能加载。
- [ ] **超管**在商机页可见「新增商机/删除选中/导入/导出/编辑」；新增一行能打开编辑抽屉录入并保存（保存后该行「最后一次更新日期」刷新、首次有内容时盖「首次登记日期」）。
- [ ] 新建一个**受限普通管理员**（限某 L4 + 勾选「重点商机进展」页）：登录后商机页**只看到该 L4 的行**、且**无**新增/删除/导入/导出/编辑入口。
- [ ] 深色模式底色比旧版更柔和（非纯黑）；`/projects/key` 历史快照为下拉菜单、导出弹窗有「全选」。
- [ ] 安全：未登录 `curl http://<IP>/pm/api/opportunities` 应 **401**；非超管打 `/pm/api/opportunities/create` 等写接口应 **403**；`/pm/data/accounts.json` 带普通 Cookie 应 **403**。
- [ ] `/pm/data/analysis_data.json` 200 且响应头含 `Content-Encoding: gzip`。
