# 升级手册 —— V2.6.11（回款数据总计行 + 四跟进页合同金额合计；从 V2.6.10 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.6.10** 升级到 **V2.6.11**。
> 本包是「**后端代码 + 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.6.11.zip`。
> 若现网早于 V2.6.10，请先用对应旧手册升到 V2.6.10，再用本包。

---

## 0. 本次升级包含什么（V2.6.10 → V2.6.11）

两点纯展示层体验完善（**均为前端改动，无口径迁移、无后端/数据结构变化**）：

- **`/payment` 回款总览「回款数据」表加总计行**：L4 组汇总表最下方新增一行「合计」，恒显示在表底、**不随排序移动**。计数/金额列（项目数、合同额、已回款、延期金额、各节点数）为各行之和；两个比率列（回款额完成率、完成节点比例）**按口径重算**（Σ已回款 ÷ Σ合同、Σ完成节点 ÷ Σ回款节点，非把各行百分比相加），与全站回款完成率口径一致。
- **四跟进页页脚加「合同金额合计」**：`/projects/key`（重点项目进展）、`/projects/temp`（临时重点跟进）、`/risk`（风险跟进）、`/payment/key`（回款重点跟进）四页页脚由「共 N 条」改为「**合同金额合计 X 万 · 共 N 条**」，数值随当前列筛选/范围联动。合计按**项目**去重（`/risk` 一行一条风险、同一项目多条风险只计一次合同额，故其合计对应「不同项目」的合同额、通常小于风险条数）。

> **【头号注意 1】✅ 本次升级无需点「更新数据」。** 改动全部在前端 `frontend/dist`（外加后端根 `*.py` 一并覆盖以保持一致），**未改** 数据预处理管线（`preprocess_data.py`）、数据契约（schema）、任何计算口径来源。覆盖文件后浏览器**强刷（Ctrl+F5）**即可生效，现有 `data/analysis_data.json` 与账号原样保留。
>
> **【头号注意 2】无新增/删除页面、无新增 pageKey、无需授权账号、无新增依赖。** 后端 `*.py` 本期实质无业务改动（随包覆盖仅为版本一致），可选重启一次更稳妥。
>
> **【头号注意 3】本包从在线基线 V2.6.10 增量。** 若现网非 V2.6.10，请先升到 V2.6.10。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.6.11.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.6.11.zip

# 1) 备份现网(至少备份代码与 dist,data/input 不动本包也不覆盖)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py + 前端 dist(不覆盖 data/ 与 input/)
cd /tmp/pmplatform-update-V2.6.11
sudo cp -f *.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f pmisdata/*.py pmisdata/*.sh /opt/pmplatform/pmisdata/ 2>/dev/null || true

# 3) 重启后端(可选、稳妥)
sudo systemctl restart pmplatform    # 服务名以现网为准

# 4) 浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号为 V2.6.11
#    本次纯前端展示改动,无需点「更新数据」。
```

## 3. 升级后验证清单

- [ ] 右上角/关于处版本号显示 **V2.6.11**。
- [ ] 打开 `/payment`（回款总览）：上方 6 卡下方「回款数据」表**最下方出现一行「合计」**；合同额/已回款为各行之和；点任意列排序时合计行**始终在表底不动**；完成率一栏是 Σ已回款/Σ合同（不是各行百分比相加的怪值）。
- [ ] 打开 `/projects/key`、`/projects/temp`、`/risk`、`/payment/key` 四页：页脚显示「**合同金额合计 X 万 · 共 N 条**」；做一次列筛选，合计随可见行变化。
- [ ] 无需点「更新数据」；其它页面数值无变化。

---

## 4. 回滚

若升级后异常：
```bash
sudo systemctl stop pmplatform
sudo rm -rf /opt/pmplatform
sudo mv /opt/pmplatform.bak-<时间戳> /opt/pmplatform
sudo systemctl start pmplatform
```
（本包不改数据，回滚后无需重跑「更新数据」。）
