# 升级手册 —— V2.8.5（/risk 两列百分比显示；从 V2.8.4 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.8.4** 升级到 **V2.8.5**。
> 本包是「**纯前端**」增量包——**只换 `frontend/dist`**，后端代码与本次一字未改。
> 随附最小更新包：`pmplatform-update-V2.8.5.zip`。
> 若现网早于 V2.8.4，请先用对应旧手册升到 V2.8.4，再用本包。

---

## 0. 本次升级包含什么（V2.8.4 → V2.8.5）

一处小改，后端零改动：

- **/risk（风险跟进）「回款完成率」「完工进展」两列改百分比显示**：原来显示原始小数（如 `1`、`0.7`），现改为百分比（`1` → `100%`、`0.7` → `70%`、空值 → `-`）。口径与 /projects 页同名两列完全一致（整数不留小数、有小数保留 1 位）。

> **【头号注意 1】✅ 纯前端，无需重启后端、无需点「更新数据」。** 本包只更新 `frontend/dist`，后端 `*.py` 与 V2.8.4 完全一致（包内附带的 .py 覆盖上去是同内容、无副作用）。
>
> **【头号注意 2】无新增页面 / 路由 / pageKey、无需授权、无新增依赖。** 从在线基线 V2.8.4 增量。仅显示格式变化，底层数据与排序/筛选口径不变。

## 1. 最小更新包内容（`pmplatform-update-V2.8.5.zip`）

解压出顶层目录 `pmplatform-update-V2.8.5/`，含：
- `frontend/dist/`（**已用 `--base=/pm/` 构建**；本期改动：RiskFollowupView 两列百分比格式化 + `version.ts`）——**这是本次唯一实质变化**。
- 全部后端根 `*.py`（与 V2.8.4 一致，覆盖无副作用）、`requirements.txt`、`deploy/升级手册-V2.8.5.md`、`pmisdata/` 脚本。
- **不含** `client/`、`data/`、`input/`、`tests/`、`docs/`、`node_modules/`。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.8.5.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.8.5.zip

# 1) 备份现网 dist(可选)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖前端 dist(核心)
cd /tmp/pmplatform-update-V2.8.5
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
#   (可选)后端 .py 与 V2.8.4 一致,覆盖与否均可:
#   sudo cp -f *.py /opt/pmplatform/

# 3) 无需重启后端(后端未变)、无需点「更新数据」。
#    浏览器强制刷新(Ctrl+F5),确认右上角版本号 V2.8.5。
```

> 注：若你的部署用 nginx 缓存了静态资源，强刷仍是旧版时，`sudo systemctl reload nginx` 或清 nginx 缓存后再强刷。

## 4. 回滚

纯前端，回滚极简：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
```
（无后端/数据改动，回滚只换回旧 dist、强刷即可。）
