# 升级手册 —— V2.7.0（账号管理审计 / 全操作审计；从 V2.6.15 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.6.15** 升级到 **V2.7.0**。
> 本包是「**后端代码 + 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.7.0.zip`。
> 若现网早于 V2.6.15，请先用对应旧手册升到 V2.6.15，再用本包。

---

## 0. 本次升级包含什么（V2.6.15 → V2.7.0）

新增一套**账号管理审计（全操作审计）**子系统，**仅超级管理员可见**，在**账号管理页**里以标签页形式呈现：

- **审计范围（全操作审计）**：
  - **认证**：登录成功、登录失败（记账号名 + 原因「密码错误 / 账号不存在」，**绝不记密码**）、登出。
  - **账号管理**：创建 / 修改 / 删除账号、修改本人密码（记操作者、目标账号、授了哪些权限、是否重置密码；**不记密码/哈希**）。
  - **数据运维**：数据更新（reprocess）、清空数据、数据回滚 / 撤销、人工导入 / 回滚、文件上传、PMIS 拉取、更新 PMIS Cookie、停止服务。
  - **业务写入**：跟进记录增删改、标签保存、项目进展、四类跟进（临时 / 商机 / 风险 / 回款重点）的范围/更新/归档、商机增删改导入。
- **查看页**：`/admin`（账号管理）页顶部新增 `[账号管理] [审计日志]` 两个标签；「审计日志」标签支持按**账号 / 事件类型（多选）/ 日期区间 / 结果（成功·失败）/ 关键字**筛选，分页浏览，并可**导出当前筛选结果为 xlsx**。沿用 `/admin` 的超管门禁，普通管理员看不到。
- **每行记录字段**：时间、账号、显示名、IP、UA（截断）、事件码、动作（中文）、method、path、HTTP 状态、成功与否、目标（如目标账号/项目号）、详情。你最初关心的「登录时间 / 登录 IP / 用户名称 / 登出时间」一一落位（登出时间取显式登出事件；多数会话因 12h 静默过期不追踪，无登出行，属预期）。
- **存储与归档**：审计写入 `data/audit_log.jsonl`（纯文本，服务器本地生成，不入包、不覆盖）；活动日志保留**最近 1 万条且 365 天**，**超出部分自动滚入按年归档 `data/audit_archive/audit-YYYY.jsonl`（纯文本、永不自动删除）**，作为长期留痕。

> **【头号注意 1】⚠️ 须重启后端。** 本包更新了 `server.py` 并新增 `audit.py`，覆盖后必须重启 systemd 服务，审计埋点与查询端点才生效。
>
> **【头号注意 2】✅ 不需点「更新数据」。** 审计子系统与 `data/analysis_data.json` **完全无关**，本次升级**无需重跑「更新数据」**（当然，跑一次也无妨、不影响审计）。
>
> **【头号注意 3】✅ 无需改 nginx。** 审计取真实客户端 IP 依赖反代传入的 `X-Forwarded-For` / `X-Real-IP`——现网 nginx 配置**早已包含**这两个头（见 `deploy/nginx-pmplatform-port80-pm.conf`），无需改动。若你的 nginx 是自行精简过、未设这两个头，则审计里 IP 会显示为 `127.0.0.1`，补上 `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;` 即可。
>
> **【头号注意 4】仅超管可见、无新增页面/路由/pageKey、无需授权账号、无新增依赖。** 审计标签内嵌在既有 `/admin` 账号管理页（本就超管专属），不新增独立路由，普通管理员账号无需任何改动。本包从在线基线 V2.6.15 增量。
>
> **【头号注意 5】✅ 隐私红线。** 审计**绝不记录**明文密码、密码哈希、salt、会话 token、cookie、完整请求体。审计文件位于 `data/`，非超管无法经界面或直链读取。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.7.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.7.0.zip

# 1) 备份现网(至少备份代码与 dist;data/input 不动、本包也不覆盖)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py + 前端 dist(不覆盖 data/ 与 input/)
cd /tmp/pmplatform-update-V2.7.0
sudo cp -f *.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f pmisdata/*.py pmisdata/*.sh /opt/pmplatform/pmisdata/ 2>/dev/null || true

# 3) 重启后端(必须,server.py 有改动 + 新增 audit.py)
sudo systemctl restart pmplatform    # 服务名以现网为准

# 4) 浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号为 V2.7.0

# 5) 无需点「更新数据」。审计从本次重启后开始积累。
```

## 3. 升级后验证清单

- [ ] 右上角/关于处版本号显示 **V2.7.0**。
- [ ] 以超级管理员登录，进入「账号管理」页，顶部出现 `[账号管理] [审计日志]` 两个标签。
- [ ] 切到「审计日志」标签：能看到刚才这次**登录成功**的记录（账号、时间、IP）。
- [ ] 随手做一次写操作（如新建/修改一个测试账号、或保存一次标签），回审计日志能看到对应的**动作/目标/结果**记录。
- [ ] 用**错误密码**登录一次 → 审计出现「登录失败」记录，原因为「密码错误」；**该记录不含你输入的密码明文**。
- [ ] 审计日志支持按账号/事件/日期/结果/关键字**筛选**，并能**导出 xlsx**。
- [ ] 用**普通管理员**账号登录：看不到「审计日志」标签（`/admin` 本就超管专属）。
- [ ] IP 列显示的是访问者真实内网 IP（而非 `127.0.0.1`）——若显示 `127.0.0.1`，见头号注意 3 补 nginx 头。

## 4. 回滚

若升级后异常：
```bash
sudo systemctl stop pmplatform
sudo rm -rf /opt/pmplatform
sudo mv /opt/pmplatform.bak-<时间戳> /opt/pmplatform
sudo systemctl start pmplatform
```
（本包不改数据；回滚后审计子系统消失，已产生的 `data/audit_log.jsonl` / `data/audit_archive/` 会保留在磁盘上但旧版不读取，无害。）
