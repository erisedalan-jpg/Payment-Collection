# 升级手册 —— V4.1.1（导出按可见列 + 成本明细表显示修复；**从 V4.1.0 单版升**）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V4.1.0** 升到 **V4.1.1**。
> 本包是「**纯前端**」增量包——**只换 dist + 重启 app + 浏览器强刷**。**不改后端数据结构、不需点「更新数据」、不需备份任何 data 文件、无新增页面或权限项。**
> 随附最小更新包：`pmplatform-update-V4.1.1.zip`。
> 若现网不是 V4.1.0，请先用对应旧手册升到 V4.1.0，再用本包。

---

## 0. 本次升级包含什么（V4.1.0 → V4.1.1）

| 版本 | 主要内容 | 动数据结构? | 需更新数据? |
|---|---|---|---|
| V4.1.1 | ① `/projects` 导出改为**按当前可见列**导出；② `/insight/costdetail` 项目成本明细表**显示修复**（不再只露约一行） | 否（纯前端） | 否 |

具体：
- **`/projects` 导出优化**：导出弹窗勾选「项目清单」时，导出的 xlsx **列 = 你当前页面选列后可见的那些列**（顺序、内容都与页面一致），不再是以前写死的固定 8 列。你在「选列」里增减/排序了哪些列，导出就跟着变。其它 sheet（项目标签/跟进记录/回款节点/里程碑）不变。
- **`/insight/costdetail` 成本明细表显示修复**：去掉了明细表的固定表头（sticky-header），修掉它在长页面下把表体高度压缩、导致**只显示约一行**的问题。修复后每页正常显示 20 行（可在下方分页选 20/50/100）。上方「L4 部门成本情况汇总」表与其它页不受影响。

> **本次无「头号注意」级别的坑**：纯前端、无数据迁移、无新页面、无新账号授权项。换 dist、重启 app、强刷即可。

## 1. 最小更新包内容（`pmplatform-update-V4.1.1.zip`）

解压出顶层目录 `pmplatform-update-V4.1.1/`，含：

- 全部后端根 `*.py`（与 V4.1.0 一致，本版未改后端）
- `frontend/dist/`（**已用 `--base=/pm/` 构建**）—— 本版真正的更新内容
- `requirements.txt`、`deploy/升级手册-V4.1.1.md`、`pmisdata/` 脚本
- **不含** `client/`、`data/`、`input/`、`tests/`、`docs/`、`node_modules/`——升级不覆盖服务器现有数据/账号/输入

## 2. 升级步骤（Ubuntu 服务器，约 2 分钟）

```bash
# 0) 上传 pmplatform-update-V4.1.1.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V4.1.1.zip

# 1) 备份 dist(回滚用) —— 纯前端升级,只需备份 dist
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS

# 2) 覆盖前端 dist
cd /tmp/pmplatform-update-V4.1.1
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) (可选)覆盖后端 .py(与现网一致,覆盖无害)
sudo cp -f *.py /opt/pmplatform/

# 4) 重启 app,让其加载新 dist
sudo systemctl restart pmplatform      # 若服务名不同,按现网实际名称
sudo systemctl status pmplatform --no-pager | head -5

# 5) 浏览器强刷(Ctrl+F5),确认左上角版本号 V4.1.1
```

> 注：若 nginx 缓存了静态资源，强刷仍是旧版时 `sudo systemctl reload nginx` 后再刷。
> **本版不需要进 /data 点「更新数据」。**

## 3. 升级后验证清单

### 版本与基础

- [ ] 左上角版本号显示 **V4.1.1**

### `/projects` 导出

- [ ] 进「选列」增删几列（或调整顺序）→ 点「导出」勾「项目清单」→ 导出的 xlsx「项目清单」sheet 的列与你**当前可见列一致**（不再是固定几列）

### `/insight/costdetail` 成本明细

- [ ] 「项目成本明细」表（页面下方那张）**正常显示多行**（每页 20 行），不再只露约一行
- [ ] 分页可切 20/50/100；排序、列头筛选、导出 Excel 仍正常

> **如果这张表升级后仍只显示很少行**，看表格下方的「**共 X 条**」判断：
> - 显示「共 600+ 条」→ 若还只露一行请反馈（应已修复）；
> - 显示「**共 1 条**」或个位数 → 这不是本次修的显示问题，而是 `/data` 里开了全局「**按标签排除**」且排除标签命中了几乎所有项目 → 去 `/data` 关掉该开关（或调整排除标签）即可，与本次升级无关。

## 4. 已知的行为变化（不是缺陷）

1. **成本明细表表头不再固定**（V4.1.1）：为修显示问题去掉了 sticky-header，上下滚动时表头会随页面滚动——与页面上其它表格（如上方 L4 汇总表）一致。
2. **导出列跟随选列**（V4.1.1）：导出「项目清单」的列 = 当前可见列。若你隐藏了某列，导出就没有该列；想导出更多列，先在「选列」里勾上。

## 5. 回滚（回到 V4.1.0）

纯前端回滚，只需还原 dist：

```bash
TS=<你升级时用的时间戳>
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-$TS /opt/pmplatform/frontend/dist
sudo systemctl restart pmplatform
```

> 本版未新增/迁移任何 `data/` 文件，未改数据结构，`input/` 一律不受影响。回滚后无需点「更新数据」。
