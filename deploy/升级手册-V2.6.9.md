# 升级手册 —— V2.6.9（代码质量修复 3 批累积；从 V2.6.6 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.6.6** 升级到 **V2.6.9**。
> 本包是「**后端代码 + 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。

> 随附最小更新包：`pmplatform-update-V2.6.9.zip`。
> 若现网早于 V2.6.6，请先用对应旧手册升到 V2.6.6，再用本包。

## 0. 本次升级包含什么（V2.6.6 → V2.6.9，代码质量修复 3 批累积）

整仓五维度质量审查后分 3 批修复，本包一次性累积交付（V2.6.7 + V2.6.8 + V2.6.9）：

### 批1（V2.6.7）数据正确性与口径（会算错数/显示错的缺陷）

- **导出「合同金额(万)」修 1 万倍错**：/projects 导出的 xlsx 该列此前写的是元值（表头却是「万」），改为正确的万元值。
- **动态页「新超支项目」修恒为 0**：项目动态的「超支出现/解除」事件此前因字段改名遗留恒不触发（真实数据 36 个超支项目此前一个都不报），已修复。
- **服务组回款达成排名口径统一**：/payment 服务组达成率此前用日期区间口径，与页面顶部完成率（全时）不一致、把 2025 年回款项目低估，已改为全站统一全时口径。
- **首页「年度已回」补齐无阶段项目流水**、**日历今日到期节点上午 8 点后不再消失**（时区修复）、以及导出单位、范围筛选单位、里程碑月末等一批口径/边界修复。

### 批2（V2.6.8）后端健壮性 + 内部重构

- **写盘原子化 + 并发事务锁**：跟进/标签/进展等本地数据的保存改为原子写（崩溃/并发不再留半截坏文件），并发编辑不再丢更新。
- **「更新数据」/「下载」防重复触发**、**上传/请求体大小上限**、**进度推送客户端断开不再报错**等健壮性加固。
- **收款台账/PMIS 解析失败计数告警**（在数据质量里可见格式漂移）。
- **内部重构 + 死代码清理**（对使用者无感，仅提升可维护性）。

### 批3a（V2.6.9）前端重构 + 修复

- **换账号后跟进数据不再残留上一账号**（风险跟进/回款重点跟进两页此前漏了清理，已闭合）。
- **商机/进度编辑保存失败现在弹错提示**（此前静默）。
- 5 个跟进页内部重构（对使用者无感，交互不变）+ 一批交互修复。

### 批3b（V2.6.9）设计一致性清理

- **状态色文字对比度改善**（里程碑「延期」等黄/绿小字此前偏浅、现为深色可读）、**日历「到期」表头改淡底深字**、**可点卡片支持键盘 Tab/Enter**、**金额列数字等宽对齐**、**停服/回滚/清空/删记录的确认弹窗改为与主题一致的样式**（不再是浏览器原生弹窗）等。

> **【头号注意 1】本次升级无需点「更新数据」。** 未改数据契约结构（schema）、未改数据来源。批2 虽在「数据质量」里新增了「解析失败计数」这一告警字段，但属宽松扩展、不需要重算数据。现有 data/ 与账号原样保留。
>
> **【头号注意 2】⚠️ 本次后端代码改动较多（批2 重写了 `server.py` 的跟进类接口与数据管线的若干模块），升级后必须重启后端进程（systemd）才生效。** 这与 V2.6.6（纯前端、可不重启）不同——**本次务必重启**。
>
> **【头号注意 3】无新增/删除页面、无新增 pageKey、无需授权账号、无新增依赖。**

## 2. 升级步骤（Ubuntu 服务器，约 2 分钟）

```bash
# 0) 上传 pmplatform-update-V2.6.9.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.6.9.zip

# 1) 备份现网(至少备份代码与 dist,data/input 不动本包也不覆盖)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py + 前端 dist(不覆盖 data/ 与 input/)
cd /tmp/pmplatform-update-V2.6.9
sudo cp -f *.py /opt/pmplatform/
sudo rm -f /opt/pmplatform/compare_payment_sources.py   # 批2 已删除退役脚本(若存在)
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f pmisdata/*.py pmisdata/*.sh /opt/pmplatform/pmisdata/ 2>/dev/null || true

# 3) 【必做】重启后端(本次后端改动多,必须重启)
sudo systemctl restart pmplatform    # 服务名以现网为准

# 4) 浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号为 V2.6.9
```

## 4. 回滚

若升级后异常：
```bash
sudo systemctl stop pmplatform
sudo rm -rf /opt/pmplatform
sudo mv /opt/pmplatform.bak-<时间戳> /opt/pmplatform
sudo systemctl start pmplatform
```
