## 0. 本次升级包含什么（V2.5.3 → V2.5.5，三项·均纯前端）

### 来自 V2.5.5（本次核心）
1. **图表悬停（hover）卡顿修复**：`/insight/costdetail`「超支项目分布」等**柱状图**鼠标悬停时卡顿。根因＝ECharts 默认用 HTML 元素画 tooltip，鼠标每次移动都要读该 DOM 的尺寸来定位，触发**整页强制重排（reflow）**；页面下方 DOM 越重（如成本明细大表）越卡（折线图因悬停触发少故不明显）。本次将全站图表 tooltip 改为**在画布上直接渲染（richText）**，无 DOM、无重排。实测（模拟慢机 6 倍降速）柱状图悬停掉帧减少 **79–91%**，折线图不受影响。

### 来自 V2.5.4（本包一并含入，此前未部署）
2. **`/insight/costdetail` 成本分析页改造**：
   - 顶部四张卡改为「成本统计项目数 / 未超支 / 总成本超支数（含"超支大于5000"子项）/ 交付成本超支数」，口径与 `/projects` 一致；点卡就地筛选下方明细。
   - 明细表新增「**交付成本状态**」列（依「交付部门剩余 / 交付外包剩余」两列正负判定：未超支 / 交付预算超支 / 交付外包超支 / 原厂外包均超支）。
   - **售前服务类**项目的「总预算 / 已核算 / 剩余预算」三列改为**回退取原项目**（修此前显 0 的取数异常）。
   - 标题去括号、超支分布图拉长、「L4 部门成本情况汇总」支持选列。
3. **全站列表列筛选交互修复（9 个页面）**：列头多选筛选改为**级联**（筛了某列，其它列的可选项随之收窄，只显示当前结果里存在的值）；点筛选按钮「▼」**不再误触发列排序**。覆盖 `/projects`、`/projects/closed`、`/opportunities`、`/insight/milestone`、`/insight/costdetail`、`/projects/key`、`/opportunities/key`、`/projects/temp`、`/risk`。

> **【头号注意】本次升级无需点「更新数据」。** 两版均为**纯前端**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何口径来源。前端复用现有 `data/analysis_data.json` 即可。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面/无新页面访问 key、未改 `auth.py`（超管不变）。**
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.5.5.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.5.5.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本包与 V2.5.3 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.5.5/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本包核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.5.5/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本包无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧样式：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。
