# 升级手册 —— V2.10.0（首页门户 / 快捷入口 Launchpad；从 V2.9.0 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.9.0** 升级到 **V2.10.0**。
> 本包**非纯前端**——除换 `frontend/dist` 外，**后端新增 `portal.py`、并改动 `server.py`/`audit.py`，须覆盖后端 `*.py` 并重启后端**。
> 随附最小更新包：`pmplatform-update-V2.10.0.zip`。
> 若现网早于 V2.9.0，请先用对应旧手册升到 V2.9.0，再用本包。

---

## 0. 本次升级包含什么（V2.9.0 → V2.10.0）

新增「首页门户 / 快捷入口（Launchpad）」：由超级管理员统一维护的一组跳转/下载入口，按账号分发，首页顶部只读展示。

- **首页顶部新增「快捷入口」区**（项目总览 `/` 顶部，KPI 卡之上）：按分组展示 url 跳转项与文件下载项；置顶项汇入顶部区；可折叠（折叠状态按登录账号记忆）。当前账号无可见项则整块不显示（超管则显「＋ 配置」入口）。
- **超管在 `/data` 配置**（数据管理页 →「首页门户 / 快捷入口」折叠区，**仅超级管理员可见可配**）：
  - 新建 / 编辑 / 删除门户项，两类：**url 跳转**（http/https，新标签打开）、**文件下载**（超管上传文件到服务器，用户点击即下载）。
  - **分组**（自定义组名）、组 / 组内项**上移 / 下移**排序、**置顶 ★**。
  - 每项设**可见账号范围**：全部账号，或勾选具体账号。
- **后端**：新增 `portal.py`（校验 / 可见性 / 文件名消毒 / 下载头）与 4 个接口（`GET/POST /api/portal/config`、`POST /api/portal/upload`、`GET /api/portal/download`）；配置存 `data/portal_links.json`，上传文件存 `data/portal_files/`（首次访问自动创建，**无需手工建**）；门户增删改进审计日志。

> **【头号注意 1】⚠ 非纯前端，必须覆盖后端 `*.py` 并重启后端。** 本包新增 `portal.py`、改动 `server.py`/`audit.py`；只换 dist 不重启后端，门户接口会 404。
>
> **【头号注意 2】无需点「更新数据」。** 门户不触碰分析数据管线，`data/analysis_data.json` 与业务口径均不变。`data/portal_links.json`、`data/portal_files/` 首次使用时自动创建。
>
> **【头号注意 3】无新增页面 / 路由 / pageKey、无需授权变更、无新增依赖。** 首页在 `/`、配置在 `/data`（均现成）；文件下载与可见性在后端强制。从在线基线 V2.9.0 增量。

## 1. 最小更新包内容（`pmplatform-update-V2.10.0.zip`）

解压出顶层目录 `pmplatform-update-V2.10.0/`，含：
- `frontend/dist/`（**已用 `--base=/pm/` 构建**；本期改动：首页门户区 + 新增 Portal 组件/计算 + /data 配置卡 + `version.ts`）。
- 全部后端根 `*.py`（**含新增 `portal.py`，及改动的 `server.py`/`audit.py`**——本次实质后端改动）、`requirements.txt`、`deploy/升级手册-V2.10.0.md`、`pmisdata/` 脚本。
- **不含** `client/`、`data/`、`input/`、`tests/`、`docs/`、`node_modules/`（升级不覆盖服务器现有数据 / 账号 / 输入）。

## 2. 升级步骤（Ubuntu 服务器，约 2 分钟）

```bash
# 0) 上传 pmplatform-update-V2.10.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.10.0.zip
cd /tmp/pmplatform-update-V2.10.0

# 1) 备份现网 dist 与后端 .py(可选但建议)
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS
sudo cp -r /opt/pmplatform /opt/pmplatform.py.bak-$TS  # 或仅备份根 *.py

# 2) 覆盖前端 dist(核心)
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) 覆盖后端 *.py(本次必须——含新增 portal.py 与改动的 server.py/audit.py)
sudo cp -f *.py /opt/pmplatform/

# 4) 重启后端(本次必须——新增接口 + 审计动作)
sudo systemctl restart pmplatform      # 服务名以实际为准(如 pmplatform.service)

# 5) 浏览器强制刷新(Ctrl+F5),确认右上角版本号 V2.10.0。
```

> 注：`data/portal_links.json`、`data/portal_files/` 由后端首次读写时自动创建，属 `/opt/pmplatform/data/` 下，随现有 data 目录权限即可（app 进程可写）。若你的部署对 `data/` 有特殊只读挂载，请确保该目录可写。
> 注：若 nginx 缓存了静态资源，强刷仍是旧版时，`sudo systemctl reload nginx` 或清 nginx 缓存后再强刷。

## 3. 升级后验证清单

- [ ] 右上角 / 关于处版本号显示 **V2.10.0**。
- [ ] 用**超级管理员**登录：`/data` →「首页门户 / 快捷入口」折叠区可见（普通管理员**看不到**该折叠区）。
- [ ] 在该区新建一个 **url 项**（https://… ，可见范围「全部账号」）+ 一个**文件项**（上传一个文件，可见范围勾选某个普通账号）→ 保存成功。
- [ ] 回首页：顶部「快捷入口」显示这些项；点 url 项**新标签打开**；点文件项**浏览器下载**（文件名正确，含中文名亦可）。
- [ ] 用该普通账号登录：首页**只见其可见项**（url 项可见、文件项按你勾选可见）；越权文件项直连 `/api/portal/download?id=<越权id>` 返回 **404**。
- [ ] 无 console 报错。

---

## 4. 回滚

回滚需还原 dist **与**后端 `*.py`，并重启：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
# 还原后端 .py(从备份目录覆盖回根 *.py),例如:
sudo cp -f /opt/pmplatform.py.bak-<时间戳>/*.py /opt/pmplatform/
sudo systemctl restart pmplatform
```
> 回滚后 `data/portal_links.json` / `data/portal_files/` 可保留（旧版后端不读它们，无副作用）；如需彻底清除，手工删除这两个路径即可。业务数据与账号不受影响。
