# 升级手册 —— V2.8.2（跟进进展就地富文本内联编辑；从 V2.8.1 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.8.1** 升级到 **V2.8.2**。
> 本包是「**纯前端**」增量包——**只换 `frontend/dist`**，后端代码与本次一字未改。
> 随附最小更新包：`pmplatform-update-V2.8.2.zip`。
> 若现网早于 V2.8.1，请先用对应旧手册升到 V2.8.1，再用本包。

---

## 0. 本次升级包含什么（V2.8.1 → V2.8.2）

**把 5 个跟进页的「点击填写 / 编辑进展」从弹窗改为就地在单元格内的富文本内联编辑，后端零改动。**

- **就地富文本编辑**：`/projects/key`（重点项目）、`/opportunities/key`（重点商机）、`/risk`（风险跟进）、`/projects/temp`（临时重点）、`/payment/key`（回款重点）五页——点进展 / 跟进单元格，**不再弹窗**，直接在格子里出现编辑框，工具栏支持：**加粗 / 下划线 / 删除线 / 斜体 / 字体颜色 / 清除格式**。`保存`/`取消`按钮（`Esc`=取消、`Ctrl+Enter`=保存）。历史快照态只读。
- **展示**：保存后单元格以富文本样式展示（含时间前缀）；旧的纯文本记录照常显示、不破版。
- **随之的小变化**：风险跟进、回款重点两页的「跟进动作 / rev结论」两列**不再可点表头排序**（自由文本排序意义低）；风险跟进页「rev结论」列**不再可筛选**；这些字段**导出 Excel 时为纯文字（去掉格式标签）**。

> **【头号注意 1】✅ 纯前端，无需重启后端、无需点「更新数据」。** 本包只更新 `frontend/dist`，后端 `*.py` 与 V2.8.1 完全一致（包内附带的 .py 覆盖上去是同内容、无副作用）。数据、审计、账号一律不动。
>
> **【头号注意 2】✅ 无新增页面 / 路由 / pageKey、无需授权、无新增依赖。** 富文本编辑器用浏览器原生能力实现，未引入任何第三方库。从在线基线 V2.8.1 增量。
>
> **【头号注意 3】历史已填的进展**（升级前用旧弹窗保存的纯文本）**照常显示、可继续编辑**，编辑后按富文本保存；不需要任何数据迁移。
>
> **【头号注意 4】安全**：编辑内容经严格白名单净化后才渲染（只保留加粗 / 下划线 / 删除线 / 斜体 / 颜色等格式标签，脚本 / 事件 / 外链一律过滤），不引入 XSS 风险。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.8.2.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.8.2.zip

# 1) 备份现网 dist(可选)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖前端 dist(核心)
cd /tmp/pmplatform-update-V2.8.2
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
#   (可选)后端 .py 与 V2.8.1 一致,覆盖与否均可:
#   sudo cp -f *.py /opt/pmplatform/

# 3) 无需重启后端(后端未变)、无需点「更新数据」。
#    浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号 V2.8.2、进展改为就地编辑。
```

> 注：若你的部署用 nginx 缓存了静态资源，强刷仍是旧版时，`sudo systemctl reload nginx` 或清 nginx 缓存后再强刷。

## 4. 回滚

纯前端，回滚极简：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
```
（无后端/数据改动，回滚只换回旧 dist、强刷即可。已保存的富文本进展是纯字符串、回滚后旧版按纯文本展示、不丢数据。）
