> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.4.0** 升级到 **V2.5.0**。

> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。

## 0. 本次升级包含什么（V2.5.0，整页重设计·项目总览首页 `/`）

- **首页「项目总览」深度重做**：由旧「6 张 KPI 卡 + 健康度卡 + 回款重点卡 + 底部三色块 + 右栏动态」改为**三段式**——
  - ① 全宽**项目体检带**：左＝项目健康度**分段条**（健康/关注/风险按占比着色 + 可点图例下钻）+ 在管/进行中/暂停；右＝**回款达成率环**（环形进度）+ 年度回款进度 / 本月待回 / 7 天临期。
  - ② **需要处理的异常**卡片网格：按严重度排列（成本超支 / 风险未闭环 / 回款延期 / 里程碑滞后…），每张卡可「查看清单」下钻、可「展开」就地看 Top 项目（「回款延期」展开显示待回金额 Top）。
  - ③ 右栏「项目动态」（不变）。
- **去掉冗余**：底部重复的「高风险/暂停/超支」三色块、与分段条重复的「健康度低」卡、与异常卡数字重叠的「四维异常行」、易混的 KPI「超支」数——信息均并入体检带或异常卡，未丢失。

> **【头号注意】本次升级无需点「更新数据」。** 本期为**纯前端**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何回款/客户/TOP1000 口径，前端复用现有 `data/analysis_data.json` 即可正确显示。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面/无新页面访问 key、未改 `auth.py`（超管不变）。**
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.5.0.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.5.0.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本期与 V2.4.0 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.5.0/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本期核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.5.0/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本期无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧首页：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。
