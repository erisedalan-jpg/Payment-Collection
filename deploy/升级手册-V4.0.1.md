# 升级手册 —— V4.0.1（/projects 日期列 + 标签筛选下沉；从 V4.0.0 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V4.0.0** 升级到 **V4.0.1**。
> 本包是「**非纯前端**」增量包——**换 dist + 覆盖后端 + 重启后端 + 点一次「更新数据」**。
> 随附最小更新包：`pmplatform-update-V4.0.1.zip`。
> 若现网早于 V4.0.0，请先用对应旧手册升到 V4.0.0，再用本包。

---

## 0. 本次升级包含什么（V4.0.0 → V4.0.1）

**四件事，都围绕「项目日期维度的可见性」与「筛选交互一致性」**：

1. `/projects/temp`（临时重点跟进）选列补上**立项日期** —— 修一个老缺陷：范围设置里早就能按立项日期筛，但选列里没有这一列
2. `/projects` 新增三个**默认隐藏**列：**原项目立项日期 / 计划终验时间 / 实际终验时间**
3. 后两列同步进 `/projects/temp` 的**范围设置**与**选列**；原项目立项日期同步进 `/project/:id` 的**原项目选项卡**
4. `/projects` 的**标签筛选**从表格上方的工具栏，下沉到表头「标签」列的筛选箭头

**三个新列的取值口径**

| 列 | 取值 |
|---|---|
| 原项目立项日期 | 售前服务类项目按映射取**原项目**的立项日期；非售前显示 `-` |
| 计划终验时间 | 售前服务类取「服务完成」的**计划**时间，其他项目取「终验」的**计划**时间 |
| 实际终验时间 | 售前服务类取「服务完成」的**实际**时间，其他项目取「终验」的**实际**时间 |

> **【头号注意 1】⚠ 本版升级后必须点一次「更新数据」。**
> `实际终验时间` 是本版新增的**管线回填字段**，不重跑管线则 `/projects` 与 `/projects/temp` 的
> 「实际终验时间」列**全部为空**。
> **V4.0.0 那版是「无需点更新数据」的，容易惯性跳过这一步。**
>
> **【头号注意 2】⚠ 本版需要重启后端。** 改动了 `milestones.py` / `preprocess_data.py` / `schema.py`。
>
> **【头号注意 3】⚠ `/projects` 的标签筛选换位置了。** 原来在表格上方工具栏的标签下拉**已移除**，
> 改为表头「标签」列的筛选箭头（`▾`）。标签列已默认放出，升级后直接可见、不用去选列里勾。
>
> **【头号注意 4】✅ 无新增页面 / 路由 / pageKey / 授权。** 现有账号授权一律不用动。

## 2. 升级步骤（Ubuntu 服务器，约 3 分钟）

```bash
# 0) 上传 pmplatform-update-V4.0.1.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V4.0.1.zip

# 1) 备份(建议:本次动后端,回滚要用)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$(date +%Y%m%d-%H%M)
sudo mkdir -p /opt/pmplatform/bak-$(date +%Y%m%d-%H%M) && sudo cp /opt/pmplatform/*.py /opt/pmplatform/bak-$(date +%Y%m%d-%H%M)/

# 2) 覆盖后端 .py
cd /tmp/pmplatform-update-V4.0.1
sudo cp -f *.py /opt/pmplatform/

# 3) 覆盖前端 dist
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 4) 重启后端(必须)
sudo systemctl restart pmplatform      # 若服务名不同,按现网实际名称
sudo systemctl status pmplatform --no-pager | head -5

# 5) 浏览器强刷(Ctrl+F5),确认左上角版本号 V4.0.1
# 6) ★ 进 /data 点一次「更新数据」,等 SSE 跑完 —— 这一步不能省
```

> 注：若 nginx 缓存了静态资源，强刷仍是旧版时 `sudo systemctl reload nginx` 后再刷。

## 4. 已知的行为变化（不是缺陷）

1. **标签筛选的选项来源变了**：原工具栏下拉列出**所有已定义标签**；表头筛选只列出**当前数据里实际用过的**标签。没有任何项目使用的标签不再出现——选了它本来也是空结果。
2. **`/projects` 默认列数 +1**：为了让标签筛选入口可见，标签列已加入默认可见列。若嫌挤，可在「选列」里取消勾选——但那样标签筛选入口也会一并隐藏。
3. **只有 `/projects` 一页的标签筛选下沉了**。`/costdetail`、`/insight`、`/milestone`、`/payment/nodes`、`/payment/projects` 这 5 个页面的标签筛选**仍在表格上方**，位置不变。
4. **倚天合规页的「问题」列筛选多了「空值」选项**：这是第 4 项改动连带的（两处共用同一套列筛选逻辑），可用来筛出「无合规问题」的记录。

## 5. 回滚

本次动了后端，回滚需同时还原 `.py` 与 `dist` 并重启：

```bash
sudo cp -f /opt/pmplatform/bak-<时间戳>/*.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
sudo systemctl restart pmplatform
```

> **回滚后无需再点「更新数据」**：`progress.实际终验时间` 会作为多余字段留在 `data/analysis_data.json` 里，
> 旧版代码不消费它，无害。`data/*.json` 与 `input/` 一律不受影响，回滚无数据风险。
>
> `data/temp_followup.json` 里已保存的范围条件本版**未改字段 key**，升级与回滚都不会失效。
