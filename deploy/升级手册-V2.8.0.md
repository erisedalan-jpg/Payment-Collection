# 升级手册 —— V2.8.0（/data 数据管理页重设计；从 V2.7.1 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.7.1** 升级到 **V2.8.0**。
> 本包是「**纯前端**」增量包——**只换 `frontend/dist`**，后端代码与本次一字未改。
> 随附最小更新包：`pmplatform-update-V2.8.0.zip`。
> 若现网早于 V2.7.1，请先用对应旧手册升到 V2.7.1，再用本包。

---

## 0. 本次升级包含什么（V2.7.1 → V2.8.0）

**只重设计了 `/data`「数据管理」页的界面与交互，功能一个不删、后端零改动。**

- **状态总览条（新）**：页面顶部一排状态——上次处理时间 / PMIS 时间 / 本机代理（已连接·绿 / 未运行·黄）/ PMIS cookie（有效含 SESSION 预览·绿 / 未设置·黄）/ 倚天 cookie（已存 / -），一眼看清数据与代理状态。
- **主区「获取与更新数据」（工作流导向）**：
  - ① 获取数据：**在线获取（PMIS）**（获取本机 cookie → 下载数据）与 **上传文件**（PMIS 九表 / 项目域文件清单 + 上传）两条路径并列，窄屏自动堆叠。
  - ② 更新看板：「更新数据（重新处理）」主按钮，作为两路径的汇聚终点。
- **「更多」折叠**：手动粘贴 cookie（高级兜底）与「获取本机倚天 cookie」收进折叠，主区更清爽。
- **维护区（折叠，默认收起）**：项目标签、人工数据导入/回滚、数据历史/回滚、清空数据（危险，两步确认）——四个折叠面板，点开才展开，减少误操作、缩短主路径。

> **【头号注意 1】✅ 纯前端，无需重启后端、无需点「更新数据」。** 本包只更新 `frontend/dist`，后端 `*.py` 与 V2.7.1 完全一致（包内附带的 .py 覆盖上去是同内容、无副作用）。
>
> **【头号注意 2】✅ 功能一个不删。** 所有操作（PMIS/项目域上传、获取本机 PMIS/倚天 cookie、手动粘贴 cookie、下载数据、更新数据、项目标签管理、按标签排除、人工导入/回滚、历史回滚/撤销、清空数据）全部保留，只是重新组织了位置：cookie 相关在主区「在线获取」与「更多」折叠；标签/导入/历史/清空在下方「维护」折叠区。
>
> **【头号注意 3】本机 cookie 代理不受影响。** V2.7.1 装的无界面代理照常用；`/data` 页的「获取本机 PMIS/倚天 cookie」按钮位置变了但功能不变，状态条会显示「本机代理：已连接/未运行」。
>
> **【头号注意 4】无新增页面/路由/pageKey、无需授权、无新增依赖。** 从在线基线 V2.7.1 增量。

- **不含** `client/`、`data/`、`input/`、`tests/`、`docs/`、`node_modules/`。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.8.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.8.0.zip

# 1) 备份现网 dist(可选)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖前端 dist(核心)
cd /tmp/pmplatform-update-V2.8.0
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
#   (可选)后端 .py 与 V2.7.1 一致,覆盖与否均可:
#   sudo cp -f *.py /opt/pmplatform/

# 3) 无需重启后端(后端未变)、无需点「更新数据」。
#    浏览器强制刷新(Ctrl+F5)访问 /data,确认右上角版本号 V2.8.0、新界面生效。
```

> 注：若你的部署用 nginx 缓存了静态资源，强刷仍是旧版时，`sudo systemctl reload nginx` 或清 nginx 缓存后再强刷。

## 3. 升级后验证清单

- [ ] 右上角/关于处版本号显示 **V2.8.0**。
- [ ] `/data` 页顶部出现**状态总览条**（处理/PMIS 时间 + 本机代理 + PMIS cookie + 倚天状态）。
- [ ] 主区「获取与更新数据」：① 获取（在线获取（PMIS）/ 上传文件 两栏）② 更新看板（更新数据按钮）。
- [ ] 「更多」折叠点开有手动粘贴 cookie 与「获取本机倚天 cookie」。
- [ ] 下方「维护」四折叠面板：项目标签 / 人工数据导入·回滚 / 数据历史·回滚 / 清空数据（点开才展开）。
- [ ] 走一遍主流程：点「获取本机 PMIS cookie 并推送」→「下载数据」→「更新数据」，与升级前效果一致。
- [ ] 展开「项目标签」折叠，标签增改禁用 + 按标签排除照常可用；展开「数据历史」可回滚；「清空数据」仍两步确认。

---

## 4. 回滚

纯前端，回滚极简：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
```
（无后端/数据改动，回滚只换回旧 dist、强刷即可。）
