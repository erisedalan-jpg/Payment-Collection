# 升级手册 —— V4.4.3（镜像修「更新数据」并发冲突同款静默闪退；**纯前端修复**）

> 适用：已用 /pm 部署在 Ubuntu 服务器（nginx:80 → app 127.0.0.1:8080，systemd，`/opt/pmplatform`，venv `/opt/pmplatform/.venv`）。
> 本包仍是**全量后端快照**（glob 打包全部根 `*.py`），但**后端逻辑较 V4.4.1/V4.4.2 无任何改动**——本次修复只在前端 `dist`。
> 随附最小更新包：`pmplatform-update-V4.4.3.zip`。

---

## 0. 本次修什么

**症状**：两个管理员并发点「更新数据」时，**后点击的人**看不到任何进度、面板一闪就消失（"闪退"）——与 V4.4.2 修的「下载数据」并发闪退是**同一个病**，只是发生在「更新数据」按钮上。

- **根因**：更新运行槽 `reprocess_state` 同样是**服务器全局单例、所有管理员共用一个**。当 A 的更新还没跑完（或有下载/回滚在跑），B 再点，后端抢不到槽 → 走 `_json_response` 回一段**普通 JSON**（`Content-Type: application/json`），**不是** SSE 流。而前端 `useReprocess` 旧代码一律按 SSE 逐行找 `data:` 帧 → 一帧都解析不出 → 提示为空、进度面板闪现即消失。
- **修法（前端）**：`useReprocess` fetch 后按 `content-type` 分流——是 `application/json`（忙分支）就直接解析并显示冲突提示（"已有数据更新正在进行，请等其完成后再试（当前：…）" / 透传"其他数据操作进行中，请稍后再更新"），且不误触发"更新完成"回调；`DataView` 进度块补 `repMessage` 显示条件，让忙/错误消息（含此前同样看不见的 `更新失败`/`无响应体`）真正可见。

> **至此**：`download`（V4.4.2）+ `reprocess`（本版）两条 SSE 忙分支的「静默吞信息」缺陷全部清除。

| 内容 | 需更新数据? |
|---|---|
| 「更新数据」并发冲突前端提示修复 | **否**（纯前端，不碰数据管线） |

> ### 头号注意
>
> **① 本次无需点「更新数据」。** 只换 `dist`（+可选覆盖后端，逻辑没变）、重启 app、浏览器 `Ctrl+F5` 硬刷即可。
>
> **② 换 dist 后务必硬刷。** 新前端资源带新哈希文件名，`Ctrl+F5` 清缓存才看得到 V4.4.3。
>
> **③ 既有账号/数据零影响。** 本包不含 `data/`/`input/`；账号、`yitian_store.json`、`holidays.csv`、PMIS 输入均不动。

## 1. 升级步骤（Ubuntu，约 3 分钟）

```bash
# 0) 上传并解压
cd /tmp && unzip -o pmplatform-update-V4.4.3.zip

# 1) 备份 dist
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS

# 2) 覆盖前端 dist（本次核心）
cd /tmp/pmplatform-update-V4.4.3
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) （可选）全量覆盖后端 .py —— 逻辑与 V4.4.1/V4.4.2 一致,覆盖只为保持文件同版
sudo cp -f *.py /opt/pmplatform/

# 4) 重启后端
sudo systemctl restart pmplatform
sudo systemctl status pmplatform --no-pager | head -5

# 5) 浏览器 Ctrl+F5 硬刷 /pm,左上角版本号应为 V4.4.3
```

## 2. 升级后验证清单

- [ ] 左上角版本号 **V4.4.3**
- [ ] 单人点「更新数据」：进度条与提示正常滚动（未受影响）
- [ ] **并发验证**（本次修复点）：A 起一个「更新数据」未跑完时，B 再点「更新数据」→ B 应看到明确提示 **"已有数据更新正在进行，请等其完成后再试（当前：…）"**，而**不再是一闪即消**
- [ ] 一并回归 V4.4.2：A 下载中 B 再点「下载数据」→ 提示"已有下载正在进行…"（同款已修）
- [ ] 现有账号登录、各页数据范围与升级前一致

---

## 3. 回滚

```bash
TS=<你升级时用的时间戳>
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-$TS /opt/pmplatform/frontend/dist
sudo systemctl restart pmplatform
```

> 回滚只换回 dist（若第 3 步覆盖了 .py，另从 `bak-$TS` 还原）；`data/`/`input/` 不受影响。
