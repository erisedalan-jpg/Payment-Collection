# 升级手册 —— V3.1.0（概算工具 /budget；从 V3.0.0 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V3.0.0** 升级到 **V3.1.0**。
> 随附最小更新包：`pmplatform-update-V3.1.0.zip`。
> 若现网早于 V3.0.0，请先用对应旧手册升到 V3.0.0，再用本包。

---

## 0. 本次升级包含什么（V3.0.0 → V3.1.0）

新增**概算工具**：把原本发给个人、双击打开的单体 HTML 报价工具（`CostBudgetEstimate.html`，3729 行，价格全硬编码、存档在浏览器里、依赖外网 CDN）**完全重构**进本平台，成为 `/budget` 页面。

- **新增 1 个页面**（侧栏「工具选项」分区，位于「数据治理」下方、「关于产品」上方）：
  `/budget` **概算工具** —— 填项目信息 + 产品实施人天 + 项目经理人天 + 其他服务人天 + 差旅天数 → 实时算出人工成本、直接成本、总成本、**成本比例**、CRM 审批建议、销售下单物料清单 → 存档 / 导出 Excel 拿去 CRM 上单。
- **报价存档上服务端**（原来存在各人浏览器里，换台电脑就没了、清缓存就全丢）：按账号隔离 —— 普通管理员只见自己的报价；**超管可切「查看全部账号」**。
- **费率全部超管可配**：`/budget` 页内「费率与目录配置」抽屉（**仅超管**），五个页签 —— 价格与阈值 / 产品目录 / 服务目录 / 物料 / 项目经理阶段。汇率、人天单价、住宿差补标准、销售物料单价、成本比例阈值、19 个产品、8 项服务，**改完立即生效、无需点「更新数据」**。
- **导出 8 个 sheet 的 Excel**（原工具只导 6 个，漏了拿去 CRM 上单最要紧的两个）：项目基本信息 / 成本比例 / 产品实施 / 项目经理 / 其他服务 / 直接成本 / **CRM审批建议** / **销售下单建议**。
- **无外网依赖**：原工具从 jsdelivr CDN 加载 exceljs + file-saver，内网必然加载失败；现在用平台已内置的 xlsx。

> **【头号注意 1】⚠ 成本比例的口径修正了，会影响审批结果。**
> 原工具的页面文案写的是「3%＜**销售下单金额**/项目金额＜15%」，但它的代码分子用的是**未含税总成本**，**漏乘了 `(1 + 毛利率)`** —— 文案对、代码错。V3.1.0 按文案把代码改对了：
> ```
> 成本比例 = 总成本 × (1 + 毛利率) ÷ (项目金额万元 × 10000)
> ```
> **后果（都是预期的，不是 bug）**：
> - 同一份报价的比例比原工具**高约 13%**（毛利率选 6% 时高 6%）
> - **毛利率下拉从此会影响成本比例**（原工具里它只影响下单金额）
> - **旧口径落在 13.28%~15% 之间的报价，新口径会变成「比例偏高」并强制要求填写异常说明**
>
> 上线后如果有人问「怎么成本比例变高了 / 怎么突然要我填异常原因了」，答案就是这一条。
>
> **【头号注意 2】⚠ 非纯前端，必须覆盖后端 `*.py` 并重启后端。** 本包新增 2 个后端模块（`budget_config.py` / `budget_store.py`），并改动 `server.py` / `audit.py`。只换 dist 不重启后端，`/api/budget/*` 全部 404，页面白板。
>
> **【头号注意 3】⚠ 必须给账号授权新页面。** 新增 pageKey：**`budget`**（显示名「概算工具」）。**超管自动可见**；普通管理员需在「账号管理」里勾选。不授权的账号看不到侧栏里的「概算工具」（这是预期）。
>
> **【头号注意 4】不需要点「更新数据」。** 概算工具**不进数据管线**，与 `analysis_data.json` 完全解耦。费率配置与报价存档都是独立文件，后端首次读写时自动创建。

## 2. 升级步骤（Ubuntu 服务器，约 3 分钟）

```bash
# 0) 上传 pmplatform-update-V3.1.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V3.1.0.zip
cd /tmp/pmplatform-update-V3.1.0

# 1) 备份现网(可选但建议)
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$TS

# 2) 覆盖前端 dist
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) 覆盖后端 *.py(本次必须 —— 含 2 个新模块)
sudo cp -f *.py /opt/pmplatform/

# 4) 重启后端(本次必须)
sudo systemctl restart pmplatform      # 服务名以实际为准

# 5) 浏览器强制刷新(Ctrl+F5),确认右上角版本号 V3.1.0
```

**然后在页面上做一件事：**

**授权页面** —— 超管 →「账号管理」→ 给需要用概算工具的普通管理员勾选「概算工具」。

> 注：`data/budget_config.json`（费率配置）与 `data/budget_estimates.json`（报价存档）由后端**首次读写时自动创建**，无需手工建。首次进 `/budget` 时费率是默认值（与原工具现值完全一致），可直接开始报价。

## 3. 日常使用要点

**填报价**：进 `/budget` → 填项目基本信息（9 项全必填）→ 添加产品（选完自动带出标准人天与设备系数）→ **把人天手动填进四个格子**（技服一类/二类、外包一类/二类）→ 填项目经理与其他服务人天 → 填差旅 → 结果区实时算出。

> **「合计参考人天」只是参考值，不参与任何金额计算** —— 金额只认你手填的那四个格子。这和原工具一致。

**保存语义（与原工具不同）**：
- **「保存」= 覆盖当前这条**（原工具是无条件新增，改个错字都多一条）
- **「另存为新报价」= 新建一条**

**费率改了，历史报价不会被改写**：每条存档都冻结了保存当时的完整费率表。改完费率后再打开旧报价，顶部会出现「本报价基于保存时的费率表」横幅，金额仍按当时的费率算；想按新费率重算，点横幅上的「按最新费率重算」再保存即可。**报价是要拿去 CRM 上单的对外产物，必须可复现 —— 这是有意为之。**

**导出 Excel**：8 个 sheet。若某一段没填（比如没加任何「其他服务」），那一张 sheet 不会出现在文件里 —— 这是正常的。

## 5. 回滚

回滚需还原 dist **与**后端 `*.py`，并重启：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /opt/pmplatform.bak-<时间戳>/frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f /opt/pmplatform.bak-<时间戳>/*.py /opt/pmplatform/
sudo systemctl restart pmplatform
```
> 回滚后 `data/budget_config.json` 与 `data/budget_estimates.json` 可保留（旧版后端不读它们，无副作用）；如需彻底清除，手工删除即可。**业务数据、账号、倚天工时、门户配置均不受影响** —— 概算域与主域完全解耦，`analysis_data.json` 的产出与口径本次零改动。
