# 升级手册 —— V2.8.4（跟进列宽×2 + /risk 行下钻；从 V2.8.3 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.8.3** 升级到 **V2.8.4**。

## 0. 本次升级包含什么（V2.8.3 → V2.8.4）

- **跟进「点击填写」列变宽一倍**：/projects/key、/projects/temp、/opportunities/key、/risk、/payment/key 五页的两个富文本进展/跟进列，列宽 240 → 480（更宽、更好写好读；仍自动换行、不截断）。

> **【头号注意 2】无新增页面 / 路由 / pageKey、无需授权、无新增依赖。** 从在线基线 V2.8.3 增量。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.8.4.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.8.4.zip

# 1) 备份现网 dist(可选)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖前端 dist(核心)
cd /tmp/pmplatform-update-V2.8.4
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
#   (可选)后端 .py 与 V2.8.3 一致,覆盖与否均可:
#   sudo cp -f *.py /opt/pmplatform/

# 3) 无需重启后端(后端未变)、无需点「更新数据」。
#    浏览器强制刷新(Ctrl+F5),确认右上角版本号 V2.8.4。
```

> 注：若你的部署用 nginx 缓存了静态资源，强刷仍是旧版时，`sudo systemctl reload nginx` 或清 nginx 缓存后再强刷。

## 4. 回滚

纯前端，回滚极简：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
```
（无后端/数据改动，回滚只换回旧 dist、强刷即可。）
