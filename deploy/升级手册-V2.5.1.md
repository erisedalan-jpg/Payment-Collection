## 0. 本次升级包含什么（V2.5.1，子页面局部调整·`/insight/costdetail` 成本分析「项目成本明细」表）

仅改造**成本分析页**（`/pm/insight/costdetail`）下方「项目成本明细（按 L4 组织排序）」表格，五项：

1. **删两列**：`L3部门`、`L3-1部门` 两列及工具栏对应两个筛选下拉。
2. **金额去 ¥**：`项目金额`、`总预算(元)`、`已核算(元)`、`剩余预算(元)` 四列去掉 `¥` 符号，仅显示千分位数字（如 `890,000` / `107,169.74`）；剩余预算保留 <0 红 / ≥0 绿染色。
3. **加两列**：新增 `交付部门剩余(元)`、`交付外包剩余(元)`，单位元，数值**与项目详情页 `/project/:id` 同源**（取自 `delivery_analysis.csv` 的「交付部门人工成本 / 交付外包服务成本」剩余预算）。
4. **全列排序 + 列头多选筛选**：明细表 12 个数据列均可点表头排序（跨全部页排序，非仅当前页），并在表头挂多选筛选下拉（弹层内可搜索、可全选/反选）。
5. **保留**：编号/名称关键词搜索、导出 Excel（导出列同步为新列集，含两个交付剩余列）。

> 配套：共享组件 `DataTable.vue` 新增一个**可选**能力（`externalSort`，跨页排序 + 排序事件），默认关闭、对其余所有用到该组件的页面**行为不变**。

> **【头号注意】本次升级无需点「更新数据」。** 本期为**纯前端**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何回款/成本/客户/TOP1000 口径；新增的「交付部门/外包剩余」字段早已存在于现有 `data/analysis_data.json`，仅在前端视图层取用与重排。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面/无新页面访问 key、未改 `auth.py`（超管不变）。**
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.5.1.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.5.1.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本期与 V2.5.0 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.5.1/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本期核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.5.1/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本期无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧表格：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。
