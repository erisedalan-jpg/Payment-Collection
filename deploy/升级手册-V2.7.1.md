# 升级手册 —— V2.7.1（本机 cookie 代理 + /data 网页驱动取 cookie；从 V2.7.0 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.7.0** 升级到 **V2.7.1**。
> 本包是「**后端代码 + 前端 dist**」的增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.7.1.zip`。
> 若现网早于 V2.7.0，请先用对应旧手册升到 V2.7.0，再用本包。

---

## 0. 本次升级包含什么（V2.7.0 → V2.7.1）

把 `client/` 桌面工具的「取 cookie」能力集成进 `/data` 网页：**网页点按钮 → 本机常驻小代理静默取 cookie → 网页传服务器**，不必再开独立桌面 App 逐步操作。

- **`/data` 页新增**：PMIS 区加「获取本机 PMIS cookie 并推送」按钮 + 「本机代理：已连接/未运行」状态；新增「倚天 Cookie（取到备用）」卡片，加「获取本机倚天 cookie 并存储」按钮。
- **服务端新增**：超管专属 `GET/POST /api/yitian/cookie`（把倚天 cookie 存 `data/yitian_config.json` 备用；倚天数据功能后续开发，本期只「取+存」）。倚天 cookie 写入纳入 V2.7.0 审计（动作「更新倚天 Cookie」）。
- **PMIS 全链路复用、脚本零改动**：取到的 cookie 经现有 `/api/pmis/cookie` 推送、现有「下载数据」跑现有 pipeline，`pmisdata/*` 一字未改。

> **【头号注意 1】⚠️ 本机代理要在管理员机器上单独装一次（本包不含）。** 浏览器自身取不到零信任 cookie，网页按钮背后需要本机跑一个无界面小代理替它取。见下方「第 3 节 本机代理安装」。**不装代理，网页的「获取本机 cookie」按钮无法工作**（会显示「本机代理：未运行」）。
>
> **【头号注意 2】⚠️ 须重启后端。** 本包更新了 `server.py` 并新增 `yitian_config.py`，覆盖后必须重启 systemd 服务。
>
> **【头号注意 3】✅ 不需点「更新数据」、无需改 nginx。** 本期与 `data/analysis_data.json` 无关；真实 IP 沿用现网已配的 `X-Forwarded-For`。
>
> **【头号注意 4】前提：开平台网页的机器 = 装零信任、能取 cookie 的机器（同一台）。** 网页才能经 `127.0.0.1` 调到本机代理。
>
> **【头号注意 5】无新增页面/路由/pageKey、无需授权账号、无新增服务端依赖。** 本包从在线基线 V2.7.0 增量。

## 2. 服务端升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.7.1.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.7.1.zip

# 1) 备份现网(至少备份代码与 dist;data/input 不动、本包也不覆盖)
sudo cp -r /opt/pmplatform /opt/pmplatform.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖后端 .py + 前端 dist(不覆盖 data/ 与 input/)
cd /tmp/pmplatform-update-V2.7.1
sudo cp -f *.py /opt/pmplatform/
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
sudo cp -f pmisdata/*.py pmisdata/*.sh /opt/pmplatform/pmisdata/ 2>/dev/null || true

# 3) 重启后端(必须,server.py 有改动 + 新增 yitian_config.py)
sudo systemctl restart pmplatform    # 服务名以现网为准

# 4) 浏览器强制刷新(Ctrl+F5)访问,确认右上角版本号为 V2.7.1
# 5) 无需点「更新数据」。
```

## 3. 本机代理安装（在管理员的 Windows 零信任机器上，装一次）

本机代理源码在仓库 `client/` 下（`cookie_agent.py` + `cookie_core.py`）。装法二选一：

**方式 A：直接跑 Python（该机器有 Python + requests）**
```
pip install requests
python client\cookie_agent.py
```
看到「[cookie-agent] 监听 127.0.0.1:8765 ...」即成功。常驻这个窗口即可。

**方式 B：打成 exe（免装 Python，推荐分发）**
```
pip install pyinstaller requests
pyinstaller --onefile --name cookie_agent client\cookie_agent.py
```
把 `dist\cookie_agent.exe` 拷到目标机器双击运行。

**开机自启（推荐）**：`Win+R` → 输入 `shell:startup` → 把 `cookie_agent.py`（或 exe）的**快捷方式**放进打开的「启动」文件夹。之后每次开机代理自动常驻。

**配置（可选）**：在代理同目录放 `agent_config.json` 可覆盖端口/白名单，例如：
```json
{ "port": 8765, "allowed_origins": ["http://10.248.105.95"] }
```
缺省已含平台 IP，一般无需配。

**安全说明**：代理只监听 `127.0.0.1`（外部访问不到）、只接受来自平台源 `Origin` 的请求、并校验 `Host` 为 `127.0.0.1/localhost`（防 DNS rebinding）；只取 cookie、不碰你的平台账号密码。

## 4. 升级后验证清单

- [ ] 右上角版本号显示 **V2.7.1**。
- [ ] 本机代理已启动（命令行/托盘可见），本机零信任已登录 PMIS。
- [ ] `/data` 页「数据下载/更新」卡片：出现「获取本机 PMIS cookie 并推送」按钮，状态显示「本机代理：已连接」。
- [ ] 点该按钮 → 提示「已获取并推送 PMIS cookie（N 项）」，下方 SESSION 预览刷新为「刚刚」。
- [ ] 接着点「下载数据」→ 正常跑 PMIS 下载流水线（与升级前一致）。
- [ ] `/data` 页出现「倚天 Cookie（取到备用）」卡片；点「获取本机倚天 cookie 并存储」→ 提示「已获取并存储倚天 cookie（N 项，备用）」。
- [ ] 未启动代理时点按钮 → 提示「本机代理未运行或无法连接」，不报红崩溃。
- [ ] PMIS 未登录（cookie 无 SESSION）时点按钮 → 提示「未检测到 PMIS 登录态」，不推送。
- [ ] 审计（账号管理页→审计日志）能看到「更新倚天 Cookie」记录。

## 5. 回滚

若升级后异常：
```bash
sudo systemctl stop pmplatform
sudo rm -rf /opt/pmplatform
sudo mv /opt/pmplatform.bak-<时间戳> /opt/pmplatform
sudo systemctl start pmplatform
```
（本包不改数据；本机代理在管理员机器上，单独停掉即可。）
