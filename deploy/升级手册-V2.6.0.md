# 升级手册 —— V2.6.0（回款域重构 P1；纯前端增量；从 V2.5.9 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.5.9** 升级到 **V2.6.0**。
> 本包是「**纯代码 + /pm 前端 dist**」的轻量增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.6.0.zip`。
> **单版增量**：在线基线 V2.5.9 直接升到 V2.6.0。若现网早于 V2.5.9，请先按对应手册升到 V2.5.9，再用本包升 V2.6.0。**本版纯前端。**

---

## 0. 本次升级包含什么（V2.5.9 → V2.6.0，回款域重构 P1·纯前端）

**背景**：用户发现「在建项目总数 638」与「回款各页看到的 557」不一致。查明：在建主域（PMIS 在建 ∩ 组织架构交付三部）共 **638** 个项目，其中只有 **557** 个挂了收款阶段节点（`collection_stages.csv` 导出的收款阶段台账），另 **81 个项目没有收款阶段节点**（并非数据错误，而是这些项目在 PMIS 收款阶段导出中没有节点）。旧版回款页对这两个口径处理不一致，本次统一并把 81 个「无回款阶段」项目**显式列出来**。

**本次改进（回款域第 1 期，共 4 期，均为 V2.6.X）**：

1. **回款总览（`/payment`）重做**：
   - 「项目数」卡改为显示**在建主域整体数量（638）**，并在下方注明「**N 个项目无回款阶段**」（可点击跳到「在建项目」清单）。
   - 「延期项目数」卡可点击下钻（同首页，跳到在建项目并按「回款延期」筛选）；「回款节点数」卡可点击跳到「回款节点」页。
   - **删除**「待回款金额」「服务组达成排名」两张卡（信息已在其它页/卡覆盖）。
   - 「回款数据（按 L4 服务组）」表**改名为「回款数据」并增大填充页面**。
   - **新增「无回款阶段数据项目」清单**：列出上述 81 个无收款阶段节点的项目（项目编号 / 名称 / 经理 / L4 组 / 合同额），**行可点击进入项目详情、支持导出 Excel**。

2. **回款项目（`/payment/projects`）表格升级**：对齐「在建项目」页的能力——**列排序、列头多选筛选、选列与列排序、导出、按标签筛选**；**新增「项目级别」列**；**去掉「来源」列**。点某行的**下钻抽屉整体加宽**（更好展示）；若该项目**没有收款阶段数据**（如 QAGD-SS-202508149002），抽屉显示「**该项目无回款阶段数据**」并提供「**查看完整详情**」按钮跳到项目详情页。

3. **回款节点（`/payment/nodes`）精简增强**：去掉上方「维度」切换与分组表，保留 5 张统计卡；主表**新增「项目经理」「L4 组」两列**，并加**排序、列头筛选、按标签筛选、导出**。

4. **删除三个回款子页**：「**回款进度**」（/payment/plan）、「**风险项目**」（/payment/risk）、「**回款台账**」（/ledger）。这三页从侧栏移除；旧网址仍可访问（自动**跳转到「回款总览」`/payment`**，不会 404）。

> **【新功能说明·按标签筛选（含「无标签」）】**
> 「回款项目 / 回款节点」两页新增一个**标签筛选**下拉，选项 = `无标签` + 你维护的各个标签，**可多选、并集展示**：选「无标签」= 只看没打任何标签的项目；选若干标签 = 看含这些标签的项目；可混选。此筛选**每页独立**，与顶部「按标签排除」（影响全站统计口径的那个）是两套不同机制、互不影响。

> **【行为变化·需知悉】**
> - 回款总览「项目数」由原来的「有回款活动数（557）」改为「**整体在建数（638）**」，差额 81 即「无回款阶段」项目，已在同页清单列出。
> - 「回款进度 / 风险项目 / 回款台账」三页已下线；收藏的旧链接会自动跳到「回款总览」。
> - 回款项目/节点下钻抽屉变宽了（同一抽屉也用于「回款多维分析」「回款日历」的下钻，那两处一并变宽）。

> **【头号注意】本次升级无需点「更新数据」。** 全部为**纯前端**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何数据来源。前端复用现有 `data/analysis_data.json` 即可。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面。** 本版**删除**了 3 个页面访问权限项（payment-plan / payment-risk / ledger），无需任何授权操作——原来给账号勾过这三项的，升级后自动失效、无副作用。
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.6.0.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.6.0.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本包与 V2.5.9 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.6.0/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本包核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.6.0/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本包无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧样式/旧行为：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。

## 4. 回滚

```bash
sudo systemctl stop pmplatform
# 还原代码与 dist
cd /opt/pmplatform && sudo tar xzf /backup/pmplatform-code-<时间>.tgz
sudo chown -R pmapp:pmapp /opt/pmplatform
sudo systemctl start pmplatform
```

> 本包纯前端、未改 schema/数据/本地 JSON 结构，回滚后**无需**点「更新数据」，数据与账号不受影响。回滚后被删的三页（回款进度/风险项目/回款台账）会重新出现。
