# 升级手册 —— V2.5.9（下钻返回保持视图状态；纯前端增量；从 V2.5.8 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的 **V2.5.8** 升级到 **V2.5.9**。
> 本包是「**纯代码 + /pm 前端 dist**」的轻量增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.5.9.zip`。
> **单版增量**：在线基线 V2.5.8 直接升到 V2.5.9。若现网早于 V2.5.8，请先按对应手册升到 V2.5.8，再用本包升 V2.5.9。**本版纯前端。**

## 0. 本次升级包含什么（V2.5.8 → V2.5.9，交互优化·纯前端）

**背景**：此前在列表页设置了**筛选条件**（列头筛选、搜索、售前/超支/标签等），点某行「下钻」进入项目详情，看完**返回**列表时，筛选条件默认被清空、还得重新设一遍；排序、翻到的页码、滚动位置也都丢失。

**本次改进（不改变任何数据与口径，只改「返回时是否保留视图状态」）**：
1. **下钻返回 = 完整恢复**：从详情/下钻页**返回**列表时，之前的**筛选 + 排序 + 分页 + 滚动位置**原样恢复，回到你离开时的样子——不必重设。
2. **菜单进入 = 恢复默认**：从**左侧菜单**重新点进该页时，仍然是干净的默认状态（符合「从菜单=重新开始」的直觉）。
3. 覆盖页面：**在建项目、成本分析、已关闭项目、重点项目进展、临时重点跟进、里程碑管理**六个「有筛选 + 点行下钻」的列表页。（商机清单、风险跟进等点行是弹抽屉、不跳页，本就不丢状态，不受影响。）

> **【行为变化·需知悉】**
> - 「下钻→返回」现在会**保留**你的筛选/排序/滚动（此前是清空）。若想要干净的页面，点左侧菜单重新进入即可（或用页面上的「清除所有筛选/重置」按钮）。
> - 「重点项目进展、临时重点跟进」两页此前「从菜单再次进入仍保留上次列筛选」，本次统一为「**菜单进入即重置、下钻返回才保留**」，与其它页一致。
> - 换账号登录后，一切恢复默认（不会带入上一账号的筛选视图）。

> **【头号注意】本次升级无需点「更新数据」。** 全部为**纯前端**改动：未改数据契约（schema）、未改数据处理（preprocess）、未改任何数据来源。前端复用现有 `data/analysis_data.json` 即可。
> **无新增 Python 依赖、无 systemd/nginx 变更、无新数据源文件、无新后端端点、无新页面/无新页面访问 key、未改 `auth.py`（超管不变）。**
> 因此升级动作 = **替换 .py + 前端 dist + 重启**，无需任何数据/授权/依赖操作。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.5.9.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.5.9.zip').extractall('.')"

# (4) 覆盖后端代码（*.py）（本包与 V2.5.8 等价，照覆盖即可）
sudo cp /tmp/pmplatform-update-V2.5.9/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）—— 本包核心变更
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.5.9/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

#   pmisdata：本版本未改下载脚本。若服务器已有 pmisdata/ 且 cookie 仍有效，跳过。

# (6) 归属仍归 pmapp（data/ 须 pmapp 可写）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (7) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

> **本包无需点「更新数据」、无需 pip 安装、无需改 systemd/nginx、无需给任何账号授权。** 起服务后直接验证即可。
> 若浏览器仍显示旧样式/旧行为：强制刷新（Ctrl+F5）清前端缓存即可（dist 文件名带 hash，正常无需）。

```bash
sudo systemctl start pmplatform
```
