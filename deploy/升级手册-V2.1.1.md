# 升级手册 —— V2.1.1（服务器端 PMIS 下载 + board 排名图随排序换口径）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`，运行在 `.venv`）的旧版本升级到 **V2.1.1**。
> 本包是「**纯代码 + /pm 前端 dist + pmisdata 下载脚本**」的轻量增量包，**不含 data/ 与 input/**，不动服务器现有数据与账号。
> 配套部署原理见 `deploy/部署-Ubuntu24-端口80-pm.md`（首次安装看那份；本份只讲**升级**）。
> 随附最小更新包：`pmplatform-update-V2.1.1.zip`。
> 现网若早于 V2.1.0，本包代码为**全量覆盖所有 .py + dist**（升到 V2.1.1 无遗漏），但被跳过版本的**首用/数据步骤**仍需照对应手册补做（如 V2.1.0 的「临时跟进范围设置」、V2.0.0 的商机 seed）。

## 0. 本次升级包含什么

V2.1.1 是**子页面/局部（Z 级）**版本。相对 V2.1.0，本轮两项：

- **新增「服务器端 PMIS 下载」**：`/data`（数据管理）页在「更新数据」按钮**左侧**新增「**下载数据**」按钮。点击后由**服务器后台**跑 `pmisdata/run_pmis_pipeline.sh`（备份现有 → 从 PMIS 在线下载 9 表 + 损益/回款/预算 → 覆盖到服务器 `input/`、`input/pmis/`），进度经 SSE 实时回传页面。**只下载、不自动「更新数据」**——下载完成后再点「更新数据」重算看板才生效。
  - **抓取需有效 PMIS 会话 cookie**，三条送达路径（任选其一）：
    1. **网页粘贴框（推荐、通用）**：在能访问 PMIS 的机器上取到 cookie 串后，粘进 `/data` 的「PMIS Cookie」框 → 点「下载数据」即把 cookie 存到服务器 `pmisdata/config.json` 再下载。
    2. **服务器本机直跑**：在服务器上跑 `pmisdata/update_cookie.py`（需图形界面/Playwright），直接写本机 `pmisdata/config.json`。
    3. **`update_cookie.py --server`**：在抓 cookie 的机器上 `python update_cookie.py --server http://<服务器IP>/pm --account <超管账号>`，自动登录 pmplatform 并把 cookie 推到服务器。
  - **下载/cookie 全部超管专属**（普通管理员看不到「下载数据」按钮，且后端 `/api/pmis/*` 对非超管 403）。
- **`/insight/board` 排名图随排序换口径**：单维「排名」模式下，切换「排序」（项目数/合同金额/完成率/延期节点）后，下方柱/折/饼图**随选定指标整体换口径**（单系列、按金额/计数/比率自适应）；完成率为比率，饼图降级为提示。

> 后端新增纯模块 `pmis_config.py`（cookie 校验/写入）+ 3 个超管端点（`GET/POST /api/pmis/cookie`、`GET /api/pmis/download` SSE）+ 下载与「更新数据」/历史回滚互斥。`/insight/board` 为纯前端。
> **新增运行依赖**：服务器跑下载需 `requests` + `openpyxl`（见 §2 步骤 6）。
> 全部细节见仓库 `PROGRESS.md` 版本史。

## 2. 升级步骤（Ubuntu /pm systemd 部署）

以下命令以部署在 `/opt/pmplatform`、运行在 `.venv`、服务用户 `pmapp` 为例。**务必先备份**。

```bash
# (1) 备份数据 + 现有代码（用于回滚）
sudo tar czf /backup/pmplatform-data-$(date +%F-%H%M).tgz -C /opt/pmplatform data
sudo tar czf /backup/pmplatform-code-$(date +%F-%H%M).tgz -C /opt/pmplatform \
  $(cd /opt/pmplatform && ls *.py) frontend/dist
# 若服务器已有 pmisdata/（含已配好的 cookie），一并备份以免被包内 config.json 覆盖：
[ -d /opt/pmplatform/pmisdata ] && sudo tar czf /backup/pmplatform-pmisdata-$(date +%F-%H%M).tgz -C /opt/pmplatform pmisdata

# (2) 停服务
sudo systemctl stop pmplatform

# (3) 放更新包并解压（U 盘/scp 拷到服务器，例如 /tmp）
cd /tmp
unzip pmplatform-update-V2.1.1.zip      # 或 python3 -c "import zipfile;zipfile.ZipFile('pmplatform-update-V2.1.1.zip').extractall('.')"

# (4) 覆盖后端代码（*.py，含新增 pmis_config.py）
sudo cp /tmp/pmplatform-update-V2.1.1/*.py /opt/pmplatform/

# (5) 替换前端 dist（先删后拷，避免残留旧 chunk）
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /tmp/pmplatform-update-V2.1.1/frontend/dist /opt/pmplatform/frontend/dist
# 校验是 /pm 构建（应有输出 /pm/assets/...）：
grep -o '/pm/assets[^"]*' /opt/pmplatform/frontend/dist/index.html | head -1

# (6) 部署 pmisdata/（下载脚本）
#   首次升级到 V2.1.1：整套拷入。
sudo cp -rn /tmp/pmplatform-update-V2.1.1/pmisdata /opt/pmplatform/
#   若服务器原已有 pmisdata/ 且 config.json 里 cookie 仍有效：只更新脚本、保留你的 config.json：
#   sudo cp /tmp/pmplatform-update-V2.1.1/pmisdata/*.py /tmp/pmplatform-update-V2.1.1/pmisdata/*.sh /opt/pmplatform/pmisdata/

# (7) 归属仍归 pmapp（data/ 与 pmisdata/ 须 pmapp 可写——下载要写备份/输出）
sudo chown -R pmapp:pmapp /opt/pmplatform

# (8) 起服务
sudo systemctl start pmplatform
sudo systemctl status pmplatform        # 期望 active (running)
```

### 关键依赖（下载功能必需，否则点「下载数据」会失败）

下载流水线由 `run_pmis_pipeline.sh` 内部以 **`python3`（系统解释器）** 跑 `fetch_*.py`（service 虽运行在 `.venv`，但子进程调裸 `python3` 走 PATH，解析到**系统 python3**，非 venv）。因此 **系统 `python3` 必须装 `requests` + `openpyxl`**：

```bash
# 在线（若服务器可访问公网 pip 源）：
sudo python3 -m pip install requests openpyxl
# 离线（本环境不可联网时）：在联网机 `python3 -m pip download requests openpyxl -d wheels`，
#   拷 wheels/ 到服务器后：
#   sudo python3 -m pip install --no-index --find-links=/path/to/wheels requests openpyxl
# 校验（必须打印 OK）：
python3 -c "import requests, openpyxl; print('download deps OK')"
```

> 服务器还需有 `bash`（Ubuntu 自带）。**服务器须能访问 `pmis.qianxin-inc.cn`**（内网直达），否则下载会失败。
> 抓 cookie 的机器另需 `playwright` + chromium（仅 `update_cookie.py` 用）；**服务器端不需要 playwright**。

## 3. 升级后必做/按需操作

1. **账号不变（不新增超管）**：本版本**未改动** `auth.py`。超管仍固定为 `admin`、`wangxutong`、`zhangyingzhe` 三个。`/api/pmis/*` 仅读已有 `isSuper` 判定。
2. **刷新 PMIS cookie（下载前必做一次）**：包内 `pmisdata/config.json` 自带的 cookie 多半已过期。三选一刷新：
   - 在能访问 PMIS 的机器上取 cookie 串 → 登录 `/pm` → 数据管理页「PMIS Cookie」框粘贴 → 点「下载数据」（粘贴的 cookie 会先存服务器再下载）；
   - 或在那台机器上 `python pmisdata/update_cookie.py --server http://<服务器IP>/pm --account <超管账号>`（按提示输密码）直推服务器；
   - 或在服务器本机（有图形界面时）跑 `update_cookie.py` 直写 `config.json`。
   - 页面「PMIS Cookie」框下方会显示「当前 SESSION xxxxxxxx · 更新于 …」，据此确认是否刷新成功。
3. **下载数据流程**：超管在 `/data` 点「下载数据」→ 看 SSE 进度（备份/3 步下载/拷贝）→ 提示「下载完成，请点更新数据生效」→ 再点「**更新数据**」重算看板。下载耗时较长（全量项目损益），期间进度条停在中段属正常。
4. **board 排名图**：随 dist 即时生效，无需额外操作。
5. **nginx/systemd 无变化**：`/pm` 的 nginx 配置已含 `proxy_buffering off`（SSE 实时），下载 SSE 复用之，无需改 nginx；systemd 单元无变化。

## 4. 上线验证清单

- [ ] `systemctl status pmplatform` = running；`curl -I http://<IP>/pm/` 返回 200。
- [ ] 浏览器开 `http://<IP>/pm` → 预制超管登录成功。
- [ ] `python3 -c "import requests, openpyxl"` 在服务器上不报错（下载依赖到位）。
- [ ] `/pm/data` 数据管理页：「数据下载 / 更新数据」卡里，「**下载数据**」按钮在「更新数据」**左侧**；有「PMIS Cookie」输入框与当前 SESSION 状态行。
- [ ] 安全：未登录 `curl http://<IP>/pm/api/pmis/cookie` 应 **401**；以**非超管**普通管理员打 `/pm/api/pmis/cookie`、`/pm/api/pmis/download` 应 **403**；普通管理员页面**看不到**「下载数据」按钮。
- [ ] （在能访问 PMIS 的机器上冒烟）刷新 cookie 后点「下载数据」：SSE 进度推进、结束提示「下载完成，请点更新数据生效」；服务器 `input/`、`input/pmis/` 文件「最近修改时间」已更新；再点「更新数据」看板刷新。失败时页面显「下载失败: …」可读原因。
- [ ] `/pm/insight/board` 单维「排名」：切「排序」到「合同金额/完成率/延期节点」，下方柱/折图随之换口径（标题如「合同金额排名」）；选「完成率」且图表类型含饼图时，饼图显「完成率为比率，不宜用饼图」提示。

# 如已备份 pmisdata 且想恢复旧 cookie/脚本：

## 6. 备注

- **cookie 是敏感的实时会话**，仅经已鉴权（超管）通道传输、落 `pmisdata/config.json`；页面/日志只显示 SESSION 前 8 位，不回显全 cookie。cookie 定期过期，过期后重走 §3-2 刷新即可，无安全顾虑。
- **下载会在 `pmisdata/` 下生成时间戳备份目录与 `pipeline_*.log`**，长期累积可定期清理（保留最近几次即可），不影响功能。
- **已知边界（PROGRESS Backlog）**：`handle_pmis_download` 的「忙检查→置位」非原子，理论上并发两次「下载数据」可能各起一次流水线同写 `input/`；因超管专属 + 每次下载带时间戳备份，实际风险低——**请勿同时多人/多标签页点「下载数据」**。
- 本次为**纯代码 + dist + pmisdata** 增量更新，未改 systemd/nginx 配置。
- 若你需要的是**全新安装整包**（非增量），在开发机执行 `python make_deploy_zip.py` 生成 `pmplatform-deploy-V2.1.1.zip`（含全部代码 + /pm dist + data/ + input/ + pmisdata/），按 `deploy/部署-Ubuntu24-端口80-pm.md` 全新部署。
- 端口 80 为明文 HTTP（仅可信内网）；不可信网络建议上 443 + TLS（见部署手册 §11）。
