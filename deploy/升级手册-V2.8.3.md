# 升级手册 —— V2.8.3（选列与排序按登录用户持久化；从 V2.8.2 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V2.8.2** 升级到 **V2.8.3**。
> 本包是「**纯前端**」增量包——**只换 `frontend/dist`**，后端代码与本次一字未改。
> 随附最小更新包：`pmplatform-update-V2.8.3.zip`。
> 若现网早于 V2.8.2，请先用对应旧手册升到 V2.8.2，再用本包。

---

## 0. 本次升级包含什么（V2.8.2 → V2.8.3）

**全站主要表格的「选列结果」与「排序结果」按登录用户记住，刷新 / 重开浏览器后保留、不再回默认；并修掉「同一浏览器多个账号互相覆盖选列」的问题。** 后端零改动。

- **选列按用户记住（11 张表）**：/projects、/projects/closed、/projects/key、/projects/temp、/payment/key、/payment/projects、/opportunities、/opportunities/key、/risk、成本分析 L4 汇总表、里程碑提醒——每个人自己勾选/排序/隐藏的列，下次登录还在，且**不同账号在同一台电脑互不干扰**。
- **排序按用户记住（13 张表）**：上述表 + 成本明细、回款节点等——点表头排的序，刷新后恢复（表头箭头也回来）。
- **存储在本机浏览器**：按「登录账号 + 表」分别存在浏览器 localStorage（换电脑 / 换浏览器不带走，属预期）。
- **列筛选不在此列**：列头的多选筛选仍是刷新清空（一贯行为，未改动）。

> **【头号注意 1】✅ 纯前端，无需重启后端、无需点「更新数据」。** 本包只更新 `frontend/dist`，后端 `*.py` 与 V2.8.2 完全一致（包内附带的 .py 覆盖上去是同内容、无副作用）。
>
> **【头号注意 2】老用户升级后「选列」会先回默认一次。** 因为存储 key 从「不含账号」改为「含账号」，旧的选列记录不迁移（避免把共享的旧记录错塞给某个账号）。**每个账号第一次进各页看到默认列，重新调一次后即按你自己的账号记住**，属一次性、正常现象。排序此前从不记住，升级后同理：排一次即记住。
>
> **【头号注意 3】无新增页面 / 路由 / pageKey、无需授权、无新增依赖。** 从在线基线 V2.8.2 增量。
>
> **【说明】** 本次覆盖全站主要数据表；/payment 总览里两张嵌入式聚合表（L4 汇总、无阶段项目）的排序持久化计划在下一小版（V2.8.4）补齐，不影响本次功能。

## 2. 升级步骤（Ubuntu 服务器，约 1 分钟）

```bash
# 0) 上传 pmplatform-update-V2.8.3.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V2.8.3.zip

# 1) 备份现网 dist(可选)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$(date +%Y%m%d-%H%M)

# 2) 覆盖前端 dist(核心)
cd /tmp/pmplatform-update-V2.8.3
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist
#   (可选)后端 .py 与 V2.8.2 一致,覆盖与否均可:
#   sudo cp -f *.py /opt/pmplatform/

# 3) 无需重启后端(后端未变)、无需点「更新数据」。
#    浏览器强制刷新(Ctrl+F5),确认右上角版本号 V2.8.3。
```

> 注：若你的部署用 nginx 缓存了静态资源，强刷仍是旧版时，`sudo systemctl reload nginx` 或清 nginx 缓存后再强刷。

## 3. 升级后验证清单

- [ ] 右上角/关于处版本号显示 **V2.8.3**。
- [ ] 进 /projects，用「选列」隐藏几列 / 调顺序，点某列表头排序 → **刷新（F5）后选列与排序都还在**。
- [ ] 换个账号登录（同一浏览器）→ 看到的是**该账号自己的**选列/排序（不被上一个账号影响）；切回原账号仍是原账号的。
- [ ] 成本分析、回款相关表同样验证排序刷新保留。
- [ ] 列头的多选**筛选**仍是刷新后清空（未纳入持久化，符合预期）。

---

## 4. 回滚

纯前端，回滚极简：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
```
（无后端/数据改动，回滚只换回旧 dist、强刷即可。持久化数据在各用户浏览器 localStorage、回滚后旧版忽略这些键、不报错、不丢业务数据。）
