# 升级手册 —— V4.1.0（倚天工时明细表 + 统计页下钻；**从 V4.0.5 单版升**）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V4.0.5** 升到 **V4.1.0**。
> 本包是「**纯前端**」增量包——**只换 dist + 重启 app + 浏览器强刷**。**不改后端数据结构、不需点「更新数据」、不需备份任何 data 文件。**
> 随附最小更新包：`pmplatform-update-V4.1.0.zip`。
> 若现网不是 V4.0.5，请先用对应旧手册升到 V4.0.5，再用本包。

---

## 0. 本次升级包含什么（V4.0.5 → V4.1.0）

| 版本 | 主要内容 | 动数据结构? | 需更新数据? |
|---|---|---|---|
| V4.1.0 | 倚天新增**工时明细页** `/yitian/detail`（逐条工时明细，形如项目清单）+ 五个统计页起加「明细」下钻入口 | 否（纯前端） | 否 |

具体：
- 新增 **`/yitian/detail` 工时明细页**：一行一条工时记录，选列 / 表头筛选 / 横向滚动 / 按登录用户持久化；顶部复用倚天工具条（日期区间 / 周模式 / L4 组织）+ 汇总条（总条数 / 总工时 / 合规·提示·问题三态计数）+「仅看异常」开关 + 导出 xlsx；含合规状态三态徽章列与问题原因列。数据全部由现有下发数据在前端还原，**不新增任何服务端数据文件、不改后端**。
- **统计页下钻**：倚天总览（组织表）、统计分析（员工表）、合规检查（问题表）、客户支持（TOP1000 表）各新增一个「明细」操作列，点击跳到工时明细页并按对应维度（组织 / 员工）自动筛选；从合规问题表下钻会自动开「仅看异常」。趋势页不加直接入口（经统计分析/合规检查两跳可达）。**各页原有的下钻去向一律不变。**

> ### 这一跳只有一件事要留意，其余都不用管：
>
> **【头号注意 1】⚠ 新增了一个普通页面 `/yitian/detail`，被限定页面访问权限的账号需要补授权。**
> 工时明细是一个**普通页面**（不是超管专属）。
> - 账号「可访问页面」是 **`*`（全部页面）** 的：**自动可见，无需任何操作。**
> - 账号被**限定了具体页面清单**的（例如只勾了某几个倚天页）：需要**超管**进「系统管理 → 账号管理」，给该账号在页面清单里**补勾「工时明细」**，它才会出现在其侧边栏、才允许访问。新页面已自动出现在账号编辑的可选页面列表里。
>
> **【头号注意 2】✅ 不需要点「更新数据」、不需要重启数据管线、不需要备份 data。**
> 本版纯前端，没有任何管线回填字段或数据结构迁移。换 dist、重启 app、强刷即可。
>
> **【头号注意 3】✅ 后端 `.py` 无实质改动。**
> 更新包为保持完整仍包含全部根 `.py`（与 V4.0.5 内容一致），覆盖无害。重启 app 只是为了让它干净加载新的前端静态资源。

## 1. 最小更新包内容（`pmplatform-update-V4.1.0.zip`）

解压出顶层目录 `pmplatform-update-V4.1.0/`，含：

## 2. 升级步骤（Ubuntu 服务器，约 2 分钟）

```bash
# 0) 上传 pmplatform-update-V4.1.0.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V4.1.0.zip

# 1) 备份 dist(回滚用) —— 纯前端升级,只需备份 dist,无需备份任何 data 文件
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS
# (可选)顺手备份后端,虽本版未改后端:
sudo mkdir -p /opt/pmplatform/bak-$TS && sudo cp /opt/pmplatform/*.py /opt/pmplatform/bak-$TS/

# 2) 覆盖前端 dist
cd /tmp/pmplatform-update-V4.1.0
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) (可选)覆盖后端 .py(与现网一致,覆盖无害;想保持文件一致性就做)
sudo cp -f *.py /opt/pmplatform/

# 4) 重启 app,让其加载新 dist
sudo systemctl restart pmplatform      # 若服务名不同,按现网实际名称
sudo systemctl status pmplatform --no-pager | head -5

# 5) 浏览器强刷(Ctrl+F5),确认左上角版本号 V4.1.0
```

> 注：若 nginx 缓存了静态资源，强刷仍是旧版时 `sudo systemctl reload nginx` 后再刷。
> **本版不需要进 /data 点「更新数据」。**

## 4. 已知的行为变化（不是缺陷）

1. **明细页「日期区间 / L4 组织」与倚天其他页共用同一套筛选状态**（同一个倚天工具条 store）：在明细页改了日期/L4，切到统计分析等页会沿用；这是倚天各页一致的既有设计。
2. **明细页不显示工作成果正文**：逐条明细只在「问题」行的「问题原因」单元格悬浮显示不超过 120 字的正文摘要；完整正文属隐私，从不下发、不导出。
3. **趋势页没有直接「明细」入口**：时间维度直钻逐条记录量大、价值低，经趋势页→统计分析/合规检查→明细两跳即可到达。

## 5. 回滚（回到 V4.0.5）

纯前端回滚，只需还原 dist：

```bash
TS=<你升级时用的时间戳>
sudo rm -rf /opt/pmplatform/frontend/dist
sudo mv /opt/pmplatform/frontend/dist.bak-$TS /opt/pmplatform/frontend/dist
# (若第 3 步覆盖过后端)还原后端,虽与现网一致:
# sudo cp -f /opt/pmplatform/bak-$TS/*.py /opt/pmplatform/
sudo systemctl restart pmplatform
```

> **回滚安全性说明**：本版未新增/迁移任何 `data/` 文件，未改数据结构，`input/` 一律不受影响。回滚后**无需点「更新数据」**。超管若已给某些账号补勾过「工时明细」页面权限，回滚后旧版没有该页面、该勾选项自然失效（无害，`canAccess` 只按 key 命中）。

## 6. 升级后浏览器端的一次性行为（不是异常）

- 首次进入「工时明细」页，选列/排序为默认状态（按登录账号存储）；调整后各账号各自记忆。换浏览器或清过浏览器数据会回到默认列，重新调整即可。
