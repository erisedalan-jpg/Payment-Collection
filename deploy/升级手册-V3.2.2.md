# 升级手册 —— V3.2.2（PMIS 立项日期接入管线 + 多处暴露；从 V3.2.1 升级）

> 适用：把**已用 /pm 部署在 Ubuntu 服务器**（nginx:80 → app 127.0.0.1:8080，systemd 托管，目录 `/opt/pmplatform`）的 **V3.2.1** 升级到 **V3.2.2**。
> 随附最小更新包：`pmplatform-update-V3.2.2.zip`。
> 若现网早于 V3.2.1，请先用对应旧手册升到 V3.2.1，再用本包。

---

## 0. 本次升级包含什么（V3.2.1 → V3.2.2）

把 PMIS 导出里本就存在、但此前从未进入系统的**「立项日期」**字段接入数据管线，并在多处展示与筛选：

- **① 数据管线接入**：后端从 PMIS 项目基础信息表提取「立项日期」（缺则回退项目状态信息表），写入项目数据。**这一步需要点一次「更新数据」才会生效。**
- **② `/projects`（在建项目清单）与 `/projects/key`（重点项目清单）**：各新增一列「立项日期」，**默认不显示**，可在「选择列」里打开；打开后支持**排序**和**列头筛选**（点列名旁 ▼ 多选）。
- **③ `/project/:id`（项目详情）**：在头部「编号」后面显示「立项日期」。
- **④ `/projects/temp`、`/payment/key`、`/risk` 三页的「范围设置」**：新增可按**「立项日期」区间**筛选（和现有「终验时间」同款：选「区间内 / 区间外」+ 起止日期选择器）。

> **【头号注意 1】本次是「非纯前端」升级——除换 `frontend/dist` 外，还要覆盖后端 `*.py`（改了 `pmis.py`、`schema.py`）、重启后端、并点一次「更新数据」。**
> **【头号注意 2】务必点一次「更新数据」。** 立项日期是在「更新数据」（重跑数据管线）时才写进 `analysis_data.json` 的。**不点的话**，前端各处的立项日期会全部显示「-」（列/详情/范围设置都取不到值）。数据源无需重新上传，用现有 `input/` 里的 PMIS 数据重跑即可。
> **【头号注意 3】务必强制刷新（Ctrl+F5）** 清掉旧 JS/CSS 缓存，否则新列 / 详情 / 范围设置字段可能加载不到。

**无新增页面 / 路由 / pageKey / 权限 / 第三方依赖。业务口径（回款达成率、成本等）零改动。**

**无新增第三方依赖。**

## 2. 升级步骤（Ubuntu 服务器，约 3 分钟）

```bash
# 0) 上传 pmplatform-update-V3.2.2.zip 到服务器任意临时目录并解压
cd /tmp && unzip -o pmplatform-update-V3.2.2.zip
cd /tmp/pmplatform-update-V3.2.2

# 1) 备份现网 dist 与后端(可选但建议)
TS=$(date +%Y%m%d-%H%M)
sudo cp -r /opt/pmplatform/frontend/dist /opt/pmplatform/frontend/dist.bak-$TS
sudo cp /opt/pmplatform/pmis.py    /opt/pmplatform/pmis.py.bak-$TS
sudo cp /opt/pmplatform/schema.py  /opt/pmplatform/schema.py.bak-$TS

# 2) 覆盖前端 dist
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r frontend/dist /opt/pmplatform/frontend/dist

# 3) 覆盖后端 *.py(本次改了 pmis.py / schema.py;整批覆盖无害,其余与 V3.2.1 相同)
sudo cp -f *.py /opt/pmplatform/

# 4) 重启后端(systemd 托管,服务名以现网为准)
sudo systemctl restart pmplatform    # 或你的实际服务名

# 5) 登录平台 → /data「数据管理」→ 点「更新数据」,等 SSE 进度跑完(把立项日期写进数据)

# 6) 浏览器强制刷新(Ctrl+F5),确认右上角版本号 V3.2.2
```

> 若走了反向代理静态缓存，第 2 步后清一下缓存。

## 3. 升级后验证清单

- [ ] 右上角版本号显示 **V3.2.2**
- [ ] **已点过「更新数据」且跑完**（否则下面立项日期全是「-」）
- [ ] `/projects`：点「选择列」→ 勾出「立项日期」→ 列出现且有值（形如 `2019-06-24`）；点该列表头可排序；点列名旁 ▼ 可多选筛选
- [ ] `/projects/key`：同样能在「选择列」里打开「立项日期」列并排序 / 筛选
- [ ] `/project/:id`（随便点开一个项目）：头部「编号」后面显示「立项日期」
- [ ] `/projects/temp`、`/payment/key`、`/risk` 的「范围设置」里，项目字段多了「立项日期」，选它出现「区间内 / 区间外」+ 起止日期选择器，能按区间筛
- [ ] 深色主题下上述列 / 详情 / 范围设置显示正常
- [ ] 原有功能（首页 / 回款达成率 / 商机 / 概算工具 / 门户 / 倚天各页）一切如常、数值未变

---

## 4. 回滚

还原 dist + 两个后端文件 + 重启（数据无需回滚，立项日期只是多出来的字段，不影响其它口径）：
```bash
sudo rm -rf /opt/pmplatform/frontend/dist
sudo cp -r /opt/pmplatform/frontend/dist.bak-<时间戳> /opt/pmplatform/frontend/dist
sudo cp /opt/pmplatform/pmis.py.bak-<时间戳>   /opt/pmplatform/pmis.py
sudo cp /opt/pmplatform/schema.py.bak-<时间戳> /opt/pmplatform/schema.py
sudo systemctl restart pmplatform
# 浏览器 Ctrl+F5
```
> 业务数据、账号、倚天工时、各类配置均不受影响 —— 本次仅新增一个只读字段，无口径 / 无破坏性数据改动。
