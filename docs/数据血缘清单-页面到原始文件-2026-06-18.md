## 五、回款完成率口径总览（V1.15.0 统一后）

> ⚠️ **全站主域口径**：Σ流水净额(paymentPmis.actualTotal，含负值/红冲，不取绝对值) ÷ Σ合同总额(paymentPmis.contract)；合同≤0→null(前端显"-")。
> 以下各页均使用此口径（除 /insight 例外）：

| 页面/组件 | 计算点 | 口径 |
|---|---|---|
| 首页 OverviewView | overview.ts `computeKpis` | 流水÷合同 |
| 项目清单 ProjectsView | projectList.ts → p.payment.paymentRatio | 流水÷合同(由9f payment_ratio_from_records 写入) |
| 回款总览 /payment DashMetrics | payDashboard.ts `payDashSummary`：rate=totalActual/totalContract | 流水÷合同 |
| 回款总览 OrgRanking | payDashboard.ts `payOrgRanking`：achievementRate=actualTotal/contractTotal | 流水÷合同 |
| 多维看板 /payment/board | paymentBoard.ts `buildGroup`：rate=actualSum/contractSum | 流水÷合同 |
| 回款项目 summaryByDim | paymentPmis.ts `summaryByDim`：rate=actualSum/contractSum | 流水÷合同 |
| 回款进度 progressBuckets | paymentPmis.ts `progressBuckets`：rate=actualSum/contractSum | 流水÷合同 |
| 日期区间版 paymentPmisInRange | paymentRange.ts：paymentRatio=actualTotal/contract | 流水÷合同 |
| 回款台账 LedgerView | ledger.ts `ledgerRows`：r=actualPayment/contract；`ledgerSummaryPmis` | 流水÷合同 |
| 项目详情六指标条 | ProjectDetailView → p.payment.paymentRatio | 流水÷合同 |
| **⚠️ /insight 项目分析(例外/技术债)** | projectPivot.ts `groupInsight`：paymentRatio=act/contractAmount(act=p.payment.actualTotal=节点Σ已收) | **节点已收÷PMIS客户合同(≠主域口径)** |

**注意**：`payment.actualTotal` = Σ收款阶段已收金额（节点级聚合，aggregate_payment_pmis）；`paymentPmis.actualTotal` = Σ流水净额（payment_records，二者不同，文档需区分）。

## 六、数据护栏备忘（子项目1 教训）

**改/删字段键时必须全仓 grep 旧键所有消费方**——schema 的 extra="allow" 让 typecheck 抓不到孤儿字段消费；自洽 fixture 让 vitest 抓不到孤儿前端消费方。子项目1 曾因此遗漏以下消费方，靠全分支评审+真实数据复核才发现：
- `deriveProgress` 仍读已删的 `paymentPmis.paymentRatio`
- `overview.ts` 仍读 `cost.超支`（旧动态字段名）
- `projectList.ts` 仍读 `cost.超支`
- `projectPivot.ts` 仍用 `签约形式` 维度

推荐流程：删/改字段 → `grep -r "旧键名" frontend/src/ backend/*.py` → 逐一确认/改掉 → vitest + pytest + 真实数据跑通。
