# 系统审计与 Ubuntu 部署建议（2026-06-20）

> 面向"部署到一台 Ubuntu 服务器、供多名管理人员通过内网并发访问"的目标，对当前系统做的整体审计。
> 审计以**实际代码**为准（CLAUDE.md/PROGRESS.md 的"技术债"段落多已过期，见 §6 文档漂移）。
> 本文是**评估与建议记录**，未改动任何业务代码。逐项落地须按 CLAUDE.md「一次只做一个功能/修复」分批执行。

## 0. 总体结论

- **鉴权地基是正确的**：口令 PBKDF2-SHA256 哈希（非明文）、会话 httpOnly cookie、`/admin` 与 `/data/analysis_data.json` 的授权**确实在后端兑现**（不是只靠前端隐藏），L4 数据隔离对主数据端点生效。记忆里"折中(后端切数据)"对 `analysis_data.json` 成立。
- **但"折中"只兑现一半**，且当前形态**尚不可直接暴露给多名管理人员**。四个部署前必须解决的硬缺口：
  1. **L4 隔离被原始静态文件整体绕过**（任意登录用户可直读 `/input/*.csv`、`/data/events.json` 拿全量未脱敏数据）。
  2. **所有写/运维端点只校验"已登录"、不校验"超管"**（任意普通账号能清空数据、停服、覆盖上传、回滚）。
  3. **绑定 `127.0.0.1` 与"内网多人访问"冲突**；且**纯 HTTP 无 TLS**（口令/数据明文传输）。
  4. **源码内置 3 组弱明文超管口令**，首启自动建号。
- **并发已不是瓶颈**：已是 `ThreadingHTTPServer`（CLAUDE.md §8"单线程阻塞全站"已过期）。
- **性能有显著优化空间**：`analysis_data.json` 实测 **~12MB**（非文档说的 2MB）全量未压缩拉取；前端**零代码分割**单 JS chunk 2.25MB。
- **数据层有一处口径技术债 + 一处覆盖率盲区**，以及一个**当下就要拍板的数据运维问题**（两份 collection_stages CSV 不一致）。

## 5. Ubuntu 部署落地方案（推荐架构）

**架构：nginx（TLS + 内网网卡）→ 反代 → app(127.0.0.1:8080，不改绑定) → systemd 托管**

这样应用层保持绑 loopback（最安全）、nginx 负责对内网暴露 + HTTPS + gzip + SSE 透传。

### 5.1 系统准备
```bash
sudo apt update && sudo apt install -y python3-venv nginx
sudo useradd -r -s /usr/sbin/nologin pmapp           # 专用低权用户
sudo timedatectl set-timezone Asia/Shanghai          # 固定时区(消除 D-5 部署侧风险)
# 代码放 /opt/pmplatform，数据目录 data/ 归 pmapp 可写
sudo mkdir -p /opt/pmplatform && sudo chown -R pmapp:pmapp /opt/pmplatform
```

### 5.2 Python 环境（依赖瘦身）
常驻 web 服务**只需 Python 3.8+ 标准库**即可起；数据更新功能才需三方包；**不使用 WPS 在线同步即可不装 playwright/chromium（最重依赖）**。
```bash
sudo -u pmapp python3 -m venv /opt/pmplatform/.venv
sudo -u pmapp /opt/pmplatform/.venv/bin/pip install pydantic openpyxl
# 仅当需要 WPS 在线同步再装: pip install playwright && playwright install chromium
```

### 5.3 systemd 服务单元 `/etc/systemd/system/pmplatform.service`
```ini
[Unit]
Description=Project Management Platform
After=network.target

[Service]
Type=simple
User=pmapp
WorkingDirectory=/opt/pmplatform
Environment=TZ=Asia/Shanghai
# 建议给 server.py 加一个环境开关跳过自动开浏览器(见 §6 改造点),无开关时无头机上 webbrowser.open 也只是静默失败
ExecStart=/opt/pmplatform/.venv/bin/python server.py
Restart=on-failure
RestartSec=3

[Install]
WantedBy=multi-user.target
```
```bash
sudo systemctl daemon-reload && sudo systemctl enable --now pmplatform
# 生命周期交 systemd:勿用 python server.py --stop(依赖 netstat/taskkill,Linux 失效)
```
> 注：app 无 SIGTERM 优雅关闭，systemd `stop` 会硬切 SSE/正在写的 JSON。可接受；若要优雅关闭，给 `main()` 注册 `signal.SIGTERM → httpd.shutdown()`。

### 5.4 nginx 反代 `/etc/nginx/sites-available/pmplatform`
```nginx
server {
    listen 443 ssl;
    server_name pm.intranet.example;            # 或内网 IP
    ssl_certificate     /etc/ssl/pmplatform/server.crt;   # 自签或内网 CA
    ssl_certificate_key /etc/ssl/pmplatform/server.key;

    client_max_body_size 100M;                  # 允许上传 PMIS/CSV 大文件
    gzip on; gzip_types application/json application/javascript text/css; gzip_min_length 1024;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    # SSE 端点必须关缓冲,否则进度流不实时(/api/sync /api/reprocess /api/pmis/* /api/import)
    location ~ ^/api/(sync|reprocess|import|pmis) {
        proxy_pass http://127.0.0.1:8080;
        proxy_buffering off; proxy_cache off; proxy_read_timeout 3600s;
        proxy_set_header Host $host;
    }
}
```
```bash
# 仅放行内网网段访问 443
sudo ufw allow from 10.0.0.0/8 to any port 443
sudo ln -s /etc/nginx/sites-available/pmplatform /etc/nginx/sites-enabled/ && sudo nginx -t && sudo systemctl reload nginx
```

### 5.5 数据与备份
- 备份 `data/accounts.json`（账号/口令哈希）、`data/followup_records.json`、`data/project_tags.json`、`data/pmis_links.json`、`data/analysis_data.json`、`data/history/`。建 cron 每日打包 `data/` 到异机。
- `data/` 目录权限仅 `pmapp` 可写；`input/` 投放新数据后由管理员在页面点"更新数据"重处理。

### 5.6 部署前应用改造点（最小集）
1. **安全 P0-1/P0-2/P0-4**（§1）必须先改：L4 静态绕过、写端点超管门、改默认口令。
2. **P0-5**：`server.py:664` 补平台守卫。
3. **自动开浏览器门控**：`server.py:1962` 的 `_open_browser()` 加 `if os.environ.get('DISPLAY')` 或环境开关，systemd 下禁用（`server.py:1917-1927`）。
4. **隐藏/超管化"停止服务"按钮**：多用户共享服务器上不该让任意管理员停服（`AppHeader.vue:45` 停止按钮 → `/api/stop`）。
