# 项目主域整体看板设计（Phase P）

> 日期：2026-06-10 · 状态：已与用户逐节确认，待审阅后出 P1 实施计划。
> 决策来源：用户决定整体重构展示与内容层；原 S2（回款×项目详情）/ S3（多角色看板）/ P（项目域看板）等路线**保留记录、不再单独推进**，由本次整体重构吸收。
> 视觉底座：全部新页面只用设计底层规范 V2 令牌（`frontend/src/styles/theme.css` + `echartsTheme.ts`，见 `2026-06-10-design-foundation-design.md`），不手写散值；收编的旧页样式不动，直至其重设计期（P6-P8）。

## 1. 目标与使用者

- 平台从「回款跟踪工具」升级为「以**项目**为主域的部门项目看板」；**回款是项目数据的重点子域**——单独展示、重点强调，但导航与信息层级上从属于项目，不与项目数据平级。
- 主要使用者：**部门负责人（交付实施三部）日常管理**——总览 → 发现异常 → 钻取到项目；兼顾**汇报/投屏**（驾驶舱式首页 + 既有三档字号），不做自动轮播大屏。
- 「动态」已确认三义：**项目动态流（时间线）+ 可交互多维透视 + 周期对比**；数据靠手动同步更新（非实时流），变化类功能基于快照 diff。

## 3. 数据架构

### 3.1 输入文件

| 文件 | 内容 | 状态 |
|---|---|---|
| 回款跟进数据 | 现有双通道：在线云文档（Playwright 同步）或离线 xlsx 导入（如「交付中心-全量项目清单-交付实施三部.xlsx」） | 现有，不动 |
| `input/pmis/` 七表 | PMIS 在建+已关闭（S1） | 现有 |
| `input/组织架构.xlsx` | 交付实施三部人员清单，用于筛选主域项目 | **待用户提供** |
| `input/A.xlsx` | 项目映射表：A 列=当前项目号，C 列=已关闭历史项目号（同一项目） | **待用户提供** |
| `input/delivery_analysis.xlsx` | 当前项目详细预算/核算表 | **待用户提供** |

三个新文件的获取沿 U2 模式：离线放置 `input/` + 数据管理页上传通道（与 PMIS 上传同构复用，API 路径在 P1 计划定）。

### 3.2 主表构建 build_projects（preprocess_data.py 新阶段）

1. PMIS 在建全量为底（约 911）。
2. **筛三部**：项目负责人 ∈ 组织架构.xlsx 人员清单（匹配字段=PMIS 项目经理/负责人姓名；同名、未匹配进治理质量数据，见 3.6）。
3. **售前整合**：`售前服务*` 项目按 A.xlsx 挂接已关闭项目信息块——**保留两份信息**，已关闭侧标记「原项目」（`relatedClosedId` 留痕）；不做字段回填合并。统计口径：部门汇总/项目计数以**当前项目**为准，原项目信息仅展示，不参与汇总（避免双计）。
4. **挂回款子域**：rawNodes 按 projectId 聚合 → 计划/已回/待回/完成率/延期数（节点明细仍在 rawNodes，不复制）。
5. **挂预算核算**：join delivery_analysis.xlsx。
6. **派生健康度**：四维三态（规则见 4.6）。

风险明细：P1 摄取 `项目风险数据.xlsx` 时为主域项目**保留记录明细**（供详情页风险 Tab；列名以实际表头为准，属数据依赖非设计未定）。

### 3.3 快照 / diff / 事件流 / 周期对比

- **快照**：每次数据处理成功后写 `data/snapshots/YYYY-MM-DD.json`（同日多次覆盖当日份，保留 90 天，超期清理）。内容为精简状态：每项目（阶段/里程碑状态/项目状态/暂停/评级/风险计数/超支/消耗比），每回款节点（状态/实际回款/计划日），另存一组部门级汇总指标。
- **diff 引擎**（纯函数，pytest 覆盖）：当前 vs 上次快照 → 事件流。事件类型：
  - 项目类：进入/移出主域、阶段变更、里程碑状态变更、项目状态变更、暂停/恢复、评级变化、风险数增减、超支出现/解除。
  - 回款类：到账（actualPayment 增加，含金额）、延期发生、回款完成、计划回款日变更、回款节点新增/移除。
- **事件存储**：`data/events.json` 追加，保留近 500 条；`analysis_data.json` 内嵌最近 100 条供前端。
- **周期对比**：preprocess 预计算三基线差值入 `analysis_data.json`（前端不读快照文件）。基线=上次同步 / 上周（7 天前最近快照）/ 上月（30 天前最近快照）；指标=阶段推进项目数、新增延期节点数、回款新增额、风险净增、新超支项目数、回款达成率变化。基线快照不足时该基线输出空（前端置灰）。
- **节点稳定键**：diff 需要回款节点的稳定标识，候选 projectId+节点名称/期次字段，在 P3 计划中按实际数据确认。

### 3.4 降级策略（任一新文件缺失不致命）

- 新文件读取与上传走 frozen/dev 双分支（CLAUDE.md 第 5 节），两条路径同改。

### 3.6 质量数据（P1 落 JSON 备用，UI 随 P8 治理页重设计）

## 4. 页面设计

### 4.1 首页「项目总览」`/`（布局 2：左主右辅，动态流常驻右栏）

- 左列（约 7/10 宽）从上到下：
  1. **KPI 条**：在管项目数 · 进行中 · 暂停 N · 高风险 N · 超支 N · 回款达成率%（重点色）。
  2. **项目健康度总览**：总评三档计数 + 四维异常计数 + 风险项目卡列表（总评=风险），点卡 → 详情页。
  3. **回款重点带**（视觉强调但层级居中）：年度达成进度条（已回/计划）+ 本月待回款额 + 7 天临期 N + 延期 Top3（金额）；各微块点击钻 `/payment` 子页或详情页。
  4. **风险焦点行**：高风险 / 暂停 / 超支三组横向卡，点击带筛选跳 `/projects`。
- 右栏（约 3/10 宽，全高）：**项目动态流**最近 10 条事件摘要，新变化置顶，点击钻详情页；底部"查看全部 →" `/activity`。
- 窄屏（≤1200px）右栏落底，退化为纵向叙事。

### 4.2 项目详情 `/project/:id`（布局 B：头部 + 左 Tab + 右栏动态）

- **头部**：项目名 · 编号 · 客户 · 合同总额 · 项目经理 · 阶段徽章 · 状态徽章（暂停/评级）；售前整合项目挂**「原项目」徽章**。
- **概览指标条**：完工% · 里程碑状态 · 计划终验 · 风险（数/最高等级）· 预算消耗比 · 回款完成率。
- **左主体 Tab**（回款为**默认 Tab**，体现重点子域）：
  - 回款：项目级回款汇总指标 + 节点明细表 + 跟进记录（复用现有 FollowupRecords 组件链）。
  - 进度里程碑：完工进展 / 项目阶段 / 里程碑进度状态 / 计划终验。
  - 风险：聚合指标（未关闭/总数/最高等级/闭环率）+ 风险记录明细表（3.2）。
  - 预算核算：PMIS 成本字段（总预算/核算/剩余/消耗比/超支）+ delivery_analysis 明细。
  - 原项目（仅售前整合项目显示）：已关闭项目信息块，标记「原项目」；若 rawNodes 存在该已关闭项目号的回款记录则一并展示（标记原项目回款，不计入当前项目汇总）。
- **右栏**：该项目的动态时间线（从 events 按 projectId 过滤），与首页同构。
- projectId 不存在 → 404 空态卡 + 返回项目清单。

### 4.3 项目清单 `/projects`

- 工具栏：搜索（项目名/编号/客户/经理）+ 筛选（阶段 / 项目状态 / 健康度 / 风险等级 / 回款状态 / 是否售前整合）。
- 表格（复用 DataTable）：项目名 · 编号 · 客户 · 项目经理 · 阶段 · 完工% · 风险 · 预算消耗比 · 回款完成率 · 健康度徽章（三态淡底+深字）；售前整合行挂「原项目」小徽章。
- 行点击 → 详情页。不做卡片墙双形态（YAGNI）。

### 4.4 项目动态 `/activity`

- 顶部：**周期对比卡条**——基线切换（上次同步/上周/上月），展示 3.3 的部门级差值；基线不足置灰。
- 主体：**全量事件时间线**（按日分组），筛选事件类型（项目类/回款类）、项目、时间范围；事件点击 → 详情页。
- 首页右栏是本页"最近 10 条摘要"。

### 4.5 项目分析 `/insight`

- 复用 `lib/pivot` 三模式（排名/交叉/透视），数据源从回款节点换成项目主表。
- 维度：阶段 / 项目状态 / 风险等级 / 项目经理 / 行业 / 签约形式 / 健康度；指标：项目数 / 合同总额 / 平均完工% / 平均预算消耗比 / 回款完成率 / 延期项目数。
- 下钻沿用 BoardDrilldownModal 模式 → 项目列表 → 详情页。

### 4.6 健康度规则（四维三态，首页/清单/分析共用）

| 维度 | 异常判定（初版，阈值可调，集中在一个纯函数便于调整） |
|---|---|
| 进度 | 里程碑进度状态含滞后/延期/超期关键词(config.MILESTONE_DELAYED_KEYWORDS,可调) |
| 风险 | 最高等级=高 且 未关闭风险数>0 |
| 成本 | 超支=true 或 消耗比>100% |
| 回款 | 存在延期节点 |

总评：0 项异常=健康 / 1 项=关注 / ≥2 项=风险；徽章用 `--ok/--warn/--danger` 三态淡底+深字（规范 V2）。
- PMIS 未匹配(matched=False)的项目健康度总评为「无数据」,不计入三态统计(P1 的 build_projects 落地)。
