# 回款数据比对报告设计（第一期：诊断）

> 状态：设计讨论中，含 2026-06-15 关键修正（节点级计划回款比例归 PMIS）。
> 这是「回款看板重建」的**第一期**——只做诊断：摸清"人工云文档 vs PMIS 系统"的偏差，并产出**数据源归属清单**（每个看板字段能否 PMIS 化）。第二期（看板重建 + 项目标签体系）据本报告另立 spec。
> 产物为**一次性离线脚本 + 报告文件**，不接入现有管线、不改看板逻辑、不升应用版本。

## 0. 背景与关键修正

历史回款看板建在 WPS 云文档（= `input/交付中心-全量项目清单-交付实施三部.xlsx`，二者完全一致，人工维护）上，节点级"计划/实际回款比例"过去被认为只能手填——**这是误解**。

**关键修正（已实测，2026-06-15）：节点级"计划回款比例"在 PMIS 里。** `input/pmis/在建项目里程碑计划数据.xlsx`、`已结项里程碑计划数据.xlsx` 的 `到货/初验/终验/驻场关联回款阶段` 四列，值形如 `到货款1，70.00%` / `终验款，100.00%`，逗号后的百分比即该节点计划回款比例。`milestones.py` 现仅把它当 `payStage` 标签存，**没解析出比例数字**（解析缺口，非数据缺口）。
- PMIS 两表共 **1823 个项目**含关联回款比例。
- 与云文档节点清单 `关联回款比例` 交集 227 项目、同阶段比例 **一致 215 / 不一致 15**（≈93.5% 吻合）——那 15 处正是要暴露的异常。

由此 PMIS 能客观提供：节点级**计划回款比例**（解析关联回款阶段）+ 节点**计划/实际日期**（里程碑达成）+ **项目级真实到账**（payment_records 流水累计/笔数）+ **合同总额**。
PMIS **仍无**：节点级"实际回款比例"（严格手填值，但可用里程碑达成 + 项目流水重定义）、`加资源可提前`、人工业务分类标签、`纳管`。

## 2. 数据接入与 join

| 源 | 文件 | 解析 |
|---|---|---|
| 云文档主表 | xlsx → `数据源表_全量项目清单 (2)` | 脚本内 openpyxl `read_only=True, data_only=True` 直读，按列名定位 |
| 云文档节点清单 | xlsx → `项目回款节点（里程碑）清单` | 同上（节点级 `关联回款比例`/`实际回款比例`/`里程碑节点`/`该节点计划完成时间`/`是否关联回款`） |
| PMIS 流水 | `input/payment_records.csv` | 复用 `profit.load_payment_records` → `{pid:{total,count,lastDate,records}}` |
| PMIS 里程碑（含计划回款比例） | `input/pmis/*里程碑计划数据.xlsx` | **read_only=False**（PMIS 表 read_only 会截断）；表头在第 2 行；解析 `到货/初验/终验/驻场关联回款阶段` 抽 `阶段→比例`（正则取 `,` 后 `%`） |
| PMIS 合同总额 | `input/pmis/项目状态信息数据.xlsx` | 复用 `pmis.load_project_pmis` 取 `customer.合同总额` |

**主键 = 在建项目编号**（PMIS 当前号空间）。主表行键取 `当前在建项目编号`（实测 ∩流水 320/395；用 `项目编号` 仅 132/395，**必须用当前在建编号**）；节点清单/PMIS 三源用其 `项目编号` 对齐（节点清单命中率脚本内打印核对，低则用主表 `项目编号↔当前在建项目编号` 交叉映射回退）。`keep_ids` = 主表全部当前在建编号集合。

## 3. 数据源归属清单（本期核心产物）

脚本据《看板数据取值条件与计算公式.md》逐字段判定，输出归属总表。结论锁定如下：

| 类 | 字段 | 来源 |
|---|---|---|
| **PMIS 可（以 PMIS 为准）** | 项目编号/金额/合同总额、tier、orgL4/项目经理、里程碑节点名+计划/实际日期、**节点级计划回款比例**（解析关联回款阶段）、是否已达成里程碑（actualDate 非空近似）、项目实际到账（流水累计）、回款笔数、项目回款%（流水÷合同）、签约形式分类（PMIS 字段在） | PMIS |
| **人工 only（无法 PMIS 化）** | **G1** 节点级实际回款比例（严格手填值；可用"里程碑达成+项目流水"重定义）；**G2** 加资源可提前 canAdvance；**G3** 业务分类标签（BH/退换货/0元单/框架/维保；"已100%回款""已关闭"可 PMIS 派生）；**G4** 纳管；**G5** 跟进记录 | 云文档/工具自有 |

> G3/G4 在第二期由"项目标签体系"承载（见 §9）；G1 在第二期改 PMIS 口径；G2 多半可弃。

## 4. 数值比对口径（仅 PMIS↔云文档可对照部分）

| 维度 | 云文档侧（人工） | PMIS 侧（系统） | 异常阈值 |
|---|---|---|---|
| **节点级计划回款比例**（最精准） | 节点清单 `关联回款比例`（逐阶段：到货/初验/终验/驻场） | 解析 PMIS `X关联回款阶段` 的 % | 同项目同阶段 \|差\| > **0.01** 标异常；一侧缺该阶段标"缺节点" |
| **回款百分比（项目级）** | 主表手填 `已回款比例`（空→标"未填"，实测仅 252/712 有值） | `流水累计 ÷ 合同总额` | \|差\| > **10pp** 异常 |
| **回款金额** | 节点 Σ`实际回款金额`=Σ(项目金额×关联回款比例×实际回款比例)；并列主表 `合同×手填比例` | `payment_records.total` | \|差\| > **5 万** 或相对差 >20% |
| **回款笔数** | 已回款节点数（实际回款比例≥1） | `payment_records.count` | 不等仅标记（语义不同，不计分） |
| **回款节点/里程碑个数** | 节点清单该项目 `是否关联回款`=是 行数 | PMIS 里程碑 `关联回款阶段` 非空条数 | 不等标记 + 列各自节点名 |
| **回款里程碑日期** | 节点清单 `该节点计划完成时间`（按里程碑节点名归类） | PMIS 里程碑对应阶段 `计划/实际时间` | 按阶段对齐 \|计划日期差\| > **30 天** 异常 |

## 5. 异常分级

每项目按命中的**强异常维度数**（计：节点计划比例、回款百分比、回款金额、里程碑日期）汇总：红 ≥2 / 黄 =1 / 绿 =0。另设独立标记位：`手填比例未填`、`PMIS无流水`、`PMIS无合同总额`、`PMIS无关联回款比例`、`节点个数不一致`、`笔数不一致`。阈值（0.01 / 0.10 / 5万 / 0.20 / 30天）为脚本顶部常量可调。

## 6. 报告结构

- **数据源归属清单.md**：§3 总表 + 每字段一句"为何 PMIS 可/不可" + G1-G5 对第二期的处置建议。
- **逐项目.csv** 列：`pid, 项目名称, L4, 合同总额, 手填已回款比例, PMIS流水比例, 比例差, 比例异常, 节点比例分歧阶段(列表), 云已回款金额, PMIS流水累计, 金额差, 金额异常, 云已回款节点数, PMIS笔数, 笔数一致, 云回款节点数, PMIS关联回款里程碑数, 个数一致, 里程碑日期异常阶段, 强异常维度数, 等级, 标记位`。
- **汇总.md**：参与项目数/红黄绿数/各标记位占比（含手填未填率、PMIS无流水率、节点比例分歧项目数）；各维度偏差分桶；Top 异常清单；域外流水项目清单；末尾附阈值与口径说明。

## 7. 实现要点

- **节点比例解析**：纯函数 `parse_pay_stage_ratio("到货款1，70.00%") -> ("到货", 0.70)`，对 PMIS 四个关联回款阶段列各取一；可先在脚本内实现，第二期再决定是否回灌 `milestones.py`。
- 复用 `profit`/`pmis` 解析器取流水与合同总额；PMIS 里程碑用 read_only=False、表头第 2 行。
- 空值语义：手填比例空→None+标"未填"（不当 0）；PMIS 无流水→量 0、比例 None+标记。
- 纯函数（比例/金额解析、差异、分级、日期分桶）抽出便于 pytest；I/O 与报告写出分离；常量集中脚本顶部。
