# S1 反馈修缮批次设计（项目动态/清单/详情/回款口径/分析配色）

> 用户实测反馈批次（2026-06-12，Phase R 收官后）。版本 **V1.0.1**（新版本策略：局部批次走第三位）。
> 性质同 P5.5：调查答疑 + 修缮，单分支单期交付。

## 0. 调查结论（实测，用户「需确认内容」的答案）

1. **完工% 出处正确无需调整**：pmis.py:171 取状态表「项目累计完工进展百分比」= 项目状态信息数据.xlsx 的 **Z 列**（idx25，已关闭表同列）。
2. **回款完成率需切口径**：现行 = 云文档节点 Σ已回款÷Σ计划回款。脏数据案例 WSGF-SF-202502100199：实回 124.65 万 > 计划 115.15 万 → 108.25%；新口径（payment_records 累计 115.15 万 ÷ 原项目合同总额 115.15 万）= 100% ✓。
3. **清单卡慢根因**：633 行 × 14 列一次性全量 DOM 渲染（el-table 无虚拟化，~9000 单元格 + 每格 overflow-tooltip），搜索每键全表重排；数据计算毫秒级非瓶颈。
4. **科目树缺行根因**：R1 剪枝规则「level>1 且四指标全 None/0 丢弃」——实测单项目 CSV 24 科目剩 12 行（2.1.2/2.2.1/2.2.3-2.2.5/2.3.1/2.3.3-2.3.5/2.4.2-2.4.4 被剪）。
5. **主域事件语义**：进入/移出主域 = 项目出现/消失于「PMIS 在建 ∩ 交付三部」集合，主因即新项目立项/项目关闭出库（极少数为数据修复入域）。
6. **PMIS 无超支金额列**：超支标志来自项目中心表六个分项 bool（是否人工/直接/安服/产线/交付/其他部门成本超支）；整体「核算−总预算」38/45 超支项目为负，不可用作金额。
7. 里程碑进度状态全枚举（实测）：正常 843 / 严重延期 304 / 延期 8 / 超期未发布 7——异常三值与用户标红清单完全一致。
8. direct 顶部「剩余预算」列与「预算成本−实际成本」在 225/632 项目不一致（PMIS 顶部列不自洽）→ 金额一律两列自算。
9. 售前桥接科目 2「项目成本」剩余预算 276/276 全可得。

## 1. 用户决策记录（2026-06-12 确认）

1. **超支金额双口径**（用户定义）：**整体超支金额**——非售前 = direct 顶部 `实际成本 − 预算成本`；售前服务\* = `当前消耗（direct 实际成本）− 原剩余预算（bridge 科目 2 剩余预算）`；>5000 元红色告警（≤5000 黄，沿原反馈阈值）。**交付费用超支**——delivery_analysis.csv 任一类目 实际发生>预算金额 即判定（无阈值），独立事件。
2. **卡慢修复=分页**：页大小 20/50/80/100 可选，默认 50（实现方定）。
3. **健康度定义落位**：清单健康度列头 info 悬浮说明 + 关于页「健康度规则」段。
4. **主域事件改名**：「新增项目/关闭项目」标绿，摘要保留"(进入/移出项目主域)"副注。

## 2. 项目动态（snapshots.py + schema.Event + 前端染色）

- **Event 模型 += `tone: str = ''`**（'ok'|'warn'|'danger'|''）。后端 diff 决定 tone，前端（EventTimeline/ActivityView）按 tone 给类型徽章染色（淡底+深字三态，规范 V2）。
- 规则表：

| 事件 | 变更 |
|---|---|
| 评级变化 | **不再生成**（快照 rating 字段保留备查） |
| 延期发生 | tone=danger |
| 进入主域 | type=「新增项目」 tone=ok，摘要「新增项目（进入项目主域）」 |
| 移出主域 | type=「关闭项目」 tone=ok，摘要「关闭项目（移出项目主域）」 |
| 里程碑状态变更 | curr∈{超期未发布,严重延期,延期} → danger；恢复正常 tone='' |
| 风险数增减 | 增 danger / 减 ok |
| 超支出现 | 摘要带整体超支金额（万）；金额>5000 元 danger、≤5000 warn；超支解除 ok |
| **交付费用超支**（新） | deliveryOver 由 false→true 触发，danger，摘要列出超支类目名 |

- **快照项目条目 += `overspendAmount`（整体超支金额，元，按 §1.1 口径，纯函数 pytest 先行）与 `deliveryOver`（bool）**；build_snapshot 需新增入参（projectProfit map + 项目 deliveryCosts 已在 projects 内）。
- 其余事件类型（阶段变更/项目状态变更/暂停/恢复/到账等）tone='' 不变。

## 3. 项目清单（/projects）

- **分页**：el-pagination，page-size 可选 [20,50,80,100] 默认 50；排序/筛选/搜索作用于全集后分页；筛选或搜索变化时回到第 1 页。
- **项目名称列去 sortable**。
- **筛选全部改多选**（el-select multiple collapse-tags）：新增 项目经理、项目级别 两个筛选；ProjectFilters 各键 string→string[]（presale/paused/overspend 保持单值）；URL query 兼容：单值（总览 KPI 链 `?riskLevel=高` 等）初始化为单元素数组，多值用重复参数。
- **筛选条顺序对齐列序**：搜索框 → 项目经理 → 服务组 → 阶段 → 风险 → 级别 → 回款状态 → 健康度 → 售前。
- **删客户列**（详情页保留客户信息）；搜索仍匹配客户字段（placeholder 不变）。
- **健康度定义**：列头 label 旁 info 图标（el-tooltip）展示四维异常+三档总评规则；关于页「工具组」后新增「健康度规则」段（同文）。规则原文：进度=里程碑进度状态含滞后/延期/超期关键词；风险=最高等级高且未关闭风险数>0；成本=超支或消耗比>100%；回款=存在延期回款节点；总评 0 项异常=健康 / 1 项=关注 / ≥2 项=风险（PMIS 未匹配=无数据，不计三态）。

## 4. 详情页（/project/:id）

- **科目树剪枝改「仅剪全 None」**：profit.py parse_profit_rows 的 level>1 剪枝条件由「全 None/0」改「全 None」——全零科目保留（恢复 2.2.1 等全量科目，单项目 12→24 行；体积增量记 L-18）。
- **进度里程碑去三色**：MilestoneTable 行级三色底/深字样式删除，恢复通用表格行色；`priority` 字段保留在数据层（后续统计用），UI 不再体现。

## 5. 回款完成率口径切换（项目域）

- `Project.payment.paymentRatio` 改：**payment_records 累计回款 ÷ 合同总额**；售前服务\* 无合同总额 → 取原项目（relatedClosedId）合同总额；分母缺失/为 0 → null（前端显 '-'）。
- 实现位：preprocess 9e 段载入 payment_records 后回填 dept_projects 的 paymentRatio（projects.py 提供纯函数 `payment_ratio_from_records(records_total, contract, closed_contract)`）。
- **降级**：payment_records 文件未提供 → 保留旧口径（Σ实回÷Σ计划）不回填。
- **影响面**：消费 `Project.payment.paymentRatio` 的位置——清单「回款完成率」列、详情页头部指标与回款 tab 汇总 chips 的完成率（出处标注补「完成率=回款流水累计÷合同总额」）、清单回款状态四态中「已回清」判定不变（基于节点 remaining/actual）。**不受影响**：/insight 回款完成率指标（分组 Σ实回÷Σ计划，云文档口径）、回款域五页与首页回款达成率（云文档口径）——两口径并存。健康度回款维（延期节点）不变。
