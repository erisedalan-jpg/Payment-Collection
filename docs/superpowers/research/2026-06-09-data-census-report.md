# 数据普查报告(input/ 全量)

## 一、一页结论

- **回款主题已可全量上线(▾绿)**：回款节点的计划/实际时间、关联比例、分层后金额(100%填充)齐全，覆盖约 100% 回款项目，足以支撑回款率、待回款额、逾期预警与按季度切片看板。前提是先做三件 P0 治理。
- **三件 P0 必须先做**：① 把散在 5+ 张表的回款节点合并去重为单一"项目×节点"事实表；② 建 SF(售前)↔SS(实施)编号映射，补齐 166 个断链项目；③ 回款比例数值化并对 >1 异常值(实测 max=2.65)封顶核查。三项打通后回款盘即可全量上线。
- **成本/进度为黄(▾)**：靠 PMIS 的总预算/核算/超支布尔(100%填充)可自算成本健康度与超支标记，但仅覆盖 51%~73% 项目，且成本状态、完工进展过半需源头回填——这是它们转绿的唯一卡点。
- **风险主题为红(✕)**：结构化风险记录仅 138 条/81 项目，与回款项目交集仅 **44/628≈7%**，项目级风险评级几乎全空(3%~4%)，短期内无法做全盘风险分析，只能做台账。
- **同名列口径普遍冲突**：成本状态、项目状态、项目金额、完成度等同名列跨表取值域/填充率不一，join 后取值矛盾。核验证实"成本状态"取值域跨表一致但填充率不同；而"项目金额"同名列在项目状态信息中**根本不存在**(其金额列是项目总预算)，二者是不同概念，不可对齐相加。

> **一句话总判断**：这套数据"回款盘可立即落地、成本与进度可随 PMIS 回填渐次转绿、风险盘暂不可全盘"——先治理回款事实表 + SF/SS 映射 + 比例数值化这三件 P0，回款看板即可上线，其余主题靠源头补录与工具侧自算逐步补齐。
