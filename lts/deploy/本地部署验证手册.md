# 本地部署验证手册

> 目标：在开发机或验收机（Windows / macOS / Linux 均可）上把「项目管理平台」精简版（LTS-1.0.0）跑起来，并完成一轮功能验证。
> 适用场景：日常开发联调、版本验收、离线单机使用。

## 1. 前置环境

- Python 3.8 及以上
- Node.js 18 及以上（含 npm）

检查：

```bash
python --version      # 或 python3 --version，期望 3.8+
node --version         # 期望 18+
npm --version
```

## 2. 安装依赖

在仓库根目录（`lts/`）执行：

```bash
# 后端依赖
pip install -r requirements.txt

# 前端依赖
cd frontend
npm install
cd ..
```

## 3. 构建前端

```bash
cd frontend
npm run build
cd ..
```

构建产物在 `frontend/dist/`，`server.py` 会以此作为静态页面来源。

> 开发联调（非验收）时也可以不构建，改用 `python server.py` 起后端 + `cd frontend && npm run dev` 起 Vite 开发服务器（默认 `:5173`，已代理 `/api`、`/data` 到 `:8080`）。

## 4. 准备数据

把数据源文件放到仓库根目录下的 `input/`：

| 文件 | 存放位置 |
|---|---|
| PMIS 九表（`*.xlsx`） | `input/pmis/` |
| `collection_stages.csv`（收款阶段台账） | `input/` |
| `payment_records.csv`（回款流水） | `input/` |
| `profit_loss_direct.csv` / `profit_loss_bridge.csv`（预算科目树） | `input/` |
| 预算科目对照数据 CSV（预算/核算两版本，文件名以「数据管理」页面文件清单为准） | `input/` |
| `delivery_analysis.xlsx`（预算核算明细） | `input/` |
| `组织架构.xlsx`（项目花名册） | `input/` |
| `A.xlsx`（售前 ↔ 原项目映射） | `input/` |
| `TOP1000.xlsx`（重点客户名录） | `input/` |

> 完整输入文件清单以「数据管理」页面「文件状态」区实时展示为准，本表仅列常见文件便于核对。

也可以先不放数据，起服务后在页面「数据管理」（`/data`）用「上传」功能补齐，效果等价。

## 5. 启动服务

```bash
python server.py
```

默认会自动打开浏览器访问 `http://localhost:8080`；未自动打开则手动访问该地址。

停止服务：

```bash
python server.py --stop
```

## 6. 登录与更新数据

1. 打开 `http://localhost:8080`，用账号登录（首次启动会建立种子超管账号，见 `auth.py`）
2. 进入「数据管理」（`/data`）页面
3. 确认页面上「文件状态」区已识别到第 4 步放置的文件（或用页面上传功能补齐）
4. 点击「更新数据」，等待 SSE 进度条走完（重跑 `preprocess_data.py`，生成 `data/analysis_data.json`）

## 8. 常见问题

- **前端页面打不开 / 白屏**：确认已执行 `cd frontend && npm run build`，且 `frontend/dist/index.html` 存在。
- **更新数据后指标未变化**：确认目标文件确实放在 `input/` 或 `input/pmis/` 下且文件名匹配约定命名，「数据管理」页文件状态区会显示最近修改时间。
- **8080 端口被占用**：先执行 `python server.py --stop` 尝试停止已运行实例，或检查是否有其它进程占用该端口。
