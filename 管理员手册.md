# 项目回款跟踪与管控平台 - 管理员手册

> 版本：V5.9.2 | 更新日期：2026-06-02

---

## 1. 安装与部署

### 1.1 环境要求
- **操作系统**：Windows 10/11
- **Python**：3.8 或更高版本（运行本程序需要 Python 环境）
- **浏览器**：Chrome 或 Edge 浏览器（同步数据时必须，系统自带Edge即可）
- **网络**：同步 WPS 云文档时需要内网访问权限

### 1.2 依赖安装
首次运行前，需安装以下 Python 依赖包：

```bash
pip install playwright openpyxl xlrd chardet beautifulsoup4 lxml
```

安装 Playwright 后还需初始化浏览器驱动（用于云文档同步）：

```bash
python -m playwright install chromium
```

> ⚠️ 如果系统已安装 Chrome 或 Edge 浏览器，Playwright 会优先使用系统浏览器，无需额外安装 Chromium。

### 1.3 安装步骤
1. 将本文件夹复制到本地任意目录（路径中避免包含中文和空格）
2. 确认 Python 已安装且依赖包已安装
3. 文件夹内应包含：
   - `server.py` — 主服务程序
   - `项目回款跟踪与管控平台_启动.bat` — 启动脚本（双击运行）
   - `停止服务.bat` — 停止服务脚本
   - `sync_data.bat` — 独立数据同步脚本
   - `用户手册.md` — 用户手册
   - `管理员手册.md` — 本手册

### 1.4 启动方式

**方式一：双击启动脚本（推荐）**
双击 **`项目回款跟踪与管控平台_启动.bat`**，脚本会自动：
1. 检测 Python 是否安装
2. 清理端口 8080 上的旧进程
3. 后台启动服务（无CMD窗口）
4. 等待 3 秒后自动打开浏览器

**方式二：命令行手动启动**
```bash
# 前台运行（可看到日志输出，Ctrl+C停止）
python server.py

# 后台运行（无CMD窗口）
pythonw server.py
```

启动后浏览器访问：**http://localhost:8080**

程序启动后会在同目录下自动创建以下文件夹：
| 目录 | 说明 | 创建时机 |
|------|------|----------|
| `log/` | 日志文件（保留7天） | 启动时自动创建 |
| `data/` | 业务数据文件 | 首次同步/导入数据时创建 |
| `yundocs_data/` | 云文档原始数据 | 首次云同步时创建 |

### 1.5 停止服务
关闭浏览器页面不会停止服务。需通过以下方式停止：
- **页面停止按钮**：点击页面右上角红色停止按钮⏹️，确认后服务自动停止
- **双击停止脚本**：双击 `停止服务.bat`
- **命令行停止**：运行 `python server.py --stop`
- **Ctrl+C**：如果使用 `python server.py` 前台运行，直接按 Ctrl+C 停止
- 在系统任务管理器中结束 `pythonw.exe` 或 `python.exe` 进程

> ⚠️ 服务停止后，浏览器中的页面将无法访问。下次使用时重新启动即可。

### 1.6 首次使用引导
首次启动后页面为空白（无数据），需要先获取数据：

**方式一：WPS云文档同步（页面操作）**
1. 点击左侧菜单"数据管理"
2. 在"WPS云文档同步"区域输入WPS云文档网址
3. 点击"开始同步"，等待同步完成
4. 页面自动刷新展示数据

**方式二：离线Excel导入（页面操作）**
1. 点击左侧菜单"数据管理"
2. 在"离线Excel导入"区域点击"选择文件"
3. 选择Excel文件后点击"开始导入"
4. 导入完成后页面自动刷新

**方式三：命令行数据同步**
```bash
# 双击 sync_data.bat 或命令行运行：
python fetch_yundocs_full.py [云文档URL]
python preprocess_data.py
```
> 此方式适用于无需启动Web服务、仅需要更新数据的场景。

> ⚠️ 云同步与离线导入互斥，不可同时使用以避免数据错乱。

### 1.7 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 启动脚本提示"未检测到Python" | Python未安装或未加入PATH | 安装Python 3.8+，安装时勾选"Add Python to PATH" |
| 启动后浏览器打开但页面空白 | 服务尚未启动完成 | 等待5秒后刷新页面 |
| 提示端口8080被占用 | 其他程序占用端口 | 关闭占用程序，或重启会自动清理旧进程 |
| 首次打开页面无数据 | 尚未导入数据 | 先同步云文档或导入Excel文件 |
| Python运行报错ModuleNotFoundError | 依赖包未安装 | 运行 `pip install playwright openpyxl xlrd chardet beautifulsoup4 lxml` |
| 同步连接中断 | 浏览器未安装或网络问题 | 确认已安装Chrome/Edge，检查云文档地址和网络 |
| 同步失败：未检测到Chrome浏览器 | 系统未安装Chrome且Edge不可用 | 安装Google Chrome或确认Edge可用后重试 |
| pip安装依赖失败 | 网络或pip配置问题 | 检查pip版本和网络连接，尝试 `pip install --upgrade pip` |

### 1.8 日志排查
日志文件位于程序同目录的 `log/server.log`，保留7天自动轮转。管理员可通过日志排查以下问题：
- 同步失败的具体错误信息
- HTTP请求异常（404等）
- 服务启停记录
- 浏览器启动状态
- 数据预处理过程日志

## 2. 计算公式与判定规则

### 2.1 数据来源
系统读取WPS云文档中3个回款节点清单Sheet页：
- **50万以下项目回款节点清单**
- **50-100万项目回款节点清单**
- **100万以上项目回款节点清单**

### 2.2 节点状态判定规则（6种状态）

仅当"是否关联回款"=**是**的节点参与状态判定；"是否关联回款"=否的节点保留原始数据，nodeStatus为空，金额字段为0。

> **100万以上区间**默认所有节点"是否关联回款"=是，全部参与状态判定。

状态按以下优先级顺序判定（命中即停止）：

| 优先级 | 状态 | 判定条件 | 说明 |
|--------|------|----------|------|
| 1 | **加资源可提前** | "是否增加资源是否可以提前完成里程碑计划"=是 **且** "当前项目完成%"<100% **且** "实际回款比例"<100% | 具备加资源提前的潜力 |
| 2 | **达到回款条件** | "当前项目完成%"=100% **且** "是否已达成里程碑"=是 **且** "实际回款比例"<100% | 项目完成但尚未回款 |
| 3 | **已提前回款** | "该节点计划完成时间">当前时间 **且** "实际回款比例">=100% | 在计划时间前已全额回款 |
| 4 | **已全额回款** | "实际回款比例">=100% | 已完成回款（含按时和逾期后回款） |
| 5 | **延期** | "该节点计划完成时间"<当前时间 **且** ("当前项目完成%"为空 或 <100%) **且** "实际回款比例"<100% | 已超过计划时间但未完成 |
| 6 | **正常实施中** | 以上条件均不满足（兜底） | 尚在正常执行中 |

#### 字段名对应关系

| 通用字段名 | 50万以下/50-100万 Sheet | 100万以上 Sheet |
|------------|------------------------|----------------|
| 计划完成时间 | 该节点计划完成时间 | 阶段计划收款日期 |
| 实际回款比例 | 实际回款比例 | 阶段实际回款比例 |
| 计划回款比例 | 关联回款比例 | 阶段计划回款比例 |
| 是否可提前 | 是否增加资源是否可以提前完成里程碑计划 | 是否增加资源是否可以提前完成里程碑计划 |

### 2.3 延期天数计算

仅**延期**状态节点计算延期天数：
```
延期天数 = max(0, 当前日期 - 该节点计划完成时间)
```

### 2.4 金额计算公式

#### 节点级别金额

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| **计划回款金额** (expectedPayment) | `项目金额 × 关联回款比例` | 仅"是否关联回款"=是时计算，否则为0 |
| **实际回款金额** (actualPayment) | `项目金额 × 关联回款比例 × 实际回款比例` | 仅"是否关联回款"=是时计算 |

#### 汇总级别金额

| 指标 | 计算公式 | 说明 |
|------|----------|------|
| **计划回款总金额** | `Σ(所有关联回款节点的计划回款金额)` | 单位：元，展示时转为万元 |
| **已回款总金额** | `Σ(所有关联回款节点的实际回款金额)` | 单位：元，展示时转为万元 |
| **待回款总金额** | `计划回款总金额 - 已回款总金额` | 单位：元，展示时转为万元 |
| **总完成率** | `已回款总金额 / 计划回款总金额` | ≥80%绿色、≥50%橘黄、<50%红色 |

#### 完成率颜色规则
| 范围 | 颜色 |
|------|------|
| ≥80% | 🟢 绿色 |
| ≥50% 且 <80% | 🟠 橘黄 |
| <50% | 🔴 红色 |

### 2.5 比例值解析规则

原始数据中的比例字段可能存在多种格式，统一按以下规则解析为0-1小数：

| 原始格式 | 示例 | 解析结果 |
|----------|------|----------|
| 百分比格式 | "50%" | 0.5 |
| 0-100整数 | "80" | 0.8 |
| 0-1小数 | "0.8" | 0.8 |
| 空值/待上报 | "" / "待上报" | None（标记为待上报） |

### 2.6 日期判定规则

| 函数 | 判定逻辑 |
|------|----------|
| `is_past(日期)` | 日期 < 当前日期 → 已过期 |
| `is_future(日期)` | 日期 > 当前日期 → 尚未到期 |

日期解析支持：`YYYY-MM-DD`、`YYYY/MM/DD`、`YYYY年MM月DD日`、`YYYY.MM.DD`，以及Excel序列号格式（40000-60000范围）。

### 2.7 风险项目判定规则

| 风险类型 | 判定条件 |
|----------|----------|
| **临近到期节点** | 7天内到期 且 未100%回款 |
| **可提前但未行动** | 具备加资源提前条件 但 未行动 |
| **高金额低完成率** | 完成率<30% 且 金额最高 |

### 2.8 数据完整性检查规则

| 检查项 | 判定条件 |
|--------|----------|
| 缺少项目金额 | 项目金额字段为空或为0 |
| 实际回款比例待上报 | 实际回款比例为空或"待上报" |
| 缺少项目经理 | 项目经理字段为空 |
| 缺少服务组 | PM L3-1部门字段为空 |
| 回款比例>100% | 实际回款比例超过100% |

> 50万/50-100万区间仅检查"是否关联回款"=是的节点；100万以上区间检查全部节点。

### 2.9 周期切换计算逻辑

| 切换方式 | 计算逻辑 | 说明 |
|----------|----------|------|
| **年度切换** | 累计计算 | 从年初至所选年度末的所有数据累加 |
| **季度切换** | 范围过滤 | 仅统计所选季度范围内的数据 |

## 3. 功能模块概览

### 3.1 看板首页
- 顶部汇总卡片：项目总数、计划回款总金额(万)、已回款总合计(万)、待回款总金额(万)、总完成率
- 周期切换：全部 / 本年度 / 下一年度 / 各季度
- 金额区间卡片：按100万以上 / 50-100万 / 50万以下分别展示
- 季度待回款、月度待回款（堆叠条形图）
- 服务组回款达成排名
- 延期项目 Top10

### 3.2 回款台账
统一展示所有金额区间的项目，不响应年份/季度切换。

### 3.3 回款日历
独立年份切换逻辑，双月视图，状态热力图，15天/30天到期节点展示。

### 3.4 项目总览 / 回款节点 / 回款状态 / 风险项目 / 数据质检
各金额区间的详细页面，支持筛选、搜索、设置展示字段。

### 3.5 项目经理视图
按项目经理汇总，支持展开负责项目信息和延期节点信息。

### 3.6 区间对比
三个金额区间的对比分析（达成对比、进度对比、状态分布、月度趋势、服务组排名）。

### 3.7 数据管理
WPS云文档同步、离线Excel导入、清空数据、数据质量总览、数据缺失检查。

## 4. 运维管理

### 4.1 配置参数
- **服务端口**：8080（可在 server.py 中修改 `PORT = 8080`）
- **云文档默认URL**：可在 fetch_yundocs_full.py 中修改 `DEFAULT_URL`
- **日志保留天数**：7天（可在 server.py 中修改 `backupCount=7`）

### 4.2 数据文件管理
| 文件/目录 | 作用 | 可否删除 |
|-----------|------|----------|
| `data/analysis_data.js` | 前端展示数据 | 删除后需重新同步/导入 |
| `yundocs_data/` | 云文档原始提取数据 | 删除后需重新同步 |
| `log/server.log` | 运行日志 | 自动轮转，可手动清理 |
| `yundocs_data/screenshot_*.png` | 同步错误截图 | 可安全删除 |

### 4.3 数据同步流程

**WPS云文档同步（页面触发）**：
1. 前端输入云文档网址，点击"开始同步"
2. 系统自动启动浏览器，连接WPS云文档
3. 提取各Sheet页数据，保存原始数据到 `yundocs_data/`
4. 自动进行数据预处理，生成前端展示数据
5. 页面自动刷新展示最新数据

**离线Excel导入（页面触发）**：
1. 前端选择Excel文件，点击"开始导入"
2. 系统解析Excel数据，保存到 `yundocs_data/`
3. 自动进行数据预处理，生成前端展示数据
4. 页面自动刷新展示数据

**命令行同步（独立脚本）**：
```bash
# 方式一：分步执行
python fetch_yundocs_full.py [云文档URL]
python preprocess_data.py

# 方式二：一键同步
sync_data.bat
```

### 4.4 打包为独立exe
如需在无Python环境的机器上部署，可将项目打包为独立可执行文件：

```bash
# 安装打包工具
pip install pyinstaller

# 执行打包
pyinstaller PaymentReviewApp.spec
```

打包完成后，`dist/` 目录下会生成 `PaymentReviewApp_v5.9.2.exe`（约76MB），可在无Python环境的Windows机器上直接运行。

> ⚠️ 打包后的exe运行时，系统仍需安装 Chrome 或 Edge 浏览器才能使用云文档同步功能。

## 6. 技术支持

如遇问题，请查看 `log/server.log` 日志文件获取详细错误信息。

运维排查要点：
- 服务启动失败：检查Python版本、端口占用、依赖包安装
- 同步失败：检查浏览器可用性、网络连通性、云文档URL有效性
- 数据异常：检查 `yundocs_data/` 原始数据完整性、`data/analysis_data.js` 是否生成
- 前端异常：浏览器F12开发者工具查看Console错误
