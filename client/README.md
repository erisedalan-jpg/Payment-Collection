# 奇安信看板 Cookie 工具

> 两种用法：**（推荐）无界面本机代理**——装一次、常驻，之后在平台 `/data` 网页点按钮取 cookie；**（兜底）桌面 GUI**——手动逐步操作。

## A. 无界面本机代理（配合平台 /data 网页，推荐，V2.7.1 起）

平台 `/data` 页的「获取本机 PMIS cookie / 倚天 cookie」按钮需要本机跑一个小代理替浏览器取 cookie（浏览器自身取不到零信任 cookie）。

- 启动：`python cookie_agent.py`（前台占终端，关窗口即停）；**推荐双击本目录 `启动代理.vbs`**（用 pythonw 隐藏静默拉起、无终端窗口），或 `pythonw cookie_agent.py`。常驻监听 `127.0.0.1:8765`，只接受来自平台源 `Origin`、且 `Host` 为 `127.0.0.1/localhost` 的请求（内网安全 + 防 DNS rebinding）。
- 停止：任务管理器结束 `pythonw.exe`；或 `taskkill /f /im pythonw.exe`。判断是否在跑：`netstat -ano | findstr 8765`，或 `/data` 页显示「本机代理：已连接」。
- 开机自启：把 `启动代理.vbs`（或 exe）的**快捷方式**放进 Windows「启动」目录（`Win+R` → `shell:startup`）→ 开机隐藏常驻。
- 配置（可选）：同目录 `agent_config.json` 可覆盖 `{"port": 8765, "allowed_origins": ["http://<平台IP>"]}`；缺省已含平台 IP。
- 用法：代理在跑 + 本机零信任已登录 PMIS/倚天 → 到平台 `/data` 页点「获取本机 PMIS cookie 并推送」/「获取本机倚天 cookie 并存储」即可；页面会显示「本机代理：已连接」。
- 前提：开平台网页的机器 = 装零信任、能取 cookie 的机器（同一台）。
- 依赖：`requests`（`pip install requests`）。倚天 cookie 目前只「取到备用」，倚天数据功能后续开发。

## B. 桌面 GUI（手动兜底）

双击 `奇安信Cookie工具.exe` 启动。

界面五区:① 看板平台接口配置 ② PMIS Cookie ③ 倚天 Cookie ④ 平台操作 ⑤ 运行日志

前置:本机零信任客户端已登录(登 PMIS 就能取 PMIS cookie,登倚天就能取倚天 cookie)。

## 一、更新 PMIS 数据

1. **① 区**:填平台地址(默认已是)、账号、密码 → 点「登录」,日志显示「登录成功」
2. **② 区**:点「获取cookie」,等日志显示「PMIS Cookie 获取成功」→ 点「推送到平台」,日志显示「推送成功」
3. **④ 区**:点「下载数据」(或「更新数据」/「下载&更新」),日志区实时显示进度,直到「完成」

## 二、更新倚天数据

1. **① 区**:填平台地址、账号、密码 → 点「登录」,日志显示「登录成功」
2. **③ 区**:点「获取cookie」,等日志显示「倚天 Cookie 获取成功」→ 点「复制Cookie」
3. 手动将复制的倚天 cookie 粘贴到平台对应位置

> 说明:④ 区的「下载数据 / 更新数据 / 下载&更新」只针对 PMIS 数据,暂不支持倚天 cookie。倚天 cookie 需复制后手动粘贴到平台。

## 权限说明

- ④ 区的「下载数据 / 更新数据 / 下载&更新」**仅超级管理员可操作**。
- **勾选「平台已登录」**时,客户端自动检查内存 session 的权限:
  - 超级管理员 → 按钮可点击,提示「超级管理员,可操作」
  - 普通用户 → 按钮置灰不可点击,提示「非超级管理员,操作受限」
  - 内存 session 无效 → 按钮置灰,请取消勾选用账号密码登录
- **输入账号密码登录**后,同样按账号的 `isSuper` 字段控制按钮。
- 普通用户即使绕过按钮直接调用,后端也会拒绝。
